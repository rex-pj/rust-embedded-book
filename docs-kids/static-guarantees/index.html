🔍 Static Guarantees là gì?

🦀 Static nghĩa là "trước khi chạy" — Rust kiểm tra code của bạn ngay lúc bạn đang viết (lúc compile), trước khi code được chạy lần nào!

Hầu hết ngôn ngữ khác chỉ phát hiện lỗi khi chạy (runtime errors). Rust bắt rất nhiều loại lỗi tại compile time! ⚡

🌟 Ví von: Giáo viên siêu giỏi

Hãy tưởng tượng hai trường học:

  • 🏫 Trường A: Học sinh nộp bài → chờ kết quả → biết sai → sửa → nộp lại (mất nhiều thời gian!)
  • 🏆 Trường B: Giáo viên nhìn qua bài làm ngay khi đang viết, chỉ ra lỗi trước khi nộp! (hiệu quả hơn!)

Rust là Trường B — compiler như giáo viên siêu giỏi đứng bên cạnh! 👨‍🏫

Rust bắt lỗi khi bạn đang viết code — C/C++ chỉ thấy lỗi khi robot đã chạy rồi (muộn quá!)

🏗️ 3 Cơ Chế Bảo Đảm

👑
Ownership (Quyền sở hữu)
Mỗi thứ chỉ có 1 chủ! Không có "hai người cùng sở hữu và sửa một thứ" → không có conflict! 🔒
🏷️
Type System (Hệ thống kiểu)
Kiểu dữ liệu mang thông tin trạng thái! Pin<Output> khác Pin<Input> — compiler biết được! 🎯
🚪
Access Control (Kiểm soát truy cập)
Ai được làm gì — pub, private, unsafe. Không ai truy cập trái phép! 🛡️

💡 Ví Dụ Thực Tế

Bật đèn LED — 3 bước:

  1. 🔌 Lấy quyền điều khiển GPIO peripheral: Peripherals::take()
  2. 🔧 Cấu hình pin thành Output: pin.into_push_pull_output()
  3. 💡 Bật đèn: output_pin.set_high()

Nếu bạn bỏ qua bước 2 và gọi set_high() trên pin Input → Compiler báo lỗi ngay!

Kiểu Pin<Input> không có method set_high() — không cần chạy code để biết sai! ✅

Loại lỗi C/C++ Python Rust
Sai kiểu dữ liệu ⚠️ Compile / Runtime ❌ Runtime ✅ Compile time

Memory leak ❌ Runtime ✅ GC lo ✅ Ownership lo
