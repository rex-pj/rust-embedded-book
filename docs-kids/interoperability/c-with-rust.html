🏷️ #[repr(C)] — Nói Tiếng C

Rust mặc định sắp xếp dữ liệu theo cách của mình. Nhưng khi dùng C, cần đảm bảo cùng cách sắp xếp:

#[repr(C)] → nói với Rust: "Sắp xếp struct này theo kiểu C"

Ví dụ:

  • C: struct SensorData { int temp; float humidity; }
  • Rust: #[repr(C)] struct SensorData { temp: i32, humidity: f32 }

Hai struct này có cùng layout trong bộ nhớ → có thể truyền qua lại giữa C và Rust! ✅

📢 extern "C" — Hàm Từ Thư Viện C

Khai báo hàm C trong Rust:

  • extern "C" { ... } — block khai báo các hàm nước ngoài
  • fn read_temperature() -> f32; — hàm trả về float
  • fn init_sensor(config: *const Config) -> i32; — hàm nhận con trỏ

Khi gọi các hàm này, phải dùng unsafe { ... } — Rust không thể kiểm tra an toàn của code C! ⚠️

🤖 bindgen — Máy Dịch Tự Động!

🌟 Ví von: Máy dịch tự động

Thay vì tự tay dịch từng hàm, từng struct từ C sang Rust — hãy để bindgen làm! Nó đọc file header C (.h) và tự động tạo ra code Rust tương ứng! 🤖

bindgen sensor.h -o src/bindings.rs

Xong! File bindings.rs có đủ mọi khai báo cần thiết! 🎉

🔨 build.rs — Chuẩn Bị Trước Khi Build

File build.rs là "script chuẩn bị" — chạy trước khi Cargo build project:

  • 🔨 Biên dịch code C thành object file
  • 🔗 Thêm vào link flags để Rust tìm thấy thư viện C
  • 🤖 Chạy bindgen để tạo bindings tự động

Cargo tự động chạy build.rs mỗi khi bạn build — tiện lắm! ⚡

💡 Công cụ hay: Crate cc giúp biên dịch C từ build.rs rất dễ dàng:
cc::Build::new().file("src/sensor.c").compile("sensor");
Chỉ một dòng là xong! 🎯
⚠️ unsafe là cần thiết: Mọi lần gọi hàm C đều cần unsafe { }. Đây là Rust nói: "Tôi không thể kiểm tra code C — bạn phải tự đảm bảo an toàn!" Hãy kiểm tra kỹ tài liệu thư viện C trước khi dùng! 📖
