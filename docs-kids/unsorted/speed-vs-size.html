Tối ưu hóa robot: chọn tốc độ hay tiết kiệm chỗ?

⚡ Bài Toán Robot Nhỏ

🤖 Robot STM32F303 của bạn chỉ có 256KB Flash! Nếu code quá lớn, nó không vừa — robot không chạy được!

Nhưng đôi khi bạn lại cần robot chạy nhanh nhất có thể — ví dụ như xử lý tín hiệu thời gian thực!

Vậy phải chọn: nhanh hay nhỏ gọn? 🤔

🌟 Ví von: Balo du lịch

Bạn đi du lịch, balo chỉ 15kg:

  • 🏎️ Tối ưu tốc độ: như xe đua — mạnh nhất nhưng tốn nhiên liệu, nặng!
  • 🎒 Tối ưu kích thước: như balo gọn nhẹ — đi xa được hơn, nhẹ hơn!
  • 📝 Không tối ưu: như bản nháp chưa gọn — to nhưng dễ đọc!

🎚️ Các Mức Tối Ưu Hóa

Chế độ Cờ opt-level Tốc độ Kích thước Dùng khi nào?
🔍 Dev (debug) opt-level = 0 🐢 Chậm 📦 To Đang phát triển, debug
⚖️ Balanced 1 opt-level = 1 🚶 Vừa 📫 Vừa Bắt đầu test performance
🏃 Speed opt-level = 2 hoặc 3 🏎️ Rất nhanh 📦 Lớn hơn Cần tốc độ tối đa
🎒 Small opt-level = "s" 🚗 Nhanh 📨 Nhỏ Flash ít, cần code nhỏ
🌱 Tiny opt-level = "z" 🚗 Nhanh 💌 Nhỏ nhất Flash rất ít, tiết kiệm tối đa

⚙️ Cách Cài Đặt Trong Cargo.toml

Thêm vào Cargo.toml:

  • Profile dev (debug):
    [profile.dev]
    opt-level = 0 # Dễ debug
  • Profile release (sản phẩm):
    [profile.release]
    opt-level = "z" # Nhỏ nhất
    lto = true # Link-time optimization
    codegen-units = 1 # Tối ưu toàn bộ

🧙‍♂️ Mẹo Hay Nhất: Tối Ưu Chọn Lọc!

🎯 Mẹo chuyên nghiệp: Tối ưu dependencies (thư viện ngoài) ở opt="z", nhưng giữ nguyên code của bạn ở opt=0 để debug dễ!

Trong Cargo.toml:

  • [profile.dev.package."*"]
  • opt-level = "z" # Tối ưu tất cả dependencies

Kết quả: code nhỏ hơn nhiều, debug vẫn dễ! 🏆

📐 LTO — Tối Ưu Toàn Bộ Cùng Lúc

🌟 Ví von: Chỉnh sửa phim

Khi làm phim, bạn có thể chỉnh sửa từng cảnh riêng lẻ (nhanh hơn), hoặc xem toàn bộ phim và chỉnh sửa cùng lúc (kết quả tốt hơn nhưng lâu hơn)!

LTO (Link-Time Optimization) = nhìn toàn bộ code cùng lúc để tìm cách tối ưu không thể tìm thấy nếu nhìn từng file riêng lẻ! 🎬

💡 Tiết kiệm 6KB Flash chỉ bằng 1 dòng trong Cargo.toml?

Thêm lto = true vào [profile.release] — xong! Compiler sẽ tối ưu xuyên suốt toàn bộ code, loại bỏ tất cả hàm không được dùng! 🗑️✨

Đây là loại "phép thuật" compiler hiếm gặp nhưng rất hiệu quả! 🧙‍♂️

💡 Quy trình khuyến nghị:
1. 🔍 Dev: không tối ưu, debug thoải mái
2. 🧪 Test: opt=1, kiểm tra performance cơ bản
3. 📦 Release: opt="z" + lto=true + codegen-units=1
4. 📏 Đo kích thước: cargo size --release để xem đã vừa Flash chưa!
