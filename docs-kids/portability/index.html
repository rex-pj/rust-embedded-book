🤝 embedded-hal — Ngôn Ngữ Chung Cho Robot

🌐 embedded-hal định nghĩa các "hợp đồng" (traits) mà bất kỳ robot nào cũng phải tuân theo:

  • 📡 I2C: Giao tiếp với cảm biến qua 2 dây
  • ⚡ SPI: Giao tiếp tốc độ cao với màn hình, thẻ nhớ
  • 🔌 GPIO: Bật tắt chân điện
  • 📊 ADC: Đọc giá trị tương tự (analog)
  • ⏱️ Timer: Hẹn giờ, đo thời gian

Miễn là robot implement những hợp đồng này, driver của bạn hoạt động được! 🎉

🎭 Ba Nhân Vật Chính

🤖
HAL Implementation
Driver của chip cụ thể (stm32f3xx-hal, esp-hal...) — implement các traits của embedded-hal
🌡️
Driver
Code cho cảm biến/thiết bị ngoài — chỉ dùng traits chung, không quan tâm chip gì
👨‍💻
Application
Code của bạn — kết nối HAL và Driver lại với nhau cho robot cụ thể

📊 Lợi Ích Toán Học!

Hãy tưởng tượng có M loại robot và N loại driver cảm biến:

Cách làm Số lượng code cần viết
❌ Không dùng embedded-hal M × N phiên bản! (mỗi robot × mỗi cảm biến)
✅ Dùng embedded-hal Chỉ cần M + N phiên bản! 🎉

Ví dụ: 10 loại robot × 20 loại cảm biến = 200 phiên bản → Dùng embedded-hal: chỉ cần 30 phiên bản! 🚀

embedded-hal giống bộ chuyển đổi điện vạn năng — driver của bạn chạy được trên mọi loại robot!
