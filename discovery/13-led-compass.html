LED Compass

Trong phần này, chúng ta sẽ implement một la bàn sử dụng LED trên micro:bit. Giống như la bàn thật, la bàn LED của chúng ta phải chỉ về hướng Bắc. Nó sẽ làm điều đó bằng cách bật một trong các LED ở vòng ngoài cùng — LED được bật nên chỉ về hướng Bắc.

Từ trường có cả cường độ (magnitude), đo bằng Gauss hoặc Tesla, và hướng (direction). Magnetometer trên micro:bit đo cả cường độ lẫn hướng của từ trường bên ngoài, nhưng nó báo cáo sự phân rã (decomposition) của từ trường đó dọc theo các trục của nó.

Magnetometer có ba trục liên kết với nó. Các trục X và Y cơ bản trải trên mặt phẳng sàn nhà. Trục Z chỉ "ra ngoài" khỏi sàn, tức là hướng lên trên.

Bạn đã có thể viết một chương trình liên tục in dữ liệu magnetometer trên console RTT từ chương I2C. Sau khi viết chương trình đó, hãy xác định hướng Bắc ở vị trí hiện tại của bạn. Sau đó canh chỉnh micro:bit theo hướng đó và quan sát xem các số đo của cảm biến trông như thế nào.

Bây giờ hãy xoay board 90 độ trong khi giữ nó song song với mặt đất. Bạn thấy giá trị X, Y và Z gì lần này? Sau đó xoay thêm 90 độ nữa. Bạn thấy giá trị gì?

Magnitude (Cường độ từ trường)

Hàm mag_data() trả về các giá trị x, y, z được đo bằng nanotesla. Để tính cường độ của từ trường, chúng ta tính độ lớn của vector 3D mà các giá trị x, y, z mô tả:

magnitude = √(x² + y² + z²)

Trong Rust, không có hàm sqrtf trong core, nên chúng ta dùng libm::sqrtf. Đây là chương trình hoàn chỉnh tính cường độ từ trường:

#![deny(unsafe_code)]
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use embedded_hal::delay::DelayNs;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

use libm::sqrtf;

use microbit::{
    hal::{twim, Timer},
    pac::twim0::frequency::FREQUENCY_A,
};

use lsm303agr::{Lsm303agr, MagMode, MagOutputDataRate};

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let i2c = { twim::Twim::new(board.TWIM0, board.i2c_internal.into(), FREQUENCY_A::K100) };
    let mut timer0 = Timer::new(board.TIMER0);

    let mut sensor = Lsm303agr::new_with_i2c(i2c);
    sensor.init().unwrap();
    sensor
        .set_mag_mode_and_odr(&mut timer0, MagMode::HighResolution, MagOutputDataRate::Hz10)
        .unwrap();
    let mut sensor = sensor.into_mag_continuous().ok().unwrap();

    loop {
        while !sensor.mag_status().unwrap().xyz_new_data() {
            timer0.delay_ms(1u32);
        }
        let (x, y, z) = sensor.magnetic_field().unwrap().xyz_nt();
        let (x, y, z) = (x as f32, y as f32, z as f32);
        let magnitude = sqrtf(x * x + y * y + z * z);
        rprintln!("{} mG", magnitude / 100.0);
    }
}

Cường độ từ trường của Trái Đất nằm trong khoảng từ 250 mG đến 650 mG tùy thuộc vào vị trí địa lý. Ví dụ, tác giả quan sát khoảng 340 mG.

Câu hỏi thảo luận:

  • Khi board đứng yên, giá trị cường độ có giữ nguyên không?
  • Khi bạn xoay board, cường độ có thay đổi không? Lý thuyết nó có nên thay đổi không?
  • Lý thuyết thì cường độ nên giữ nguyên bất kể hướng xoay vì ta chỉ xoay vector, không thay đổi độ dài của nó.

Hiệu chỉnh (Calibration)

Một điều rất quan trọng cần làm trước khi sử dụng cảm biến và cố gắng phát triển ứng dụng với nó là xác minh rằng đầu ra của nó thực sự chính xác. Nếu không phải như vậy, chúng ta cần hiệu chỉnh (calibrate) cảm biến (hoặc nó có thể bị hỏng, nhưng điều đó khá khó xảy ra trong trường hợp này).

Trong thực tế, trên hai chiếc micro:bit khác nhau, magnetometer mà không có hiệu chỉnh đã cho kết quả sai khá nhiều so với những gì nó đáng ra phải đo. Do đó, trong chương này chúng ta sẽ giả định rằng cảm biến cần được hiệu chỉnh.

Việc hiệu chỉnh liên quan đến khá nhiều toán học (ma trận) nên chúng ta sẽ không đề cập ở đây, nhưng Design Note của ST Microelectronics mô tả quy trình nếu bạn quan tâm.

May mắn thay, nhóm đã xây dựng phần mềm gốc cho micro:bit đã implement một cơ chế hiệu chỉnh bằng C++ ở đây.

Bạn có thể tìm thấy bản dịch sang Rust trong src/calibration.rs. Cách sử dụng được minh họa trong file src/main.rs mặc định. Cách hiệu chỉnh hoạt động như sau:

Bạn phải nghiêng micro:bit cho đến khi tất cả các LED trên ma trận LED sáng lên.

Nếu bạn không muốn chơi "trò chơi" đó mỗi lần khởi động lại ứng dụng trong quá trình phát triển, bạn có thể tự do chỉnh sửa template src/main.rs để chỉ sử dụng cùng một giá trị hiệu chỉnh tĩnh một khi bạn đã có lần đầu tiên.

Bây giờ sau khi đã xong phần hiệu chỉnh cảm biến, hãy cùng tìm hiểu cách thực sự xây dựng ứng dụng này!

Take 1: Phương pháp quadrant (góc phần tư)

Vì la bàn thường được cầm theo chiều ngang, chúng ta chủ yếu quan tâm đến các thành phần X và Y của từ trường. Nếu chỉ nhìn vào dấu của các thành phần X và Y, ta có thể xác định từ trường thuộc góc phần tư nào. Qua quan sát khi xoay micro:bit, ta có thể ánh xạ các góc phần tư đó thành hướng la bàn.

Ví dụ, khi micro:bit chỉ về hướng Đông Bắc (NorthEast), cả X lẫn Y đều dương.

Bảng ánh xạ quadrant sang hướng:

Dấu X Dấu Y Quadrant Hướng
dương (+)dương (+)INorthEast (Đông Bắc)
âm (−)dương (+)IINorthWest (Tây Bắc)
âm (−)âm (−)IIISouthWest (Tây Nam)
dương (+)âm (−)IVSouthEast (Đông Nam)

Màn hình LED hiển thị mũi tên theo hướng tương ứng. Ví dụ:

Hướng LED sáng (hàng, cột) — 0-indexed
North (Bắc)góc trên giữa
NorthEastgóc trên phải
East (Đông)giữa phải
SouthEastgóc dưới phải
South (Nam)góc dưới giữa
SouthWestgóc dưới trái
West (Tây)giữa trái
NorthWestgóc trên trái

Dưới đây là template code để bạn hoàn thiện:

#![deny(unsafe_code)]
#![no_main]
#![no_std]

use crate::calibration::calc_calibration;

#[cfg(feature = "v1")]

#[cfg(feature = "v2")]
use microbit::{hal::twim, pac::twim0::frequency::FREQUENCY_A};

use lsm303agr::{AccelOutputDataRate, Lsm303agr, MagOutputDataRate};

#[entry]

    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let i2c = { twi::Twi::new(board.TWI0, board.i2c.into(), FREQUENCY_A::K100) };

    #[cfg(feature = "v2")]
    let i2c = { twim::Twim::new(board.TWIM0, board.i2c_internal.into(), FREQUENCY_A::K100) };

    let mut display = Display::new(board.display_pins);

    sensor.set_mag_odr(MagOutputDataRate::Hz10).unwrap();

Take 2: Phương pháp atan2f (góc chính xác)

Cách tiếp cận theo quadrant chỉ cho 4 hướng. Để có độ chính xác hơn (8 hướng), ta có thể tính góc chính xác mà từ trường tạo với các trục X và Y của magnetometer bằng hàm atan2. Hàm này trả về một góc trong khoảng -PI đến PI.

Một vòng tròn = 360 độ = 2·PI radian. PI radian = 180 độ. Khi theta bằng 0, micro:bit đang chỉ về hướng Đông. Khi theta tăng dần (ngược chiều kim đồng hồ), ta đến các hướng khác.

Dưới đây là template code để bạn hoàn thiện phần xác định hướng từ theta:

#![deny(unsafe_code)]
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

mod calibration;
use crate::calibration::calc_calibration;
use crate::calibration::calibrated_measurement;

mod led;
use crate::led::Direction;
use crate::led::direction_to_led;

// You'll find this useful ;-)
use core::f32::consts::PI;
use libm::atan2f;

use microbit::{display::blocking::Display, hal::Timer};

    let mut timer = Timer::new(board.TIMER0);
    let mut display = Display::new(board.display_pins);

    let mut sensor = Lsm303agr::new_with_i2c(i2c);
    sensor.init().unwrap();
    sensor.set_mag_odr(MagOutputDataRate::Hz10).unwrap();
    sensor.set_accel_odr(AccelOutputDataRate::Hz10).unwrap();
    let mut sensor = sensor.into_mag_continuous().ok().unwrap();

Solution 1: Lời giải quadrant

Đây là lời giải hoàn chỉnh cho cách tiếp cận theo quadrant, xác định 4 hướng chéo:

#![deny(unsafe_code)]
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

mod calibration;
use crate::calibration::calc_calibration;
use crate::calibration::calibrated_measurement;

mod led;
use crate::led::Direction;

    #[cfg(feature = "v1")]
    let i2c = { twi::Twi::new(board.TWI0, board.i2c.into(), FREQUENCY_A::K100) };

    #[cfg(feature = "v2")]
    let i2c = { twim::Twim::new(board.TWIM0, board.i2c_internal.into(), FREQUENCY_A::K100) };

    let mut timer = Timer::new(board.TIMER0);
    let mut display = Display::new(board.display_pins);

    let mut sensor = Lsm303agr::new_with_i2c(i2c);
    sensor.init().unwrap();
    sensor.set_mag_odr(MagOutputDataRate::Hz10).unwrap();
    sensor.set_accel_odr(AccelOutputDataRate::Hz10).unwrap();
    let mut sensor = sensor.into_mag_continuous().ok().unwrap();

        while !sensor.mag_status().unwrap().xyz_new_data {}
        let mut data = sensor.mag_data().unwrap();
        data = calibrated_measurement(data, &calibration);

        let dir = match (data.x > 0, data.y > 0) {
            // Quadrant I
            (true, true) => Direction::NorthEast,
            // Quadrant II
            (false, true) => Direction::NorthWest,
            // Quadrant III
            (false, false) => Direction::SouthWest,
            // Quadrant IV
            (true, false) => Direction::SouthEast,
        };

        // dùng module led để chuyển hướng thành mũi tên LED
        // và dùng các hàm hiển thị LED từ chương 5 để hiển thị mũi tên
        display.show(&mut timer, direction_to_led(dir), 100);
    }
}

Solution 2: Lời giải atan2f (16 hướng)

Đây là lời giải hoàn chỉnh từ bản gốc — dùng atan2f và floorf để xác định 16 hướng la bàn, mỗi hướng tương ứng một LED trên vòng ngoài ma trận 5×5:

#![deny(unsafe_code)]
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use embedded_hal::delay::DelayNs;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

use core::f32::consts::PI;
use libm::{atan2f, floorf};

use microbit::{
    display::blocking::Display,
    hal::{twim, Timer},
    pac::twim0::frequency::FREQUENCY_A,
};

use lsm303agr::{Lsm303agr, MagMode, MagOutputDataRate};

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let i2c = { twim::Twim::new(board.TWIM0, board.i2c_internal.into(), FREQUENCY_A::K100) };
    let mut timer0 = Timer::new(board.TIMER0);
    let mut display = Display::new(board.display_pins);

    let mut sensor = Lsm303agr::new_with_i2c(i2c);
    sensor.init().unwrap();
    sensor
        .set_mag_mode_and_odr(&mut timer0, MagMode::HighResolution, MagOutputDataRate::Hz10)
        .unwrap();
    let mut sensor = sensor.into_mag_continuous().ok().unwrap();

    let mut leds = [[0u8; 5]; 5];

    // 16 vị trí LED trên vòng ngoài, bắt đầu từ hướng Tây, đi theo chiều kim đồng hồ
    let indices = [
        (2, 0), /* W  */
        (3, 0), /* W-SW */
        (3, 1), /* SW */
        (4, 1), /* S-SW */
        (4, 2), /* S  */
        (4, 3), /* S-SE */
        (3, 3), /* SE */
        (3, 4), /* E-SE */
        (2, 4), /* E  */
        (1, 4), /* E-NE */
        (1, 3), /* NE */
        (0, 3), /* N-NE */
        (0, 2), /* N  */
        (0, 1), /* N-NW */
        (1, 1), /* NW */
        (1, 0), /* W-NW */
    ];

    loop {
        while !sensor.mag_status().unwrap().xyz_new_data() {
            timer0.delay_ms(1u32);
        }
        let (x, y, _) = sensor.magnetic_field().unwrap().xyz_nt();

        // theta: góc từ trục X, khoảng -PI đến PI
        let theta = atan2f(y as f32, x as f32);
        // Chia vòng tròn thành 32 phần (mỗi phần PI/16 radian), lấy nửa
        let seg = floorf(16.0 * theta / PI) as i8;

        let index = if seg >= 15 || seg <= -15 {
            8 // East
        } else if seg >= 0 {
            (seg / 2) as usize
        } else {
            ((31 + seg) / 2) as usize
        };

        let (r, c) = indices[index];
        leds[r][c] = 255u8;
        display.show(&mut timer0, leds, 50);
        leds[r][c] = 0u8;
        display.show(&mut timer0, leds, 50);
    }
}
