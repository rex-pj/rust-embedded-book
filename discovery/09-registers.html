Registers 📟

Chương này là phần khám phá kỹ thuật về cách tương tác phần cứng ở mức thấp. Bạn có thể bỏ qua và quay lại sau nếu muốn.

Câu hỏi: điều gì thực sự xảy ra khi gọi display_pins.row1.set_high() hay button_a_pin.is_high()? Câu trả lời: chúng ta đang ghi vào các vùng bộ nhớ đặc biệt gọi là registers.

Ví dụ trực tiếp với register

Thư mục 09-registers chứa code demo thao tác register trực tiếp:

rust
#![no_main]
#![no_std]

#[allow(unused_imports)]
use registers::entry;

#[entry]
fn main() -> ! {
    registers::init();

    unsafe {
        // Địa chỉ "magic" của GPIO Port 0 Output register
        const PORT_P0_OUT: u32 = 0x50000504;

        // Bật hàng trên cùng (bit 21)
        *(PORT_P0_OUT as *mut u32) |= 1 << 21;

        // Bật hàng dưới (bit 19)
        *(PORT_P0_OUT as *mut u32) |= 1 << 19;

        // Tắt hàng trên
        *(PORT_P0_OUT as *mut u32) &= !(1 << 21);

        // Tắt hàng dưới
        *(PORT_P0_OUT as *mut u32) &= !(1 << 19);
    }

    loop {
        core::hint::spin_loop();
    }
}

Register là gì?

Địa chỉ 0x50000504 tham chiếu đến một register — vùng bộ nhớ đặc biệt kiểm soát một peripheral. Peripheral cung cấp tính năng mở rộng ngoài khả năng tính toán/logic gốc của processor.

🗺️ Memory Map của nRF52833 (đơn giản hóa)
Địa chỉVùngMô tả
0x00000000FlashCode của chương trình (512KB)
0x20000000RAMBiến, stack, heap (128KB)
0x40000000APB PeripheralsUART, SPI, I2C, Timer...
0x50000000AHB PeripheralsGPIO (P0 tại 0x50000504)
0xE0000000Core PeripheralsNVIC, SysTick, Debug...

Register tại 0x50000504 cụ thể quản lý các GPIO pin trong nhóm P0. nRF52833 có hơn 32 GPIO pin tổ chức thành hai nhóm ("P0" và "P1"), mỗi nhóm có bộ register riêng.

LED, Digital Outputs và Mức điện áp

💡 Cách LED hoạt động
HIGH (3.3V) = 1

Pin đang cấp điện áp 3.3V. LED kết nối đúng chiều sẽ sáng.

LOW (0V) = 0

Pin ở mức 0V (GND). LED tắt.

LED là diode — chỉ dẫn điện một chiều. Đấu ngược chiều → không sáng (không hỏng).

RTRM — Reading the Reference Manual

Khi cần tra cứu địa chỉ register và bit fields chính xác, hãy xem nRF52833 Product Specification (Reference Manual). Đây là tài liệu dứt khoát cho mọi thứ liên quan đến peripheral registers.

📖
Đọc Reference Manual hiệu quả

Tìm chapter "GPIO" (General Purpose Input/Output) → tìm mục "Registers" → tìm đến register "OUT" hay "IN". Reference manual sẽ cho biết từng bit có nghĩa gì.

(Mis)Optimization — Khi compiler "tối ưu" nhầm

Khi ghi trực tiếp vào hardware registers, compiler có thể tối ưu bỏ đi những lần ghi mà nó nghĩ là "thừa". Trong embedded, những lần ghi đó có tác dụng thực trên phần cứng — không được bỏ.

rust
// KHÔNG tốt: compiler có thể tối ưu bỏ một số lần ghi
*(PORT_P0_OUT as *mut u32) = 1 << 21;
*(PORT_P0_OUT as *mut u32) = 0;  // có thể bị bỏ!

// TỐT hơn: dùng volatile write để đảm bảo mọi lần ghi đều thực thi
use core::ptr;
ptr::write_volatile(PORT_P0_OUT as *mut u32, 1 << 21);
ptr::write_volatile(PORT_P0_OUT as *mut u32, 0);

Đây là một trong những lý do tại sao HAL và PAC tồn tại — chúng xử lý volatile access tự động cho bạn.

Địa chỉ 0xBAAAAAAD — Lỗi truy cập bộ nhớ

Truy cập vào địa chỉ không hợp lệ gây ra hardware fault. Processor phát hiện truy cập bộ nhớ không hợp lệ và kích hoạt fault handler — thường dẫn đến reset hoặc dừng hệ thống.

💥
HardFault — Không như Exception trong .NET

Trong .NET, access violation ném ra exception và bạn có thể catch. Trong embedded bare-metal, không có OS, không có exception handler mặc định. HardFault thường có nghĩa là board reset hoặc treo máy. Đây là lý do tại sao Rust's type system giúp prevent rất nhiều lỗi loại này.

Spooky action at a distance — Tác dụng phụ không ngờ

Ghi vào một peripheral register có thể ảnh hưởng đến các peripheral khác do kết nối phần cứng. "Tác dụng từ xa" này khiến lập trình embedded phức tạp và là một trong những lý do tại sao dùng HAL abstractions an toàn hơn thao tác register trực tiếp.

Type-safe Manipulation với PAC

Peripheral Access Crate (PAC) cung cấp wrappers type-safe quanh registers, ngăn chặn nhiều loại lỗi. Thay vì magic numbers như 0x50000504, bạn dùng named types và methods:

rust
// Cách thủ công với raw pointer — dễ sai
unsafe {
    *(0x50000504 as *mut u32) |= 1 << 21;
}

// Cách dùng PAC — type-safe, tự document
let peripherals = pac::Peripherals::take().unwrap();
peripherals.P0.out.modify(|_, w| w.pin21().high());
