Interrupts ⚡

Polling lãng phí điện năng và giới hạn chức năng — CPU phải liên tục kiểm tra xem peripheral đã sẵn sàng chưa. Interrupt cho phép peripheral "ngắt" luồng chương trình bất cứ lúc nào khi có sự kiện xảy ra.

Interrupt hoạt động như thế nào

⚡ Luồng xử lý Interrupt

NVIC và Interrupt Priority

nRF52833 dùng NVIC (Nested Vectored Interrupt Controller) để quản lý interrupt priority. Interrupt có priority cao hơn có thể preempt interrupt có priority thấp hơn. Điều này đảm bảo các sự kiện time-critical được xử lý trước.

Chia sẻ dữ liệu với ISR

Thách thức lớn nhất: chia sẻ dữ liệu giữa chương trình chính và ISR. Vì ISR có thể chạy bất cứ lúc nào, dữ liệu chia sẻ phải được xử lý cẩn thận để tránh race conditions.

Trong Rust, thường dùng:

rust
use core::sync::atomic::{AtomicBool, Ordering};
use cortex_m::interrupt::Mutex;
use core::cell::Cell;

// Cách 1: Atomic types (đơn giản, thread-safe)
static BUTTON_PRESSED: AtomicBool = AtomicBool::new(false);

// ISR có thể set:
BUTTON_PRESSED.store(true, Ordering::SeqCst);

// Main loop kiểm tra:
if BUTTON_PRESSED.load(Ordering::SeqCst) {
    BUTTON_PRESSED.store(false, Ordering::SeqCst);
    // Xử lý sự kiện
}

// Cách 2: Mutex + Cell/RefCell cho dữ liệu phức tạp hơn
static COUNTER: Mutex<Cell<u32>> = Mutex::new(Cell::new(0));

// Trong ISR (đã ở trong critical section):
cortex_m::interrupt::free(|cs| {
    let val = COUNTER.borrow(cs).get();
    COUNTER.borrow(cs).set(val + 1);
});

Cấu hình GPIOTE Interrupt cho Button A

rust
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use core::sync::atomic::{AtomicBool, Ordering};
use microbit::Board;
use nrf52833_hal::gpiote::Gpiote;
use nrf52833_pac::interrupt;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

static BUTTON_A_PRESSED: AtomicBool = AtomicBool::new(false);

#[interrupt]
fn GPIOTE() {
    // QUAN TRỌNG: xóa event để tránh trigger lại
    let gpiote = unsafe { &*nrf52833_pac::GPIOTE::ptr() };
    gpiote.events_in[0].write(|w| w);

    BUTTON_A_PRESSED.store(true, Ordering::SeqCst);
}

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = Board::take().unwrap();

    // Cấu hình GPIOTE channel 0 cho Button A (P0.14)
    let gpiote = Gpiote::new(board.GPIOTE);
    gpiote.channel0()
        .input_pin(&board.buttons.button_a.degrade())
        .hi_to_lo()  // trigger khi nhấn (HIGH → LOW)
        .enable_interrupt();

    // Enable GPIOTE interrupt trong NVIC
    unsafe {
        cortex_m::peripheral::NVIC::unmask(nrf52833_pac::Interrupt::GPIOTE);
    }

    rprintln!("Đang chờ Button A...");
    loop {
        if BUTTON_A_PRESSED.load(Ordering::SeqCst) {
            BUTTON_A_PRESSED.store(false, Ordering::SeqCst);
            rprintln!("Button A được nhấn! (qua interrupt)");
        }

        // Ngủ cho đến interrupt tiếp theo — tiết kiệm điện
        cortex_m::asm::wfi();
    }
}
⚡
Macro #[interrupt]

Macro này đánh dấu hàm là ISR với các yêu cầu compiler cụ thể: hàm không nhận argument, không trả về value, và tên phải khớp với tên interrupt trong PAC. NVIC còn phải được configure riêng để enable interrupt đó.

Debouncing — Xử lý bounce của nút bấm

Nút bấm vật lý có hiện tượng "bounce" — tiếp xúc điện tử nảy lên/xuống nhiều lần khi nhấn, tạo ra hàng chục interrupt events từ một lần nhấn duy nhất.

🔘 Hiện tượng Bounce

Giải pháp: bỏ qua các sự kiện xảy ra trong vòng ~50ms sau sự kiện đầu tiên (software debounce)

WFI — Chờ interrupt để tiết kiệm điện

Thay vì spin trong busy loop, processor có thể vào trạng thái sleep và chờ interrupt:

rust
loop {
    // Xử lý công việc đang pending...

    // Ngủ đến khi có interrupt tiếp theo
    // CPU clock gần như dừng → tiết kiệm điện đáng kể
    cortex_m::asm::wfi();
}

Addendum: PWM

PWM (Pulse Width Modulation) là kỹ thuật kiểm soát công suất hiệu dụng bằng cách bật/tắt nhanh. Tỉ lệ thời gian bật trên tổng chu kỳ gọi là duty cycle:

⚙️ PWM Duty Cycle
50% duty cycle
LED mờ / âm chuẩn
80% duty cycle
LED sáng hơn
20% duty cycle
LED mờ nhất

nRF52833 có dedicated PWM peripheral tạo PWM signal mà không cần CPU involvement — phù hợp cho motor control, LED dimming và âm thanh.
