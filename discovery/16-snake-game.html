Snake Game

Bây giờ chúng ta sẽ implement một trò chơi snake cơ bản mà bạn có thể chơi trên MB2 sử dụng ma trận LED 5×5 làm màn hình và hai nút bấm làm điều khiển. Trong quá trình đó, chúng ta sẽ xây dựng dựa trên một số khái niệm đã được đề cập trong các chương trước của cuốn sách, và cũng học về một số peripheral và khái niệm mới.

Tính module hóa

Code ở đây được module hóa chi tiết hơn mức có lẽ cần thiết. Cách module hóa chi tiết này cho phép chúng ta xem code từng chút một. Chúng ta sẽ xây dựng code từ dưới lên: chúng ta đầu tiên sẽ xây dựng ba module — game, controls và display, rồi kết hợp chúng để tạo ra chương trình cuối cùng. Mỗi module sẽ có một file code top-level và một hoặc nhiều file code được include: ví dụ, module game sẽ bao gồm src/game.rs, src/game/coords.rs, src/game/movement.rs, v.v. Câu lệnh Rust mod được dùng để kết hợp các thành phần khác nhau của module. The Rust Programming Language có một mô tả tốt về hệ thống module của Rust.

Game logic

Module đầu tiên chúng ta sẽ xây dựng là game logic. Bạn có lẽ đã quen với trò chơi snake, nhưng nếu chưa, ý tưởng cơ bản là người chơi dẫn dắt một con rắn trên một lưới 2D. Tại bất kỳ thời điểm nào, có một "thức ăn" tại một vị trí ngẫu nhiên trên lưới và mục tiêu của trò chơi là đưa con rắn đến "ăn" càng nhiều thức ăn càng tốt. Mỗi lần snake ăn thức ăn nó sẽ dài thêm. Người chơi thua nếu snake va vào đuôi của chính nó.

Trong một số biến thể của trò chơi, người chơi cũng thua nếu snake chạm vào rìa lưới, nhưng vì kích thước lưới của chúng ta nhỏ, chúng ta sẽ implement quy tắc "wraparound": nếu snake đi ra ngoài một cạnh của lưới, nó sẽ tiếp tục từ cạnh đối diện.

Module game

Chúng ta sẽ xây dựng cơ chế game trong module game.

Tọa độ

Chúng ta bắt đầu bằng cách định nghĩa một hệ tọa độ cho game (src/game/coords.rs).

use super::Prng;

use heapless::index_set::FnvIndexSet;

/// A single point on the grid.
#[derive(Debug, Copy, Clone, Eq, PartialEq, Hash)]
pub struct Coords {
    // Signed ints to allow negative values (handy when checking if we have gone
    // off the top or left of the grid)
    pub row: i8,
    pub col: i8,
}

impl Coords {
    /// Get random coordinates within a grid. `exclude` is an optional set of
    /// coordinates which should be excluded from the output.
    pub fn random(rng: &mut Prng, exclude: Option<&FnvIndexSet<Coords, 32>>) -> Self {
        let mut coords = Coords {
            row: ((rng.random_u32() as usize) % 5) as i8,
            col: ((rng.random_u32() as usize) % 5) as i8,
        };
        while exclude.is_some_and(|exc| exc.contains(&coords)) {
            coords = Coords {
                row: ((rng.random_u32() as usize) % 5) as i8,
                col: ((rng.random_u32() as usize) % 5) as i8,
            }
        }
        coords
    }

    /// Whether the point is outside the bounds of the grid.
    pub fn is_out_of_bounds(&self) -> bool {
        self.row < 0 || self.row >= 5 || self.col < 0 || self.col >= 5
    }
}

Chúng ta dùng struct Coords để tham chiếu đến một vị trí trên lưới. Vì Coords chỉ chứa hai số nguyên, chúng ta yêu cầu compiler derive implementation của trait Copy cho nó, để chúng ta có thể truyền các struct Coords mà không cần lo về ownership.

Sinh số ngẫu nhiên

Chúng ta định nghĩa một associated function Coords::random, cho chúng ta một vị trí ngẫu nhiên trên lưới. Chúng ta sẽ dùng điều này sau để xác định nơi đặt thức ăn cho snake.

Để sinh tọa độ ngẫu nhiên, chúng ta cần một nguồn số ngẫu nhiên. nRF52833 có một peripheral hardware random number generator (HWRNG), được ghi ở mục 6.19 của spec nRF52833. HAL cho chúng ta một interface đơn giản đến HWRNG qua struct microbit::hal::rng::Rng. HWRNG có thể không đủ nhanh cho game; cũng tiện để kiểm tra khi có thể reproduce chuỗi số ngẫu nhiên giữa các lần chạy, điều không thể thực hiện với HWRNG theo thiết kế. Do đó chúng ta cũng định nghĩa một pseudo-random number generator (PRNG). PRNG dùng thuật toán xorshift để sinh các giá trị u32 giả ngẫu nhiên. Thuật toán này cơ bản và không an toàn về mật mã, nhưng nó hiệu quả, dễ implement và đủ tốt cho trò chơi snake khiêm tốn của chúng ta. Struct Prng của chúng ta cần một giá trị seed ban đầu, mà chúng ta lấy từ RNG peripheral.

Tất cả những điều này tạo nên src/game/rng.rs.

use crate::Rng;

/// A basic pseudo-random number generator.
pub struct Prng {
    value: u32,
}

impl Prng {
    pub fn seeded(rng: &mut Rng) -> Self {
        Self::new(rng.random_u32())
    }

    pub fn new(seed: u32) -> Self {
        Self { value: seed }
    }

    /// Basic xorshift PRNG function: see <https://en.wikipedia.org/wiki/Xorshift>
    fn xorshift32(mut input: u32) -> u32 {
        input ^= input << 13;
        input ^= input >> 17;
        input ^= input << 5;
        input
    }

    /// Return a pseudo-random u32.
    pub fn random_u32(&mut self) -> u32 {
        self.value = Self::xorshift32(self.value);
        self.value
    }
}

Di chuyển

Chúng ta cũng cần định nghĩa một vài enum giúp chúng ta quản lý trạng thái của game: hướng di chuyển, hướng rẽ, trạng thái game hiện tại và kết quả của một "bước" cụ thể trong game (tức là một lần di chuyển đơn của snake). src/game/movement.rs chứa những thứ này.

use super::Coords;

/// Define the directions the snake can move.
pub enum Direction {
    Up,
    Down,
    Left,
    Right,
}

/// What direction the snake should turn.
#[derive(Debug, Copy, Clone)]
pub enum Turn {
    Left,
    Right,
    None,
}

/// The current status of the game.
pub enum GameStatus {
    Won,
    Lost,
    Ongoing,
}

/// The outcome of a single move/step.
pub enum StepOutcome {
    /// Grid full (player wins)
    Full,
    /// Snake has collided with itself (player loses)
    Collision,
    /// Snake has eaten some food
    Eat(Coords),
    /// Snake has moved (and nothing else has happened)
    Move(Coords),
}

Một con rắn (Một Rắn!)

Tiếp theo chúng ta định nghĩa struct Snake, theo dõi các tọa độ bị snake chiếm và hướng di chuyển của nó. Chúng ta dùng một queue (heapless::spsc::Queue) để theo dõi thứ tự tọa độ và một hash set (heapless::FnvIndexSet) để phát hiện va chạm nhanh chóng. Snake có các method để di chuyển. src/game/snake.rs chứa phần này.

use super::{Coords, Direction, FnvIndexSet, Turn};

use heapless::spsc::Queue;

pub struct Snake {
    /// Coordinates of the snake's head.
    pub head: Coords,
    /// Queue of coordinates of the rest of the snake's body. The end of the tail is
    /// at the front.
    pub tail: Queue<Coords, 32>,
    /// A set containing all coordinates currently occupied by the snake (for fast
    /// collision checking).
    pub coord_set: FnvIndexSet<Coords, 32>,
    /// The direction the snake is currently moving in.
    pub direction: Direction,
}

impl Snake {
    pub fn make_snake() -> Self {
        let head = Coords { row: 2, col: 2 };
        let initial_tail = Coords { row: 2, col: 1 };
        let mut tail = Queue::new();
        tail.enqueue(initial_tail).unwrap();
        let mut coord_set: FnvIndexSet<Coords, 32> = FnvIndexSet::new();
        coord_set.insert(head).unwrap();
        coord_set.insert(initial_tail).unwrap();
        Self {
            head,
            tail,
            coord_set,
            direction: Direction::Right,
        }
    }

    /// Move the snake onto the tile at the given coordinates. If `extend` is false,
    /// the snake's tail vacates the rearmost tile.
    pub fn move_snake(&mut self, coords: Coords, extend: bool) {
        // Location of head becomes front of tail
        self.tail.enqueue(self.head).unwrap();
        // Head moves to new coords
        self.head = coords;
        self.coord_set.insert(coords).unwrap();
        if !extend {
            let back = self.tail.dequeue().unwrap();
            self.coord_set.remove(&back);
        }
    }

    fn turn_right(&mut self) {
        self.direction = match self.direction {
            Direction::Up => Direction::Right,
            Direction::Down => Direction::Left,
            Direction::Left => Direction::Up,
            Direction::Right => Direction::Down,
        }
    }

    fn turn_left(&mut self) {
        self.direction = match self.direction {
            Direction::Up => Direction::Left,
            Direction::Down => Direction::Right,
            Direction::Left => Direction::Down,
            Direction::Right => Direction::Up,
        }
    }

    pub fn turn(&mut self, direction: Turn) {
        match direction {
            Turn::Left => self.turn_left(),
            Turn::Right => self.turn_right(),
            Turn::None => (),
        }
    }
}

Game Module Top-Level

Struct Game theo dõi trạng thái game. Nó chứa một đối tượng Snake, tọa độ hiện tại của thức ăn, tốc độ của game (được dùng để xác định thời gian trôi qua giữa mỗi lần di chuyển của snake), trạng thái của game (game đang diễn ra hay người chơi đã thắng hoặc thua) và điểm số của người chơi.

Struct này chứa các method xử lý từng bước của game, xác định bước đi tiếp theo của snake và cập nhật trạng thái game cho phù hợp. Nó cũng chứa hai method — game_matrix và score_matrix — xuất ra các mảng 2D các giá trị có thể được dùng để hiển thị trạng thái game hoặc điểm số của người chơi trên ma trận LED (như chúng ta sẽ thấy sau).

Chúng ta đặt struct Game ở top của module game, trong src/game.rs.

mod coords;
mod movement;
mod rng;
mod snake;

use crate::Rng;

pub use coords::Coords;
pub use movement::{Direction, GameStatus, StepOutcome, Turn};
pub use rng::Prng;
pub use snake::Snake;

use heapless::index_set::FnvIndexSet;

/// Struct to hold game state and associated behaviour
pub struct Game {
    pub status: GameStatus,
    rng: Prng,
    snake: Snake,
    food_coords: Coords,
    speed: u8,
    score: u8,
}

impl Game {
    pub fn new(rng: &mut Rng) -> Self {
        let mut rng = Prng::seeded(rng);
        let snake = Snake::make_snake();
        let food_coords = Coords::random(&mut rng, Some(&snake.coord_set));
        Self {
            rng,
            snake,
            food_coords,
            speed: 1,
            status: GameStatus::Ongoing,
            score: 0,
        }
    }

    /// Reset the game state to start a new game.
    pub fn reset(&mut self) {
        self.snake = Snake::make_snake();
        self.place_food();
        self.speed = 1;
        self.status = GameStatus::Ongoing;
        self.score = 0;
    }

    /// Randomly place food on the grid.
    fn place_food(&mut self) -> Coords {
        let coords = Coords::random(&mut self.rng, Some(&self.snake.coord_set));
        self.food_coords = coords;
        coords
    }

    /// "Wrap around" out of bounds coordinates (eg, coordinates that are off to the
    /// left of the grid will appear in the rightmost column). Assumes that
    /// coordinates are out of bounds in one dimension only.
    fn wraparound(&self, coords: Coords) -> Coords {
        if coords.row < 0 {
            Coords { row: 4, ..coords }
        } else if coords.row >= 5 {
            Coords { row: 0, ..coords }
        } else if coords.col < 0 {
            Coords { col: 4, ..coords }
        } else {
            Coords { col: 0, ..coords }
        }
    }

    /// Determine the next tile that the snake will move on to (without actually
    /// moving the snake).
    fn get_next_move(&self) -> Coords {
        let head = &self.snake.head;
        let next_move = match self.snake.direction {
            Direction::Up => Coords {
                row: head.row - 1,
                col: head.col,
            },
            Direction::Down => Coords {
                row: head.row + 1,
                col: head.col,
            },
            Direction::Left => Coords {
                row: head.row,
                col: head.col - 1,
            },
            Direction::Right => Coords {
                row: head.row,
                col: head.col + 1,
            },
        };
        if next_move.is_out_of_bounds() {
            self.wraparound(next_move)
        } else {
            next_move
        }
    }

    /// Assess the snake's next move and return the outcome. Doesn't actually update
    /// the game state.
    fn get_step_outcome(&self) -> StepOutcome {
        let next_move = self.get_next_move();
        if self.snake.coord_set.contains(&next_move) {
            // We haven't moved the snake yet, so if the next move is at the end of
            // the tail, there won't actually be any collision (as the tail will have
            // moved by the time the head moves onto the tile)
            if next_move != *self.snake.tail.peek().unwrap() {
                StepOutcome::Collision
            } else {
                StepOutcome::Move(next_move)
            }
        } else if next_move == self.food_coords {
            if self.snake.tail.len() == 23 {
                StepOutcome::Full
            } else {
                StepOutcome::Eat(next_move)
            }
        } else {
            StepOutcome::Move(next_move)
        }
    }

    /// Handle the outcome of a step, updating the game's internal state.
    fn handle_step_outcome(&mut self, outcome: StepOutcome) {
        self.status = match outcome {
            StepOutcome::Collision => GameStatus::Lost,
            StepOutcome::Full => GameStatus::Won,
            StepOutcome::Eat(c) => {
                self.snake.move_snake(c, true);
                self.place_food();
                self.score += 1;
                if self.score.is_multiple_of(5) {
                    self.speed += 1
                }
                GameStatus::Ongoing
            }
            StepOutcome::Move(c) => {
                self.snake.move_snake(c, false);
                GameStatus::Ongoing
            }
        }
    }

    pub fn step(&mut self, turn: Turn) {
        self.snake.turn(turn);
        let outcome = self.get_step_outcome();
        self.handle_step_outcome(outcome);
    }

    /// Calculate the length of time to wait between game steps, in milliseconds.
    /// Generally this will get lower as the player's score increases, but need to
    /// be careful it cannot result in a value below zero.
    pub fn step_len_ms(&self) -> u32 {
        let result = 1000 - (200 * ((self.speed as i32) - 1));
        if result < 200 {
            200u32
        } else {
            result as u32
        }
    }

    /// Return an array representing the game state, which can be used to display the
    /// state on the microbit's LED matrix. Each `_brightness` parameter should be a
    /// value between 0 and 9.
    pub fn game_matrix(
        &self,
        head_brightness: u8,
        tail_brightness: u8,
        food_brightness: u8,
    ) -> [[u8; 5]; 5] {
        let mut values = [[0u8; 5]; 5];
        values[self.snake.head.row as usize][self.snake.head.col as usize] = head_brightness;
        for t in &self.snake.tail {
            values[t.row as usize][t.col as usize] = tail_brightness
        }
        values[self.food_coords.row as usize][self.food_coords.col as usize] = food_brightness;
        values
    }

    /// Return an array representing the game score, which can be used to display the
    /// score on the microbit's LED matrix (by illuminating the equivalent number of
    /// LEDs, going left->right and top->bottom).
    pub fn score_matrix(&self) -> [[u8; 5]; 5] {
        let mut values = [[0u8; 5]; 5];
        let full_rows = (self.score as usize) / 5;
        #[allow(clippy::needless_range_loop)]
        for r in 0..full_rows {
            values[r] = [1; 5];
        }
        #[allow(clippy::needless_range_loop)]
        for c in 0..(self.score as usize) % 5 {
            values[full_rows][c] = 1;
        }
        values
    }
}

Controls

Nhân vật chính của chúng ta sẽ được điều khiển bởi hai nút bấm ở mặt trước của micro:bit. Nút A sẽ rẽ về bên trái của snake, và nút B sẽ rẽ về bên phải của snake.

Chúng ta sẽ dùng macro microbit::pac::interrupt để xử lý các lần nhấn nút theo cách concurrent. Interrupt sẽ được sinh ra bởi peripheral GPIOTE (General Purpose Input/Output Tasks and Events) của MB2.

Module controls

Chúng ta cần theo dõi hai mảnh trạng thái global mutable riêng biệt: một tham chiếu đến peripheral GPIOTE và một bản ghi về hướng rẽ đã chọn tiếp theo.

Dữ liệu được chia sẻ được bọc trong RefCell để cho phép interior mutability và locking. Bạn có thể tìm hiểu thêm về RefCell bằng cách đọc tài liệu RefCell và chương interior mutability của Rust Book. RefCell lần lượt được bọc trong cortex_m::interrupt::Mutex để cho phép truy cập an toàn. Mutex được cung cấp bởi crate cortex_m sử dụng khái niệm critical section. Dữ liệu trong Mutex chỉ có thể được truy cập từ bên trong một function hoặc closure được truyền vào cortex_m::interrupt::free (được đặt lại tên ở đây thành interrupt_free cho rõ ràng), đảm bảo rằng code trong function hoặc closure đó không thể bị ngắt.

Khởi tạo

Đầu tiên, chúng ta sẽ khởi tạo các nút bấm (src/controls/init.rs).

use super::{Buttons, GPIO};

use cortex_m::interrupt::free as interrupt_free;
use microbit::{
    hal::{
        gpio::{Floating, Input, Pin},
        gpiote::{Gpiote, GpioteChannel},
    },
    pac,
};

/// Initialise the buttons and enable interrupts.
pub fn init_buttons(board_gpiote: pac::GPIOTE, board_buttons: Buttons) {
    let gpiote = Gpiote::new(board_gpiote);

    fn init_channel(channel: &GpioteChannel<'_>, button: &Pin<Input<Floating>>) {
        channel.input_pin(button).hi_to_lo().enable_interrupt();
        channel.reset_events();
    }

    let channel0 = gpiote.channel0();
    init_channel(&channel0, &board_buttons.button_a.degrade());

    let channel1 = gpiote.channel1();
    init_channel(&channel1, &board_buttons.button_b.degrade());

    interrupt_free(move |cs| {
        *GPIO.borrow(cs).borrow_mut() = Some(gpiote);

        unsafe {
            pac::NVIC::unmask(pac::Interrupt::GPIOTE);
        }
        pac::NVIC::unpend(pac::Interrupt::GPIOTE);
    });
}

Peripheral GPIOTE trên nRF52 có 8 "channel", mỗi channel có thể được kết nối với một pin GPIO và được cấu hình để phản hồi với một số sự kiện nhất định, bao gồm rising edge (chuyển đổi từ tín hiệu thấp sang cao) và falling edge (cao sang thấp). Một nút bấm là một pin GPIO có tín hiệu cao khi không được nhấn và thấp khi nhấn. Do đó, một lần nhấn nút là một falling edge.

Lưu ý cách dùng hàm init_channel() trong khởi tạo để tránh copy-paste code khởi tạo nút bấm. Các kiểu mà các crate embedded khác nhau cho MB2 đã ẩn khỏi bạn đôi khi hơi đáng sợ. Tôi khuyến khích bạn khám phá cấu trúc kiểu của các crate HAL và PAC vào một thời điểm nào đó, vì nó hơi lạ và cần thời gian để quen. Đặc biệt, lưu ý rằng mỗi pin trên microbit có kiểu riêng của nó. Mục đích của hàm degrade() trong khởi tạo là chuyển đổi chúng sang một kiểu chung có thể được dùng hợp lý như một đối số cho init_channel() và từ đó đến input_pin().

Chúng ta kết nối channel0 với button_a và channel1 với button_b. Trong mỗi trường hợp, chúng ta thiết lập nút để sinh event trên falling edge (hi_to_lo). Chúng ta lưu tham chiếu đến peripheral GPIOTE của mình trong Mutex GPIO. Sau đó chúng ta unmask interrupt GPIOTE, cho phép chúng được truyền bởi phần cứng, và gọi unpend để xóa bất kỳ interrupt nào có trạng thái pending (có thể đã được sinh ra trước khi interrupt được unmask).

Interrupt handler

Tiếp theo, chúng ta viết code xử lý interrupt. Chúng ta dùng macro interrupt được re-export từ crate nrf52833_hal. Chúng ta định nghĩa một function có cùng tên với interrupt chúng ta muốn xử lý (bạn có thể xem tất cả ở đây) và annotate nó với #[interrupt] (src/controls/interrupt.rs).

use super::{Turn, GPIO, TURN};

use cortex_m::interrupt::free as interrupt_free;
use microbit::pac::{self, interrupt};

#[pac::interrupt]
fn GPIOTE() {
    interrupt_free(|cs| {
        if let Some(gpiote) = GPIO.borrow(cs).borrow().as_ref() {
            let a_pressed = gpiote.channel0().is_event_triggered();
            let b_pressed = gpiote.channel1().is_event_triggered();

            let turn = match (a_pressed, b_pressed) {
                (true, false) => Turn::Left,
                (false, true) => Turn::Right,
                _ => Turn::None,
            };

            gpiote.channel0().reset_events();
            gpiote.channel1().reset_events();

            *TURN.borrow(cs).borrow_mut() = turn;
        }
    });
}

Khi một interrupt GPIOTE được sinh ra, chúng ta kiểm tra mỗi nút để xem nó có được nhấn hay không. Nếu chỉ có nút A được nhấn, chúng ta ghi lại rằng snake nên rẽ sang trái. Nếu chỉ có nút B được nhấn, chúng ta ghi lại rằng nó nên rẽ sang phải. Trong bất kỳ trường hợp nào khác, chúng ta ghi lại rằng snake không nên rẽ. (Có cả hai nút được nhấn "cùng lúc" là cực kỳ khó xảy ra: lần nhấn nút được ghi nhận gần như ngay lập tức, và interrupt handler này chạy rất nhanh — sẽ khó để nhấn cả hai nút kịp thời. Tuy nhiên, Rust buộc bạn phải lên kế hoạch cho các trường hợp bất ngờ này: code sẽ không compile trừ khi bạn kiểm tra tất cả các khả năng.) Lượt rẽ liên quan được lưu trong Mutex TURN. Tất cả điều này xảy ra trong một block interrupt_free, để đảm bảo rằng chúng ta không thể bị ngắt bởi một số event khác khi xử lý interrupt này.

Cuối cùng, chúng ta expose một function đơn giản để lấy lượt rẽ tiếp theo (src/controls.rs).

mod init;
mod interrupt;

pub use init::init_buttons;

use crate::game::Turn;
use core::cell::RefCell;
use cortex_m::interrupt::{free as interrupt_free, Mutex};
use microbit::{board::Buttons, hal::gpiote::Gpiote};
pub static GPIO: Mutex<RefCell<Option<Gpiote>>> = Mutex::new(RefCell::new(None));
pub static TURN: Mutex<RefCell<Turn>> = Mutex::new(RefCell::new(Turn::None));

/// Get the next turn (ie, the turn corresponding to the most recently pressed button).
pub fn get_turn(reset: bool) -> Turn {
    interrupt_free(|cs| {
        let turn = *TURN.borrow(cs).borrow();
        if reset {
            *TURN.borrow(cs).borrow_mut() = Turn::None
        }
        turn
    })
}

Function này đơn giản trả về giá trị hiện tại của Mutex TURN. Nó nhận một đối số boolean, reset. Nếu reset là true, giá trị của TURN được reset, tức là đặt về Turn::None.

Tiếp theo chúng ta sẽ xây dựng hỗ trợ cho một màn hình game độ trung thực cao.

Dùng display non-blocking

Tiếp theo chúng ta sẽ hiển thị snake và thức ăn trên các LED của màn hình MB2. Cho đến nay, chúng ta đã dùng interface blocking, cung cấp cho các LED có thể sáng tối đa hoặc tắt. Với điều này, một game snake cơ bản đang hoạt động sẽ là có thể. Nhưng bạn có thể thấy rằng khi snake dài hơn một chút, sẽ khó phân biệt snake với thức ăn, và xác định hướng snake đang đi. Hãy tìm cách cho phép độ sáng LED thay đổi: chúng ta có thể làm cho thân snake hơi tối hơn, điều này sẽ giúp giải quyết sự lộn xộn.

Thư viện microbit cung cấp hai interface khác nhau đến ma trận LED. Có interface blocking chúng ta đã thấy trong các chương trước. Cũng có một interface non-blocking cho phép bạn tùy chỉnh độ sáng của từng LED. Ở cấp độ phần cứng, mỗi LED chỉ "bật" hoặc "tắt", nhưng module microbit::display::nonblocking mô phỏng mười mức độ sáng cho mỗi LED bằng cách nhanh chóng chuyển đổi LED bật và tắt.

(Không có lý do gì tuyệt vời để hai chế độ display của crate thư viện microbit phải tách biệt và dùng code riêng. Một thiết kế hoàn chỉnh hơn sẽ cho phép sử dụng non-blocking hoặc blocking của một API display duy nhất với mức độ sáng thay đổi và refresh rate được chỉ định bởi người dùng. Đừng bao giờ cho rằng những thứ bạn được giao là hoàn hảo, hoặc thậm chí gần hoàn hảo. Luôn nghĩ về những gì bạn có thể làm khác đi. Tuy nhiên, bây giờ, chúng ta sẽ làm việc với những gì chúng ta có, đủ cho mục đích ngay trước mắt.)

Code để tương tác với interface non-blocking (src/display.rs) khá đơn giản và sẽ theo cấu trúc tương tự như code chúng ta đã dùng để tương tác với các nút bấm. Lần này chúng ta sẽ bắt đầu từ top level.

Module Display

pub mod interrupt;
pub mod show;

pub use show::{clear_display, display_image};

use core::cell::RefCell;
use cortex_m::interrupt::{free as interrupt_free, Mutex};
use microbit::display::nonblocking::Display;
use microbit::gpio::DisplayPins;
use microbit::pac;
use microbit::pac::TIMER1;

static DISPLAY: Mutex<RefCell<Option<Display<TIMER1>>>> = Mutex::new(RefCell::new(None));

pub fn init_display(board_timer: TIMER1, board_display: DisplayPins) {
    let display = Display::new(board_timer, board_display);

    interrupt_free(move |cs| {
        *DISPLAY.borrow(cs).borrow_mut() = Some(display);
    });
    unsafe { pac::NVIC::unmask(pac::Interrupt::TIMER1) }
}

Đầu tiên, chúng ta khởi tạo một struct microbit::display::nonblocking::Display đại diện cho màn hình LED, truyền vào peripheral TIMER1 và DisplayPins của board. Sau đó chúng ta lưu display trong một Mutex. Cuối cùng, chúng ta unmask interrupt TIMER1.

Display API

Sau đó chúng ta định nghĩa một vài convenience function cho phép chúng ta dễ dàng set (hoặc unset) hình ảnh được hiển thị (src/display/show.rs).

use super::DISPLAY;

use cortex_m::interrupt::free as interrupt_free;

use tiny_led_matrix::Render;

/// Display an image.
pub fn display_image(image: &impl Render) {
    interrupt_free(|cs| {
        if let Some(display) = DISPLAY.borrow(cs).borrow_mut().as_mut() {
            display.show(image);
        }
    })
}

/// Clear the display (turn off all LEDs).
pub fn clear_display() {
    interrupt_free(|cs| {
        if let Some(display) = DISPLAY.borrow(cs).borrow_mut().as_mut() {
            display.clear();
        }
    })
}

display_image nhận một hình ảnh và yêu cầu display hiển thị nó. Giống như method Display::show mà nó gọi, function này nhận một struct implement trait tiny_led_matrix::Render. Trait đó đảm bảo rằng struct chứa dữ liệu và các method cần thiết để Display render trên ma trận LED. Hai implementation của Render được cung cấp bởi module microbit::display::nonblocking là BitImage và GreyscaleImage. Trong một BitImage, mỗi "pixel" (hoặc LED) hoặc được chiếu sáng hoặc không (như khi chúng ta dùng blocking interface), trong khi đó trong GreyscaleImage mỗi "pixel" có thể có độ sáng khác nhau.

clear_display làm đúng như tên gọi của nó.

Xử lý interrupt cho Display

Cuối cùng, chúng ta dùng macro interrupt để định nghĩa một handler cho interrupt TIMER1. Interrupt này kích hoạt nhiều lần trong một giây, và đây là điều cho phép Display nhanh chóng chu kỳ các LED khác nhau bật và tắt để tạo ra ảo giác về các mức độ sáng khác nhau. Tất cả code handler của chúng ta làm là gọi method Display::handle_display_event, xử lý điều này (src/display/interrupt.rs).

use super::DISPLAY;

use cortex_m::interrupt::free as interrupt_free;
use microbit::pac::{self, interrupt};

#[pac::interrupt]
fn TIMER1() {
    interrupt_free(|cs| {
        if let Some(display) = DISPLAY.borrow(cs).borrow_mut().as_mut() {
            display.handle_display_event();
        }
    })
}

Bây giờ chúng ta có thể hiểu cách hàm main của chúng ta sẽ làm việc với display: chúng ta sẽ gọi init_display và dùng các function mới mà chúng ta đã định nghĩa để tương tác với nó.

Snake game: lắp ráp cuối cùng

Code trong file src/main.rs của chúng ta kết hợp tất cả các cơ chế đã được thảo luận trước đó để tạo ra game cuối cùng.

#![no_main]
#![no_std]

mod controls;
mod display;
pub mod game;

use controls::{get_turn, init_buttons};
use display::{clear_display, display_image, init_display};
use game::{Game, GameStatus};

use cortex_m_rt::entry;
use embedded_hal::delay::DelayNs;
use microbit::{
    display::nonblocking::{BitImage, GreyscaleImage},
    hal::{Rng, Timer},
    Board,
};
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = Board::take().unwrap();
    let mut timer = Timer::new(board.TIMER0).into_periodic();
    let mut rng = Rng::new(board.RNG);
    let mut game = Game::new(&mut rng);

    init_buttons(board.GPIOTE, board.buttons);
    init_display(board.TIMER1, board.display_pins);

    loop {
        loop {
            // Game loop
            let image = GreyscaleImage::new(&game.game_matrix(6, 3, 9));
            display_image(&image);
            timer.delay_ms(game.step_len_ms());
            match game.status {
                GameStatus::Ongoing => game.step(get_turn(true)),
                _ => {
                    for _ in 0..3 {
                        clear_display();
                        timer.delay_ms(200u32);
                        display_image(&image);
                        timer.delay_ms(200u32);
                    }
                    clear_display();
                    display_image(&BitImage::new(&game.score_matrix()));
                    timer.delay_ms(2000u32);
                    break;
                }
            }
        }
        game.reset();
    }
}

Sau khi khởi tạo board và các peripheral timer và RNG của nó, chúng ta khởi tạo một struct Game và một Display từ module microbit::display::blocking.

Trong "vòng lặp game" (chạy bên trong "vòng lặp chính" mà chúng ta đặt trong hàm main), chúng ta liên tục thực hiện các bước sau:

  1. Lấy một mảng 5×5 bytes biểu diễn lưới. Method Game::get_matrix nhận ba đối số nguyên (nên từ 0 đến 9, bao gồm) để biểu diễn cuối cùng độ sáng của đầu, đuôi và thức ăn.

  2. Hiển thị ma trận, trong một khoảng thời gian được xác định bởi method Game::step_len_ms. Như hiện tại được implement, method này cơ bản cung cấp 1 giây giữa các bước, giảm 200ms mỗi khi người chơi ghi được 5 điểm (ăn 1 thức ăn = 1 điểm), với mức tối thiểu là 200ms.

  3. Kiểm tra trạng thái game. Nếu là Ongoing (giá trị ban đầu của nó), chạy một bước game và cập nhật trạng thái game (bao gồm thuộc tính status của nó). Nếu không, game đã kết thúc, vì vậy nhấp nháy hình ảnh hiện tại ba lần, sau đó hiển thị điểm số của người chơi (được biểu diễn bằng số lượng LED sáng tương ứng với điểm số), và thoát vòng lặp game.

Vòng lặp chính của chúng ta chỉ chạy vòng lặp game liên tục, reset trạng thái game sau mỗi lần lặp.
