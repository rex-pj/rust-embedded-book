UART

Microcontroller có một peripheral gọi là UART, viết tắt của Universal Asynchronous Receiver/Transmitter. Peripheral này có thể được cấu hình để hoạt động với nhiều giao thức truyền thông, bao gồm giao thức serial communication.

Xuyên suốt chương này, chúng ta sẽ dùng serial communication để trao đổi thông tin giữa microcontroller và máy tính của bạn.

LƯU Ý: trên micro:bit v2, chúng ta sẽ dùng peripheral gọi là UARTE, hoạt động giống hệt UART thông thường, chỉ khác là HAL phải giao tiếp với nó theo cách khác. Tuy nhiên, điều này không phải là mối bận tâm của chúng ta.

Setup

Như thường lệ từ bây giờ, bạn sẽ phải chỉnh sửa Embed.toml cho phù hợp với phiên bản micro:bit của bạn:

[default.general]
chip = "nrf52833_xxAA" # uncomment this line for micro:bit V2
# chip = "nrf51822_xxAA" # uncomment this line for micro:bit V1

[default.reset]
halt_afterwards = false

[default.rtt]
enabled = true

[default.gdb]
enabled = false

Gửi một byte đơn

Nhiệm vụ đầu tiên của chúng ta sẽ là gửi một byte đơn từ microcontroller đến máy tính qua kết nối serial.

Để làm điều đó, chúng ta sẽ dùng đoạn code sau (đoạn này đã có sẵn trong 07-uart/src/main.rs):

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

#[cfg(feature = "v1")]
use microbit::{
    hal::prelude::*,
    hal::uart,
    hal::uart::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
use microbit::{
    hal::prelude::*,
    hal::uarte,
    hal::uarte::{Baudrate, Parity},
};

#[cfg(feature = "v1")]
use embedded_io::Write;

#[cfg(feature = "v2")]
use embedded_hal_nb::serial::Write;

#[cfg(feature = "v2")]
mod serial_setup;
#[cfg(feature = "v2")]
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let mut serial = {
        let serial = uart::Uart::new(
            board.UART0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        serial
    };

    #[cfg(feature = "v2")]
    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    #[cfg(feature = "v1")]
    serial.write(&[b'X']).unwrap();

    #[cfg(feature = "v2")]
    {
        nb::block!(serial.write(b'X')).unwrap();
        nb::block!(serial.flush()).unwrap();
    }

    loop {}
}

Điều mới nhất ở đây rõ ràng là các directive cfg để bao gồm/loại trừ các phần code có điều kiện. Điều này chủ yếu vì chúng ta muốn làm việc với UART thông thường cho micro:bit v1 và với UARTE cho micro:bit v2.

Bạn cũng sẽ nhận thấy đây là lần đầu tiên chúng ta include code không phải từ thư viện, cụ thể là module serial_setup. Mục đích duy nhất của nó là cung cấp một wrapper đẹp quanh UARTE để chúng ta có thể dùng nó cùng cách hệt như UART thông qua các trait embedded_hal::serial. Nếu muốn, bạn có thể xem module này làm gì cụ thể, nhưng không bắt buộc phải hiểu nó để hiểu chương này nói chung.

Ngoài những khác biệt đó, quy trình khởi tạo cho UART và UARTE khá tương tự nên chúng ta sẽ chỉ thảo luận về khởi tạo UARTE. UARTE được khởi tạo bằng đoạn code:

uarte::Uarte::new(
    board.UARTE0,
    board.uart.into(),
    Parity::EXCLUDED,
    Baudrate::BAUD115200,
);

Hàm này lấy ownership của biểu diễn peripheral UARTE trong Rust (board.UARTE0) và các pin TX/RX trên board (board.uart.into()) để không ai khác có thể can thiệp vào peripheral UARTE hoặc các pin của chúng ta khi đang dùng. Sau đó chúng ta truyền hai tùy chọn cấu hình cho constructor: baudrate (quen thuộc rồi) cũng như một tùy chọn gọi là "parity". Parity là cách cho phép các đường serial communication kiểm tra xem dữ liệu nhận được có bị hỏng trong quá trình truyền không. Chúng ta không muốn dùng nó ở đây nên loại trừ nó. Sau đó wrap nó trong kiểu UartePort để dùng cùng cách như serial của micro:bit v1.

Sau khởi tạo, chúng ta gửi ký tự X qua uart instance mới tạo. Macro block! ở đây là nb::block!. nb là (trích dẫn từ mô tả của nó) "Minimal and reusable non-blocking I/O layer". Nó cho phép chúng ta viết code có thể thực hiện các thao tác phần cứng ở background trong khi chúng ta đi làm việc khác (non-blocking). Tuy nhiên, trong trường hợp này và nhiều trường hợp khác, chúng ta không muốn làm việc gì khác nên chỉ gọi block! sẽ chờ cho đến khi thao tác I/O hoàn thành.

Cuối cùng nhưng không kém quan trọng, chúng ta flush() serial port. Điều này vì một implementor của các trait embedded-hal::serial có thể quyết định buffer output cho đến khi nhận đủ một số lượng byte nhất định để gửi (đây là trường hợp với implementation UARTE). Gọi flush() ép nó ghi ngay các byte hiện có thay vì chờ thêm.

Thử nghiệm

Trước khi flash, bạn nên đảm bảo đã khởi động minicom/PuTTY vì dữ liệu nhận qua serial communication không được lưu trữ hay gì cả — chúng ta phải xem trực tiếp. Khi serial monitor đã chạy, bạn có thể flash chương trình:

# Cho micro:bit v2
$ cargo embed --features v2 --target thumbv7em-none-eabihf
# Cho micro:bit v1
$ cargo embed --features v1 --target thumbv6m-none-eabi

Sau khi flash xong, bạn sẽ thấy ký tự X xuất hiện trên terminal minicom/PuTTY của bạn.

Gửi một chuỗi

Bài tập tiếp theo là gửi chuỗi "The quick brown fox jumps over the lazy dog." từ microcontroller đến máy tính của bạn. Đến lượt bạn tự viết chương trình này.

Lời giải

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

#[cfg(feature = "v1")]
use microbit::{
    hal::prelude::*,
    hal::uart,
    hal::uart::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
use microbit::{
    hal::prelude::*,
    hal::uarte,
    hal::uarte::{Baudrate, Parity},
};

#[cfg(feature = "v1")]
use embedded_io::Write;

#[cfg(feature = "v2")]
use embedded_hal_nb::serial::Write;

#[cfg(feature = "v2")]
mod serial_setup;
#[cfg(feature = "v2")]
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let mut serial = {
        uart::Uart::new(
            board.UART0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        )
    };

    #[cfg(feature = "v2")]
    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    #[cfg(feature = "v1")]
    serial.write(b"The quick brown fox jumps over the lazy dog.\r\n").unwrap();

    #[cfg(feature = "v2")]
    {
        let s = b"The quick brown fox jumps over the lazy dog.\r\n";
        for byte in s.iter() {
            nb::block!(serial.write(*byte)).unwrap();
        }
        nb::block!(serial.flush()).unwrap();
    }

    loop {}
}

Nhận một byte đơn

Bây giờ chúng ta sẽ làm chiều ngược lại: nhận dữ liệu từ máy tính vào microcontroller.

Thư viện embedded-hal cung cấp hàm read() để chờ cho đến khi có một byte và đọc nó:

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

#[cfg(feature = "v1")]
use microbit::{
    hal::prelude::*,
    hal::uart,
    hal::uart::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
use microbit::{
    hal::prelude::*,
    hal::uarte,
    hal::uarte::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
mod serial_setup;
#[cfg(feature = "v2")]
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let mut serial = {
        uart::Uart::new(
            board.UART0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        )
    };

    #[cfg(feature = "v2")]
    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    loop {
        let byte = nb::block!(serial.read()).unwrap();
        rprintln!("{}", byte);
    }
}

Nếu bạn flash chương trình này và gõ ký tự trong minicom để gửi đến microcontroller, bạn sẽ chỉ thấy các con số trên RTT console vì code nhận giá trị u8 thô. Việc chuyển đổi giá trị u8 đó thành char để in ra là bài tập cho bạn tự làm.

Lời giải

    loop {
        let byte = nb::block!(serial.read()).unwrap();
        rprintln!("{}", byte as char);
    }

Echo Server

Hãy kết hợp truyền và nhận vào một chương trình để viết một echo server. Echo server gửi lại cho client chính xác những gì nó nhận được — microcontroller đóng vai server, máy tính của bạn đóng vai client.

Điều này khá đơn giản để implement. Gợi ý: làm từng byte một.

Lời giải

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

#[cfg(feature = "v1")]

#[cfg(feature = "v1")]
use embedded_io::Write;

#[cfg(feature = "v2")]
use embedded_hal_nb::serial::Write;

#[cfg(feature = "v2")]
mod serial_setup;
#[cfg(feature = "v2")]
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let mut serial = {
        uart::Uart::new(
            board.UART0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        )
    };

    #[cfg(feature = "v2")]
    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    loop {
        let byte = nb::block!(serial.read()).unwrap();
        nb::block!(serial.write(byte)).unwrap();
        nb::block!(serial.flush()).unwrap();
    }
}

Đảo ngược chuỗi

Lần này, server cần phản hồi mỗi khi client nhấn ENTER, với mỗi phản hồi trên một dòng mới. Mỗi phản hồi là chuỗi ký tự mà client gửi — nhưng bị đảo ngược.

Lời giải của tôi

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use core::fmt::Write;
use heapless::Vec;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

#[cfg(feature = "v1")]
use microbit::{
    hal::prelude::*,
    hal::uart,
    hal::uart::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
use microbit::{
    hal::prelude::*,
    hal::uarte,
    hal::uarte::{Baudrate, Parity},
};

#[cfg(feature = "v2")]
mod serial_setup;
#[cfg(feature = "v2")]
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    #[cfg(feature = "v1")]
    let mut serial = {
        uart::Uart::new(
            board.UART0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        )
    };

    #[cfg(feature = "v2")]
    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    // A buffer with 32 bytes of capacity
    let mut buffer: Vec<u8, 32> = Vec::new();

    loop {
        buffer.clear();

        loop {
            // We assume that the receiving cannot fail
            let byte = nb::block!(serial.read()).unwrap();

            if buffer.push(byte).is_err() {
                write!(serial, "error: buffer full\r\n").unwrap();
                break;
            }

            if byte == 13 {
                for byte in buffer.iter().rev().chain(&[b'\n', b'\r']) {
                    nb::block!(serial.write(*byte)).unwrap();
                }
                break;
            }
        }
        nb::block!(serial.flush()).unwrap()
    }
}
