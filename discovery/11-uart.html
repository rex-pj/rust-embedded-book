UART

Microcontroller có một peripheral gọi là UART, viết tắt của Universal Asynchronous Receiver/Transmitter. Peripheral này có thể được cấu hình để hoạt động với nhiều giao thức truyền thông, bao gồm giao thức serial communication.

Xuyên suốt chương này, chúng ta sẽ dùng serial communication để trao đổi thông tin giữa microcontroller và máy tính của bạn.

Có hai loại UART peripheral trên MB2: loại cũ hơn là UART và loại mới hơn là UARTE ("UART with Easy DMA"). Trong chương này, chúng ta sẽ dùng peripheral UARTE để giao tiếp serial giữa microcontroller và máy tính.

Gửi một byte đơn

Nhiệm vụ đầu tiên của chúng ta sẽ là gửi một byte đơn từ microcontroller đến máy tính qua kết nối serial.

Để làm điều đó, chúng ta sẽ dùng đoạn code sau (file examples/send-byte.rs):

#![no_main]
#![no_std]

use cortex_m::asm::wfi;
use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

use microbit::{
    hal::uarte,
    hal::uarte::{Baudrate, Parity},
};

use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    serial.write(b'X').unwrap();
    serial.flush().unwrap();

    loop {
        wfi();
    }
}

Bạn có thể nhận thấy rằng một trong những thư viện được dùng ở đây, module serial_setup, không đến từ crates.io mà được viết riêng cho dự án này. Mục đích của serial_setup là cung cấp một wrapper tiện lợi xung quanh peripheral UARTE. Nếu muốn, bạn có thể xem module này làm gì, nhưng bạn không cần phải hiểu nó để theo dõi chương này nói chung.

Tiếp theo chúng ta sẽ thảo luận về việc khởi tạo UARTE. UARTE được khởi tạo bằng đoạn code sau:

uarte::Uarte::new(
    board.UARTE0,
    board.uart.into(),
    Parity::EXCLUDED,
    Baudrate::BAUD115200,
);

Cụ thể, chúng ta lấy ownership của peripheral UARTE trong Rust (board.UARTE0) và các pin TX/RX trên board (board.uart.into()) để không ai khác có thể can thiệp vào chúng khi đang dùng. Sau đó chúng ta truyền hai tùy chọn cấu hình: baudrate và parity. Parity là cách để kiểm tra dữ liệu có bị hỏng trong quá trình truyền không — chúng ta không dùng nó ở đây nên loại trừ nó. Cuối cùng, wrap UARTE trong kiểu UartePort.

Sau khởi tạo, chúng ta gửi X (giá trị byte ASCII là 88) qua instance uart mới tạo.

💡
Các hàm serial là "blocking"

Các hàm write() và read() chờ cho đến khi dữ liệu được gửi/nhận xong trước khi trả về. Microcontroller sẽ bị kẹt ở đó và không làm gì khác trong thời gian chờ. Đây không phải lúc nào cũng là điều mong muốn, nhưng trong trường hợp của chúng ta nó tiện lợi.

💡
Tại sao cần gọi flush()?

UARTE có thể buffer output cho đến khi nhận đủ một số byte nhất định rồi mới gửi cùng lúc. Gọi flush() buộc nó gửi ngay các byte đang có thay vì chờ thêm — tránh tình trạng byte "mắc kẹt" trong buffer.

Kiểm tra

⚠️
Khởi động minicom/PuTTY trước khi flash

Dữ liệu nhận qua serial communication không được lưu trữ — chúng ta phải xem trực tiếp. Nếu bạn flash trước rồi mới mở terminal, bạn sẽ bỏ lỡ ký tự được gửi. Hãy đảm bảo serial monitor đã chạy trước khi flash.

Khi serial monitor đã chạy, bạn có thể flash chương trình:

$ cargo embed --example send-byte

Sau khi flash xong, bạn sẽ thấy ký tự X xuất hiện trên terminal minicom/PuTTY của bạn, xin chúc mừng!

Nếu bạn bỏ lỡ, bạn có thể nhấn nút reset ở mặt sau của MB2. Điều này sẽ khiến chương trình bắt đầu lại từ đầu và gửi lại X.

Gửi một chuỗi

Nhiệm vụ tiếp theo sẽ là gửi toàn bộ chuỗi từ microcontroller đến máy tính của bạn.

Tôi muốn bạn gửi chuỗi "The quick brown fox jumps over the lazy dog." từ microcontroller đến máy tính của bạn.

Đến lượt bạn viết chương trình.

Naive approach và write!

Cách gửi chuỗi ở phần trước (duyệt từng byte) hoạt động được, nhưng thiếu đi sự tiện lợi của print! như formatting tham số. Có hai cách tiếp cận:

Naive approach

Bạn có lẽ đã nghĩ ra một chương trình tương tự như sau (examples/naive-send-string.rs):

rust
#![no_main]
#![no_std]

use cortex_m::asm::wfi;
use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

use microbit::hal::uarte::{self, Baudrate, Parity};

use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    for byte in b"The quick brown fox jumps over the lazy dog.\r\n".iter() {
        serial.write(*byte).unwrap();
    }
    serial.flush().unwrap();

    loop {
        wfi();
    }
}

Đây là một implementation hoàn toàn hợp lệ, nhưng tại một thời điểm nào đó bạn có thể muốn có tất cả những tính năng tiện lợi của print! như formatting đối số v.v. Nếu bạn đang tự hỏi cách làm, hãy đọc tiếp.

write! và core::fmt::Write

Trait core::fmt::Write cho phép chúng ta dùng bất kỳ struct nào implement nó theo cách cơ bản giống như chúng ta dùng print! trong thế giới std. Trong trường hợp này, struct Uart từ HAL nrf có implement core::fmt::Write, vì vậy chúng ta có thể refactor chương trình trước thành này (examples/send-string.rs):

rust
#![no_main]
#![no_std]

use core::fmt::Write;
use cortex_m::asm::wfi;
use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::rtt_init_print;

use microbit::hal::uarte::{self, Baudrate, Parity};

use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    write!(serial, "The quick brown fox jumps over the lazy dog.\r\n").unwrap();
    serial.flush().unwrap();

    loop {
        wfi();
    }
}

Nếu bạn flash chương trình này lên micro:bit, bạn sẽ thấy rằng nó tương đương về mặt chức năng với chương trình dựa trên iterator mà bạn đã nghĩ ra.

Nhận một byte đơn

Cho đến nay chúng ta có thể gửi dữ liệu từ microcontroller đến máy tính của bạn. Đã đến lúc thử chiều ngược lại: nhận dữ liệu từ máy tính (examples/receive-byte.rs).

#![no_main]
#![no_std]

use cortex_m_rt::entry;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

use microbit::hal::uarte::{self, Baudrate, Parity};

use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    loop {
        let byte = serial.read().unwrap();
        rprintln!("{}", byte);
    }
}

Phần duy nhất thay đổi, so với chương trình gửi byte của chúng ta, là vòng lặp ở cuối main(). Ở đây chúng ta dùng hàm serial.read() để chờ cho đến khi có một byte và đọc nó. Sau đó chúng ta in byte đó vào RTT debugging console để xem liệu mọi thứ có thực sự đến hay không.

Lưu ý rằng nếu bạn flash chương trình này và bắt đầu gõ ký tự trong minicom để gửi chúng đến microcontroller của bạn, bạn sẽ chỉ có thể thấy các con số trong RTT console vì chúng ta không chuyển đổi u8 mà chúng ta nhận được thành char thực sự. Vì việc chuyển đổi từ u8 sang char khá đơn giản, tôi sẽ để nhiệm vụ này cho bạn nếu bạn thực sự muốn thấy các ký tự trong RTT console.

Echo server

Hãy hợp nhất truyền và nhận vào một chương trình duy nhất và viết một echo server. Một echo server gửi lại cho client cùng text mà nó nhận được. Đối với ứng dụng này, microcontroller sẽ là server và bạn cùng máy tính của bạn sẽ là client.

Điều này sẽ đơn giản để implement. (gợi ý: làm từng byte một)

Đảo ngược chuỗi

Được rồi, tiếp theo hãy làm server thú vị hơn bằng cách cho nó phản hồi với client với chuỗi ngược của text mà chúng đã gửi. Server sẽ phản hồi với client mỗi khi họ nhấn phím ENTER. Mỗi phản hồi của server sẽ trên một dòng mới.

Lời giải của tôi

Bạn sẽ tìm thấy lời giải của tôi trong src/main.rs:

#![no_main]
#![no_std]

use core::fmt::Write;
use cortex_m_rt::entry;
use heapless::Vec;
use microbit::hal::uarte::{self, Baudrate, Parity};
use panic_rtt_target as _;
use rtt_target::rtt_init_print;
use serial_setup::UartePort;

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = microbit::Board::take().unwrap();

    let mut serial = {
        let serial = uarte::Uarte::new(
            board.UARTE0,
            board.uart.into(),
            Parity::EXCLUDED,
            Baudrate::BAUD115200,
        );
        UartePort::new(serial)
    };

    // A buffer with 32 bytes of capacity
    let mut buffer: Vec<u8, 32> = Vec::new();

    loop {
        buffer.clear();

        loop {
            // We assume that the receiving cannot fail
            let byte = serial.read().unwrap();

            if buffer.push(byte).is_err() {
                write!(serial, "error: buffer full\r\n").unwrap();
                break;
            }

            if byte == b'\r' {
                for byte in buffer.iter().rev().chain(b"\r\n") {
                    serial.write(*byte).unwrap();
                }
                break;
            }
        }
        serial.flush().unwrap()
    }
}
