Cài đặt môi trường phát triển

Làm việc với microcontroller đòi hỏi một bộ công cụ đặc biệt vì chúng ta sẽ làm việc với một kiến trúc khác với máy tính của bạn, và cần phải chạy cũng như debug chương trình trên một thiết bị "remote".

Tài liệu tham khảo

Tuy nhiên công cụ không phải là tất cả. Nếu không có tài liệu, việc làm việc với microcontroller gần như là bất khả thi. Chúng ta sẽ tham khảo tất cả những tài liệu này xuyên suốt cuốn sách:

  • LSM303AGR

Công cụ cần thiết

Chúng ta sẽ dùng tất cả các công cụ được liệt kê bên dưới. Nếu không chỉ định phiên bản tối thiểu, bất kỳ phiên bản gần đây nào cũng sẽ hoạt động, nhưng chúng tôi đã liệt kê phiên bản đã được thử nghiệm.

  • Rust 1.57.0 hoặc toolchain mới hơn.
  • gdb-multiarch. Phiên bản đã thử nghiệm: 10.2. Các phiên bản khác rất có thể cũng hoạt động. Nếu bản phân phối/nền tảng của bạn không có gdb-multiarch, thì arm-none-eabi-gdb cũng được. Hơn nữa, một số binary gdb thông thường cũng được build với khả năng multiarch — bạn có thể tìm thêm thông tin về điều này trong các phần con.
  • cargo-binutils. Phiên bản 0.3.3 hoặc mới hơn.
  • cargo-embed. Phiên bản 0.24.0 hoặc mới hơn.
  • minicom trên Linux và macOS. Phiên bản đã thử nghiệm: 2.7.1. Các phiên bản khác rất có thể cũng hoạt động.
  • PuTTY trên Windows.

Tiếp theo, hãy làm theo hướng dẫn cài đặt chung (không phụ thuộc hệ điều hành) cho một số công cụ:

rustc & Cargo

Cài đặt rustup bằng cách làm theo hướng dẫn tại https://rustup.rs.

Nếu bạn đã cài rustup, hãy đảm bảo bạn đang ở kênh stable và toolchain stable của bạn đã được cập nhật. rustc -V nên trả về ngày mới hơn ngày hiển thị bên dưới:

$ rustc -V
rustc 1.53.0 (53cb7b09b 2021-06-17)

cargo-binutils

$ rustup component add llvm-tools
$ cargo install cargo-binutils --vers 0.3.3
$ cargo size --version
cargo-size 0.3.3

cargo-embed

Để cài đặt cargo-embed, trước hết hãy cài đặt các prerequisites của nó (lưu ý: những hướng dẫn này là một phần của bộ công cụ debug embedded tổng quát hơn: probe-rs). Sau đó cài đặt bằng cargo:

$ cargo install --locked probe-rs-tools --vers '^0.24'

LƯU Ý: Quá trình này có thể thất bại do các thay đổi thường xuyên trong probe-rs. Nếu vậy, hãy truy cập https://probe.rs và làm theo hướng dẫn cài đặt hiện tại ở đó.

Cuối cùng, xác nhận rằng bạn đã cài đặt thành công cargo-embed bằng cách chạy:

$ cargo embed --version
cargo-embed 0.24.0 (git commit: crates.io)

Repository của sách này

Vì cuốn sách này cũng chứa một số code base Rust nhỏ được dùng trong các chương khác nhau, bạn cũng sẽ cần tải mã nguồn của nó. Bạn có thể làm điều này bằng một trong các cách sau:

  • Truy cập repository, nhấn nút "Code" màu xanh rồi nhấn "Download Zip"
  • Clone bằng git (nếu bạn biết git thì chắc đã cài rồi) từ cùng repository đã nêu trên

Hướng dẫn theo hệ điều hành

Bây giờ hãy làm theo hướng dẫn dành riêng cho hệ điều hành bạn đang dùng:

🐧 Linux

Dưới đây là các lệnh cài đặt cho một số bản phân phối Linux.

Ubuntu 20.04 trở lên / Debian 10 trở lên

LƯU Ý: gdb-multiarch là lệnh GDB bạn sẽ dùng để debug chương trình ARM Cortex-M

$ sudo apt-get install \
    gdb-multiarch \
    minicom

Fedora 32 trở lên

LƯU Ý: gdb là lệnh GDB bạn sẽ dùng để debug chương trình ARM Cortex-M

$ sudo dnf install \
    gdb \
    minicom

Arch Linux

LƯU Ý: arm-none-eabi-gdb là lệnh GDB bạn sẽ dùng để debug chương trình ARM Cortex-M

$ sudo pacman -S \
    arm-none-eabi-gdb \
    minicom

Các bản phân phối khác

LƯU Ý: arm-none-eabi-gdb là lệnh GDB bạn sẽ dùng để debug chương trình ARM Cortex-M

Với các bản phân phối không có package cho pre-built toolchain của ARM, hãy tải file "Linux 64-bit" và đưa thư mục bin của nó vào PATH. Đây là một cách làm:

$ mkdir -p ~/local && cd ~/local
$ tar xjf /path/to/downloaded/file/gcc-arm-none-eabi-9-2020-q2-update-x86_64-linux.tar.bz2

Sau đó, dùng trình soạn thảo yêu thích để thêm vào PATH trong file init của shell phù hợp (ví dụ ~/.zshrc hoặc ~/.bashrc):

PATH=$PATH:$HOME/local/gcc-arm-none-eabi-9-2020-q2-update/bin

Cấu hình udev rules

Các rules này cho phép bạn dùng thiết bị USB như micro:bit mà không cần quyền root, tức là không cần sudo.

Tạo file này trong /etc/udev/rules.d với nội dung bên dưới.

$ cat /etc/udev/rules.d/69-microbit.rules
# CMSIS-DAP for microbit
ACTION!="add|change", GOTO="microbit_rules_end"
SUBSYSTEM=="usb", ATTR{idVendor}=="0d28", ATTR{idProduct}=="0204", TAG+="uaccess"
LABEL="microbit_rules_end"

Sau đó reload udev rules bằng:

$ sudo udevadm control --reload

Nếu bạn đã cắm board nào vào máy tính, hãy rút ra rồi cắm lại, hoặc chạy lệnh sau:

$ sudo udevadm trigger

🪟 Windows

arm-none-eabi-gdb

ARM cung cấp file .exe installer cho Windows. Tải một file từ đây, và làm theo hướng dẫn. Ngay trước khi quá trình cài đặt hoàn tất, hãy tick/chọn tùy chọn "Add path to environment variable". Sau đó xác nhận rằng các công cụ đã nằm trong %PATH% của bạn:

$ arm-none-eabi-gcc -v
(..)
gcc version 5.4.1 20160919 (release) (..)

PuTTY

Tải file putty.exe mới nhất từ trang này và đặt nó vào đâu đó trong %PATH% của bạn.

🍎 macOS

Tất cả các công cụ có thể được cài đặt bằng Homebrew:

$ # ARM GCC debugger
$ brew install arm-none-eabi-gdb
$ # Minicom
$ brew install minicom
$ # lsusb (liệt kê thiết bị USB kết nối)
$ brew install lsusb

Xác nhận cài đặt

Hãy xác nhận rằng tất cả các công cụ đã được cài đặt đúng cách.

Chỉ Linux

Xác nhận quyền truy cập

Kết nối micro:bit với máy tính bằng cáp USB.

Micro:bit bây giờ sẽ xuất hiện dưới dạng thiết bị USB (file) trong /dev/bus/usb. Hãy tìm xem nó được gán số nào:

$ lsusb | grep -i "NXP ARM mbed"
Bus 001 Device 065: ID 0d28:0204 NXP ARM mbed
$ #                ]+]+]+]+]+]+]]]]]]]]]]]]]]  ^^^]+]+]]+]]+]]+]^^^

Trong trường hợp của tôi, micro:bit được kết nối vào bus #1 và được gán là device #65. Điều này có nghĩa file /dev/bus/usb/001/065 chính là micro:bit. Hãy kiểm tra quyền của file:

$ ls -l /dev/bus/usb/001/065
crw-rw-r--+ 1 nobody nobody 189, 64 Sep  5 14:27 /dev/bus/usb/001/065

Quyền nên là crw-rw-r--+, lưu ý dấu + ở cuối, sau đó xem quyền truy cập của bạn bằng lệnh sau:

$ getfacl /dev/bus/usb/001/065
getfacl: Removing leadin '/' from absolute path names
# file: dev/bus/usb/001/065
# owner: nobody
# group: nobody
user::rw-
user:<YOUR-USER-NAME>:rw-
group::rw-
mask::rw-
other::r-

Bạn nên thấy username của mình trong danh sách trên với quyền rw-. Nếu không, hãy kiểm tra lại udev rules và thử reload chúng bằng:

$ sudo udevadm control --reload
$ sudo udevadm trigger

Tất cả hệ điều hành

Xác nhận cargo-embed

Đầu tiên, kết nối micro:bit với máy tính bằng cáp USB.

Ít nhất một đèn LED màu cam gần cổng USB của micro:bit sẽ sáng lên. Hơn nữa, nếu bạn chưa bao giờ flash chương trình nào khác lên micro:bit, chương trình mặc định đi kèm sẽ bắt đầu nhấp nháy các LED đỏ ở mặt sau — bạn có thể bỏ qua chúng.

Bây giờ hãy xem probe-rs, và qua đó cargo-embed, có nhìn thấy micro:bit của bạn không. Bạn có thể làm điều này bằng cách chạy lệnh sau:

$ probe-rs list
The following debug probes were found:
[0]: BBC micro:bit CMSIS-DAP -- 0d28:0204:990636020005282030f57fa14252d446000000006e052820 (CMSIS-DAP)

Hoặc nếu bạn muốn thêm thông tin về khả năng debug của micro:bit thì có thể chạy:

$ probe-rs info
Probing target via JTAG

 Error identifying target using protocol JTAG: The probe does not support the JTAG protocol.

Probing target via SWD

ARM Chip with debug port Default:
Debug Port: DPv1, DP Designer: ARM Ltd
├── 0 MemoryAP
│   └── ROM Table (Class 1), Designer: Nordic VLSI ASA
│       ├── Cortex-M4 SCS   (Generic IP component)
│       │   └── CPUID
│       │       ├── IMPLEMENTER: ARM Ltd
│       │       ├── VARIANT: 0
│       │       ├── PARTNO: Cortex-M4
│       │       └── REVISION: 1
│       ├── Cortex-M3 DWT   (Generic IP component)
│       ├── Cortex-M3 FBP   (Generic IP component)
│       ├── Cortex-M3 ITM   (Generic IP component)
│       ├── Cortex-M4 TPIU  (Coresight Component)
│       └── Cortex-M4 ETM   (Coresight Component)
└── 1 Unknown AP (Designer: Nordic VLSI ASA, Class: Undefined, Type: 0x0, Variant: 0x0, Revision: 0x0)

Tiếp theo, bạn sẽ cần chỉnh sửa Embed.toml trong thư mục src/03-setup của mã nguồn sách. Trong phần default.general, bạn sẽ thấy hai biến chip bị comment:

[default.general]
# chip = "nrf52833_xxAA" # uncomment this line for micro:bit V2
# chip = "nrf51822_xxAA" # uncomment this line for micro:bit V1

Nếu bạn đang làm việc với board micro:bit v2, hãy bỏ comment dòng đầu tiên. Với v1, hãy bỏ comment dòng thứ hai.

Tiếp theo, chạy một trong các lệnh sau:

$ # đảm bảo bạn đang ở thư mục src/03-setup của mã nguồn sách
$ # Nếu bạn đang dùng micro:bit v2
$ rustup target add thumbv7em-none-eabihf
$ cargo embed --target thumbv7em-none-eabihf

$ # Nếu bạn đang dùng micro:bit v1
$ rustup target add thumbv6m-none-eabi
$ cargo embed --target thumbv6m-none-eabi

Nếu mọi thứ hoạt động đúng, cargo-embed sẽ đầu tiên compile chương trình ví dụ nhỏ trong thư mục này, sau đó flash nó, và cuối cùng mở một giao diện dạng văn bản hiển thị Hello World.

(Nếu không hoạt động, hãy xem hướng dẫn khắc phục sự cố chung.)

Output này đến từ chương trình Rust nhỏ mà bạn vừa flash lên micro:bit. Mọi thứ đang hoạt động đúng và bạn có thể tiếp tục với các chương tiếp theo!
