Cài đặt môi trường phát triển

Làm việc với microcontroller đòi hỏi một bộ công cụ đặc biệt — chúng ta đang viết code cho một kiến trúc khác với máy tính của bạn, và cần chạy/debug trên một thiết bị "từ xa".

📚
Tài liệu kỹ thuật chính thức

Tài liệu kỹ thuật của MB2 có tại https://tech.microbit.org. Đây là nguồn tham khảo đáng tin cậy khi bạn gặp vấn đề về phần cứng.

Các công cụ cần thiết

Hướng dẫn cài đặt

1. Rust & Cargo

Cài rustup từ https://rustup.rs, sau đó thêm target cho ARM Cortex-M4:

bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup target add thumbv7em-none-eabihf

2. cargo-binutils

bash
rustup component add llvm-tools
cargo install cargo-binutils --vers '^0.3'

3. probe-rs-tools

Xem hướng dẫn cài đặt mới nhất tại https://probe.rs. Hoặc cài thẳng qua cargo:

bash
cargo install --locked probe-rs-tools

4. Lấy source code của sách

bash
git clone http://github.com/rust-embedded/discovery-mb2

Cài đặt theo hệ điều hành

🐧 Linux

Ubuntu 20.04+ / Debian 10+:

bash
sudo apt-get install gdb-multiarch minicom libunwind-dev

Fedora 32+:

bash
sudo dnf install gdb minicom libunwind-devel

Arch Linux:

bash
sudo pacman -S arm-none-eabi-gdb minicom libunwind

Cấu hình udev Rules (Linux)

Để dùng micro:bit mà không cần quyền root, tạo file:

bash
sudo nano /etc/udev/rules.d/69-microbit.rules

Thêm nội dung sau (vendor ID của BBC micro:bit):

text
SUBSYSTEM=="usb", ATTR{idVendor}=="0d28", ATTR{idProduct}=="0204", MODE="0666"

Sau đó reload rules:

bash
sudo udevadm control --reload
sudo udevadm trigger

🪟 Windows

arm-none-eabi-gdb: Tải file .exe installer từ Arm GNU Toolchain Downloads. Trong quá trình cài, tick vào "Add path to environment variable".

Kiểm tra cài đặt:

bash
arm-none-eabi-gcc -v

PuTTY: Tải file putty.exe từ trang chính thức và đặt vào thư mục trong %PATH%.

🍎 macOS

bash
brew install arm-none-eabi-gdb minicom

Xác nhận cài đặt thành công

Cắm micro:bit vào máy tính qua cáp USB

Đèn LED màu cam gần cổng USB sẽ sáng lên. Có thể thấy chương trình blinky mặc định chạy.

Chạy probe-rs list

Lệnh này liệt kê các debug probe đang kết nối. Nếu thành công, bạn sẽ thấy:

text
BBC micro:bit CMSIS-DAP -- [device ID] (CMSIS-DAP)

Chạy probe-rs info

Lệnh này hiển thị thông tin chi tiết về chip ARM, bao gồm Cortex-M4 processor và debug port.

Test build đầu tiên

Từ thư mục src/03-setup trong repository:

bash
rustup target add thumbv7em-none-eabihf
cargo embed --target thumbv7em-none-eabihf

Nếu thành công, cargo-embed sẽ compile, flash lên micro:bit và hiển thị giao diện terminal với output "Hello World".

Cấu hình IDE

Để có IDE support đầy đủ (rust-analyzer trong VS Code hoặc IntelliJ), cần cấu hình cross-compilation target đúng:

💡
VS Code với rust-analyzer

Thêm vào .vscode/settings.json:

json
{
  "rust-analyzer.cargo.target": "thumbv7em-none-eabihf"
}
⚠️
IntelliJ/RustRover

Cần cấu hình thêm build settings để xử lý embedded target. Xem tài liệu của plugin Rust trong IntelliJ để biết chi tiết.
