Inputs, Outputs & Polling 🕹️

Cho đến nay chúng ta chỉ dùng GPIO làm output (điều khiển LED). Chương này chuyển sang input — đọc tín hiệu từ thế giới bên ngoài như nút bấm.

Đọc trạng thái nút bấm

Micro:bit v2 có hai nút bấm vật lý:

🔘 Nút bấm → GPIO Pin
NútGPIO PinLogic khi nhấnLogic khi thả
Button AP0.14LOW (0V)HIGH (3.3V)
Button BP0.23LOW (0V)HIGH (3.3V)
🔌
Tại sao nhấn = LOW?

Các nút được kết nối với điện trở pull-up nội. Khi không nhấn, pin được kéo lên 3.3V (HIGH). Khi nhấn, pin được nối đất (GND) → 0V (LOW). Đây là thiết kế phổ biến trong embedded — gọi là active-low.

Polling — Cách kiểm tra input đơn giản nhất

Polling là kỹ thuật liên tục hỏi trạng thái input trong vòng lặp. Đơn giản về mặt khái niệm và phù hợp để bắt đầu:

rust
#![no_main]
#![no_std]

use cortex_m_rt::entry;
use embedded_hal::digital::InputPin;
use microbit::Board;
use panic_rtt_target as _;
use rtt_target::{rprintln, rtt_init_print};

#[entry]
fn main() -> ! {
    rtt_init_print!();
    let board = Board::take().unwrap();

    let mut button_a = board.buttons.button_a;
    let mut button_state = false;

    loop {
        if button_a.is_low().unwrap() {
            // Nút đang được nhấn (LOW = pressed)
            if !button_state {
                button_state = true;
                rprintln!("Button A pressed");
            }
        } else if button_state {
            // Nút vừa được thả
            button_state = false;
            rprintln!("Button A not pressed");
        }
    }
}

Chúng ta liên tục theo dõi trạng thái nút, và báo cáo mỗi khi trạng thái đó thay đổi.
