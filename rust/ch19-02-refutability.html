Refutability: Khi Pattern Có Thể Thất Bại

Pattern có hai dạng: refutable (có thể bác bỏ) và irrefutable (không thể bác bỏ). Pattern nào khớp được với mọi giá trị truyền vào thì gọi là irrefutable. Ví dụ kinh điển là x trong câu lệnh let x = 5; — vì x khớp với bất kỳ thứ gì nên nó không bao giờ thất bại. Ngược lại, pattern nào có thể không khớp với một số giá trị thì gọi là refutable. Ví dụ: Some(x) trong biểu thức if let Some(x) = a_value — nếu a_value là None thay vì Some, thì Some(x) sẽ bị "hụt chân" ngay.

Tham số hàm, câu lệnh let, và vòng lặp for chỉ chấp nhận pattern irrefutable, vì chương trình chẳng biết làm gì khi giá trị không khớp. Các biểu thức if let, while let, và câu lệnh let...else thì chấp nhận cả hai loại, nhưng compiler sẽ cảnh báo khi bạn dùng pattern irrefutable ở đó — vì vốn dĩ các cấu trúc điều kiện này được sinh ra để xử lý trường hợp thất bại: sức mạnh của chúng nằm ở khả năng "rẽ nhánh" tùy tình huống.

Nói chung, bạn không cần quá lo lắng về sự phân biệt này. Tuy nhiên, bạn cần nắm được khái niệm refutability để khi gặp nó trong thông báo lỗi, bạn biết mình phải làm gì. Khi đó, bạn cần thay đổi pattern hoặc cấu trúc đang sử dụng pattern đó, tùy thuộc vào hành vi bạn muốn.

Hãy xem điều gì xảy ra khi ta cố dùng refutable pattern ở nơi Rust đòi irrefutable, và ngược lại. Listing 19-8 dưới đây có câu lệnh let, nhưng pattern được chỉ định là Some(x) — một refutable pattern. Đương nhiên đoạn code này sẽ không compile được.

Nếu some_option_value là None, nó sẽ không khớp với Some(x) — nghĩa là pattern này là refutable. Nhưng câu lệnh let chỉ chấp nhận irrefutable pattern vì không có gì hợp lệ để làm với giá trị None. Lúc compile, Rust sẽ "mắng" ngay:

$ cargo run
   Compiling patterns v0.1.0 (file:///projects/patterns)
error[E0005]: refutable pattern in local binding
 --> src/main.rs:3:9
  |
3 |     let Some(x) = some_option_value;
  |         ^^^^^^^ pattern `None` not covered
  |
  = note: `let` bindings require an "irrefutable pattern", like a `struct` or an `enum` with only one variant
  = note: for more information, visit https://doc.rust-lang.org/book/ch19-02-refutability.html
  = note: the matched value is of type `Option<i32>`
help: you might want to use `let else` to handle the variant that isn't matched
  |
3 |     let Some(x) = some_option_value else { todo!() };
  |                                     ++++++++++++++++

For more information about this error, try `rustc --explain E0005`.
error: could not compile `patterns` (bin "patterns") due to 1 previous error

Vì ta không bao quát (và không thể bao quát!) toàn bộ giá trị hợp lệ với pattern Some(x), Rust đã chính xác khi báo lỗi compiler.

Nếu có refutable pattern ở nơi cần irrefutable, ta có thể sửa bằng cách thay let bằng let...else. Khi đó, nếu pattern không khớp, code trong cặp ngoặc nhọn sẽ xử lý phần còn lại. Listing 19-9 cho thấy cách sửa code ở Listing 19-8.

Chúng ta đã cho code một "lối thoát"! Đoạn code này hoàn toàn hợp lệ. Tuy nhiên, điều đó cũng có nghĩa là ta không thể dùng irrefutable pattern mà không nhận cảnh báo. Nếu truyền cho let...else một pattern luôn khớp như x (xem Listing 19-10), compiler sẽ phàn nàn.

Rust phàn nàn rằng dùng let...else với irrefutable pattern là vô nghĩa:

$ cargo run
   Compiling patterns v0.1.0 (file:///projects/patterns)
warning: irrefutable `let...else` pattern
 --> src/main.rs:2:5
  |
2 |     let x = 5 else {
  |     ^^^^^^^^^
  |
  = note: this pattern will always match, so the `else` clause is useless
  = help: consider removing the `else` clause
  = note: `#[warn(irrefutable_let_patterns)]` on by default

warning: `patterns` (bin "patterns") generated 1 warning
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.39s
     Running `target/debug/patterns`

Vì lý do này, các nhánh trong match phải dùng refutable pattern, ngoại trừ nhánh cuối cùng — nhánh này nên dùng irrefutable pattern để bắt mọi giá trị còn lại. Rust cho phép dùng irrefutable pattern trong match chỉ có một nhánh, nhưng cú pháp đó không thực sự hữu ích và hoàn toàn có thể thay bằng câu lệnh let đơn giản hơn.

Giờ bạn đã biết nơi dùng pattern và sự khác biệt giữa refutable và irrefutable pattern rồi. Tiếp theo, hãy cùng khám phá toàn bộ cú pháp để tạo pattern nhé!
