📦 Hello, Cargo!

Cargo là hệ thống build và trình quản lý package của Rust. Hầu hết Rustacean đều dùng công cụ này để quản lý dự án Rust vì Cargo xử lý rất nhiều nhiệm vụ cho bạn, chẳng hạn build mã nguồn, tải xuống các thư viện mà mã của bạn phụ thuộc, và build các thư viện đó. (Chúng ta gọi các thư viện mà mã nguồn của bạn cần là dependencies — phụ thuộc.)

Các chương trình Rust đơn giản nhất, giống như chương trình chúng ta vừa viết, không có dependency nào. Nếu chúng ta build dự án "Hello, world!" bằng Cargo, nó chỉ sử dụng phần Cargo phụ trách việc build mã nguồn. Khi bạn viết các chương trình Rust phức tạp hơn, bạn sẽ thêm dependencies, và nếu bạn bắt đầu dự án bằng Cargo, việc thêm dependencies sẽ dễ dàng hơn nhiều.

Vì phần lớn dự án Rust đều dùng Cargo, phần còn lại của cuốn sách này giả định bạn cũng đang dùng Cargo. Cargo được cài sẵn cùng Rust nếu bạn dùng trình cài đặt chính thức như đã thảo luận trong phần "Cài đặt". Nếu bạn cài Rust bằng cách khác, hãy kiểm tra xem Cargo đã được cài chưa bằng cách nhập lệnh sau vào terminal:

$ cargo --version

Nếu bạn thấy số phiên bản, nghĩa là bạn đã có Cargo! Nếu bạn thấy lỗi như command not found, hãy xem tài liệu cho phương pháp cài đặt của bạn để tìm cách cài Cargo riêng.

🆕 Tạo dự án với Cargo

Hãy tạo một dự án mới bằng Cargo và xem nó khác gì so với dự án "Hello, world!" ban đầu. Quay lại thư mục projects của bạn (hoặc nơi bạn quyết định lưu mã nguồn). Sau đó, trên bất kỳ hệ điều hành nào, chạy lệnh sau:

$ cargo new hello_cargo
$ cd hello_cargo

Lệnh đầu tiên tạo một thư mục và dự án mới tên hello_cargo. Chúng ta đặt tên dự án là hello_cargo, và Cargo tạo các file trong thư mục cùng tên.

Đi vào thư mục hello_cargo và liệt kê các file. Bạn sẽ thấy Cargo đã tạo cho chúng ta hai file và một thư mục: file Cargo.toml và thư mục src với file main.rs bên trong.

Nó cũng đã khởi tạo một repository Git mới cùng file .gitignore. File Git sẽ không được tạo nếu bạn chạy cargo new bên trong một repository Git đã có sẵn; bạn có thể ghi đè hành vi này bằng cách dùng cargo new --vcs=git.

Lưu ý: Git là hệ thống quản lý phiên bản phổ biến. Bạn có thể thay đổi cargo new để dùng hệ thống quản lý phiên bản khác hoặc không dùng hệ thống nào bằng cờ --vcs. Chạy cargo new --help để xem các tùy chọn có sẵn.

Mở file Cargo.toml trong trình soạn thảo bạn chọn. Nó sẽ trông tương tự như mã trong Listing 1-2.

File này ở định dạng TOML (Tom's Obvious, Minimal Language), là định dạng cấu hình của Cargo.

Dòng đầu tiên, [package], là tiêu đề phần cho biết các câu lệnh tiếp theo đang cấu hình một package. Khi chúng ta thêm nhiều thông tin hơn vào file này, chúng ta sẽ thêm các phần khác.

Ba dòng tiếp theo thiết lập thông tin cấu hình mà Cargo cần để biên dịch chương trình: tên, phiên bản, và edition của Rust để sử dụng. Chúng ta sẽ nói về khóa edition trong Phụ lục E.

Dòng cuối cùng, [dependencies], là phần bắt đầu để bạn liệt kê các dependency của dự án. Trong Rust, các package mã nguồn được gọi là crate. Chúng ta không cần crate nào khác cho dự án này, nhưng sẽ cần trong dự án đầu tiên ở Chương 2, vì vậy chúng ta sẽ dùng phần dependencies này lúc đó.

Bây giờ hãy mở src/main.rs và xem:

Tên file: src/main.rs

fn main() {
    println!("Hello, world!");
}

Cargo đã tạo sẵn chương trình "Hello, world!" cho bạn, giống hệt chương trình chúng ta viết trong Listing 1-1! Cho đến giờ, sự khác biệt giữa dự án của chúng ta và dự án Cargo tạo ra là Cargo đặt mã nguồn trong thư mục src, và chúng ta có file cấu hình Cargo.toml ở thư mục gốc.

Cargo mong đợi các file nguồn nằm trong thư mục src. Thư mục gốc của dự án chỉ dành cho file README, thông tin bản quyền, file cấu hình, và những thứ không liên quan đến mã nguồn. Dùng Cargo giúp bạn tổ chức dự án gọn gàng. Mỗi thứ đều có chỗ của nó.

Nếu bạn bắt đầu một dự án không dùng Cargo, như chúng ta đã làm với dự án "Hello, world!", bạn có thể chuyển đổi sang dự án có dùng Cargo. Di chuyển mã nguồn vào thư mục src và tạo file Cargo.toml phù hợp. Một cách dễ dàng để có file Cargo.toml đó là chạy cargo init, lệnh này sẽ tự động tạo file cho bạn.

🔨 Build và chạy dự án Cargo

Bây giờ hãy xem có gì khác khi chúng ta build và chạy chương trình "Hello, world!" bằng Cargo! Từ thư mục hello_cargo, build dự án bằng lệnh sau:

$ cargo build
   Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 2.85 secs

Lệnh này tạo file thực thi trong target/debug/hello_cargo (hoặc target\debug\hello_cargo.exe trên Windows) thay vì trong thư mục hiện tại. Vì bản build mặc định là debug build, Cargo đặt file nhị phân trong thư mục tên debug. Bạn có thể chạy file thực thi bằng lệnh này:

$ ./target/debug/hello_cargo # hoặc .\target\debug\hello_cargo.exe trên Windows
Hello, world!

Nếu mọi thứ suôn sẻ, Hello, world! sẽ in ra terminal. Chạy cargo build lần đầu cũng khiến Cargo tạo file mới ở thư mục gốc: Cargo.lock. File này theo dõi phiên bản chính xác của các dependency trong dự án. Dự án này không có dependency, nên file khá thưa. Bạn không bao giờ cần thay đổi file này thủ công; Cargo quản lý nội dung cho bạn.

Chúng ta vừa build dự án bằng cargo build và chạy bằng ./target/debug/hello_cargo, nhưng cũng có thể dùng cargo run để biên dịch mã và chạy file thực thi kết quả trong một lệnh:

$ cargo run
    Finished dev [unoptimized + debuginfo] target(s) in 0.0 secs
     Running `target/debug/hello_cargo`
Hello, world!

Dùng cargo run tiện hơn so với phải nhớ chạy cargo build rồi dùng đường dẫn đầy đủ đến file nhị phân, vì vậy hầu hết lập trình viên dùng cargo run.

Lưu ý lần này chúng ta không thấy đầu ra cho biết Cargo đang biên dịch hello_cargo. Cargo nhận ra rằng các file chưa thay đổi, nên nó không build lại mà chỉ chạy file nhị phân. Nếu bạn đã sửa mã nguồn, Cargo sẽ build lại dự án trước khi chạy, và bạn sẽ thấy đầu ra như sau:

$ cargo run
   Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.33 secs
     Running `target/debug/hello_cargo`
Hello, world!

Cargo cũng cung cấp lệnh cargo check. Lệnh này nhanh chóng kiểm tra mã để đảm bảo nó biên dịch được nhưng không tạo file thực thi:

$ cargo check
   Checking hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.32 secs

Tại sao bạn lại không muốn file thực thi? Thường thì cargo check nhanh hơn nhiều so với cargo build vì nó bỏ qua bước tạo file thực thi. Nếu bạn liên tục kiểm tra công việc trong khi viết mã, dùng cargo check sẽ tăng tốc quá trình cho bạn biết dự án vẫn biên dịch được! Do đó, nhiều Rustacean chạy cargo check định kỳ khi viết chương trình để đảm bảo nó biên dịch được. Sau đó, họ chạy cargo build khi sẵn sàng sử dụng file thực thi.

Hãy tóm tắt những gì chúng ta đã học về Cargo:

  • Chúng ta có thể tạo dự án bằng cargo new.
  • Chúng ta có thể build dự án bằng cargo build.
  • Chúng ta có thể build và chạy dự án trong một bước bằng cargo run.
  • Chúng ta có thể build dự án mà không tạo file nhị phân để kiểm tra lỗi bằng cargo check.
  • Thay vì lưu kết quả build trong cùng thư mục với mã nguồn, Cargo lưu trong thư mục target/debug.

Một ưu điểm bổ sung khi dùng Cargo là các lệnh giống nhau bất kể bạn đang dùng hệ điều hành nào. Vì vậy, từ đây trở đi, chúng ta sẽ không cung cấp hướng dẫn riêng cho Linux, macOS và Windows nữa.

🚀 Build cho Release

Khi dự án cuối cùng đã sẵn sàng cho release, bạn có thể dùng cargo build --release để biên dịch với tối ưu hóa. Lệnh này sẽ tạo file thực thi trong target/release thay vì target/debug. Các tối ưu hóa giúp mã Rust chạy nhanh hơn, nhưng bật chúng sẽ kéo dài thời gian biên dịch. Đây là lý do có hai profile khác nhau: một cho phát triển, khi bạn muốn build lại nhanh và thường xuyên, và một cho việc build chương trình cuối cùng mà bạn sẽ đưa cho người dùng — không cần build lại nhiều lần và chạy nhanh nhất có thể. Nếu bạn đang benchmarking thời gian chạy của mã, hãy chắc chắn chạy cargo build --release và benchmark với file thực thi trong target/release.

💡 Tận dụng các quy ước của Cargo

Với các dự án đơn giản, Cargo không mang lại nhiều giá trị hơn so với chỉ dùng rustc, nhưng nó sẽ chứng minh giá trị khi chương trình trở nên phức tạp hơn. Khi chương trình phát triển thành nhiều file hoặc cần dependency, việc để Cargo điều phối quá trình build sẽ dễ dàng hơn nhiều.

Dù dự án hello_cargo đơn giản, giờ nó dùng phần lớn công cụ thực sự mà bạn sẽ dùng trong suốt sự nghiệp Rust. Thực tế, để làm việc trên bất kỳ dự án nào, bạn có thể dùng các lệnh sau để checkout mã bằng Git, chuyển đến thư mục dự án, và build:

$ git clone example.org/someproject
$ cd someproject
$ cargo build

Để biết thêm thông tin về Cargo, hãy xem tài liệu của nó.

📋 Tóm tắt

Bạn đã có khởi đầu tuyệt vời trên hành trình Rust! Trong chương này, bạn đã học cách:

  • Cài đặt phiên bản stable mới nhất của Rust dùng rustup.
  • Cập nhật lên phiên bản Rust mới hơn.
  • Mở tài liệu được cài đặt cục bộ.
  • Viết và chạy chương trình "Hello, world!" dùng rustc trực tiếp.
  • Tạo và chạy dự án mới dùng các quy ước của Cargo.

Đây là thời điểm tuyệt vời để xây dựng một chương trình đáng kể hơn để làm quen với việc đọc và viết mã Rust. Vì vậy, trong Chương 2, chúng ta sẽ xây dựng chương trình trò chơi đoán số. Nếu bạn muốn bắt đầu bằng cách học các khái niệm lập trình phổ biến hoạt động như thế nào trong Rust, hãy xem Chương 3 rồi quay lại Chương 2.
