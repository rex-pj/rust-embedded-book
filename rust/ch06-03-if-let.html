Cấu trúc điều khiển ngắn gọn với if let và let...else

Cú pháp if let cho phép bạn kết hợp if và let thành một cách ít rườm rà hơn để xử lý các giá trị khớp với một mẫu nhất định trong khi bỏ qua các giá trị khác. Xem xét chương trình trong Listing 6-6, đoạn mã này matching trên một giá trị Option<u8> trong biến config_max nhưng chỉ muốn thực thi mã nếu giá trị đó là variant Some.

Nếu giá trị là Some, chúng ta in ra giá trị bên trong Some bằng cách ràng buộc giá trị đó vào biến max trong pattern. Chúng ta không muốn làm gì với giá trị None. Để thỏa mãn biểu thức match, chúng ta phải thêm _ => () sau khi xử lý chỉ một variant, đây là một đoạn mã khuôn mẫu bị thừa (boilerplate) khá là khó chịu.

Thay vào đó, chúng ta có thể viết điều này một cách ngắn gọn hơn bằng cách sử dụng if let. Mã sau hoạt động tương tự như match trong Listing 6-6:

fn main() {
    let config_max = Some(3u8);
    if let Some(max) = config_max {
        println!("The maximum is configured to be {max}");
    }
}

Cú pháp if let nhận vào một mẫu và một biểu thức được phân tách bằng dấu bằng. Nó hoạt động theo cùng một cách như match, nơi biểu thức được cung cấp cho match và mẫu là nhánh (arm) đầu tiên của nó. Trong trường hợp này, pattern là Some(max) và biến max liên kết đến giá trị bên trong Some. Sau đó, chúng ta có thể sử dụng max trong phần thân của khối if let theo cùng cách chúng ta đã sử dụng max trong arm match tương ứng. Đoạn mã trong khối if let sẽ chỉ chạy nếu giá trị khớp với mẫu.

Sử dụng if let đồng nghĩa với việc bạn gõ ít hơn, thụt lề ít hơn và ít boilerplate code hơn. Tuy nhiên, bạn mất chức năng kiểm tra tính toàn vẹn cạn kiệt (exhaustive checking) mà match bắt buộc để đảm bảo rằng bạn không quên xử lý bất kỳ trường hợp nào. Chọn giữa match và if let tùy thuộc vào những gì bạn đang làm trong tình huống cụ thể của bạn, và liệu đánh đổi tính toàn vẹn cạn kiệt để đạt được sự ngắn gọn có phù hợp hay không.

Nói cách khác, bạn có thể coi if let như một cú pháp đường mật (syntax sugar) cho match để chạy mã khi giá trị khớp với một mẫu duy nhất và bỏ qua tất cả các giá trị còn lại.

Chúng ta có thể bao gồm một else với một if let. Khối mã đi với else giống với khối mã nằm ở phần _ trong biểu thức match đối sánh tương đương với if let và else này. Nhớ lại định nghĩa Enum Coin trong Listing 6-4, nơi nhánh Quarter cũng chứa một giá trị UsState. Nếu chúng ta muốn đếm tất cả các đồng xu không phải là loại quý trong khi cũng thông báo tiểu bang của các đồng quarter này, chúng ta có thể làm điều đó với một biểu thức match, như thế này:

#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
    // --snip--
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn main() {
    let coin = Coin::Penny;
    let mut count = 0;
    match coin {
        Coin::Quarter(state) => println!("State quarter from {state:?}!"),
        _ => count += 1,
    }
}

Hoặc chúng ta có thể sử dụng một biểu thức if let kèm else như sau:

#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
    // --snip--
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn main() {
    let coin = Coin::Penny;
    let mut count = 0;
    if let Coin::Quarter(state) = coin {
        println!("State quarter from {state:?}!");
    } else {
        count += 1;
    }
}

Đi trên "Con Đường Vui Vẻ" (Happy Path) với let...else

Một mẫu phổ biến cần dùng là thực hiện một số tính toán khi một giá trị có tồn tại và trả về một giá trị mặc định nếu không. Tiếp tục với ví dụ về đồng xu của chúng ta với giá trị UsState, nếu chúng ta muốn nói một điều gì đó thú vị tùy thuộc vào tuổi của tiểu bang trên đồng quarter này, chúng ta có thể tạo ra một phương thức trên UsState để kiểm tra độ tuổi của một tiểu bang, như thế này:

#[derive(Debug)] // so we can inspect the state in a minute
enum UsState {
    Alabama,
    Alaska,
    // --snip--
}

impl UsState {
    fn existed_in(&self, year: u16) -> bool {
        match self {
            UsState::Alabama => year >= 1819,
            UsState::Alaska => year >= 1959,
            // -- snip --
        }
    }
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn describe_state_quarter(coin: Coin) -> Option<String> {
    if let Coin::Quarter(state) = coin {
        if state.existed_in(1900) {
            Some(format!("{state:?} is pretty old, for America!"))
        } else {
            Some(format!("{state:?} is relatively new."))
        }
    } else {
        None
    }
}

fn main() {
    if let Some(desc) = describe_state_quarter(Coin::Quarter(UsState::Alaska)) {
        println!("{desc}");
    }
}

Sau đó, chúng ta có thể sẽ dùng if let để match với loại đồng xu, qua đó đưa một biến state vào phần thân của điều kiện giống như trong Listing 6-7.

Nó giúp giải quyết được vấn đề thực tế, nhưng nó lại đẩy hết công việc vào phần thân (body) của lệnh if let, và nếu công việc cần làm phức tạp hơn ở mức độ nào đó, có thể khó theo dõi chính xác cách mà các khối trên cùng trong nhánh này liên hệ ra làm sao. Chúng ta cũng có thể tận dụng thực tế là các biểu thức tạo ra một giá trị hoặc để trích rút được state từ if let hoặc để return sớm giống như trong Listing 6-8. (Bạn cũng có thể làm một điều gì đó tương tự với câu lệnh match.)

Điều này tuy thế mà lại tự có phần khó để làm theo! Cụ thể thì một nhánh của lệnh if let lại tạo ra và đưa về một giá trị, và một nhánh còn lại tiến hành trả về hoàn toàn (return entirely) từ hàm đó.

Và để làm cho khuôn mẫu chung này trở nên tường minh và hiệu quả hơn, ngôn ngữ Rust cung cấp let...else. Cú pháp let...else lấy về một mẫu (pattern) nằm ở phía bên trái và một biểu thức (expression) ở bên phải, tương đối giống với if let, nhưng nó có điểm khác là không có một rẽ nhánh if nào cả mà lại chỉ có một rẽ nhánh else mà thôi. Trường hợp pattern mà ăn khớp, nó sẽ tiến hành ràng buộc phần giá trị đối chiếu được từ chính pattern này với vùng không gian bên ngoài (outer scope). Ngược lại, nếu pattern đó mà không khớp, thì khi đó chương trình sẽ đi ngược vào bên trong nhánh else, bắt buộc và sẽ phải thoát khỏi hàm thông qua từ khóa return.

Và thông qua Listing 6-9, thì bạn sẽ thấy là qua cách đoạn code ở Listing 6-8 được trình bày lúc mà áp dụng vào sử dụng let...else thay cho if let thì như thế nào.

Làm vậy, ta sẽ bảo lưu phần mã nằm trong chức năng chính trên "con đường hạnh phúc" một cách chuẩn nhất, sẽ chẳng hề có phần luồng chạy mã khác đáng kể qua cả hai nhánh phụ như khi ta phải dùng if let cho việc này.

Khi bạn mà thấy rơi vào tình trạng mà mã ứng dụng đang dùng cái cấu trúc logic quá ư phức tạp để ghi ra bằng lệnh match, thì hãy chớ quên bạn luôn có if let cùng lệnh let...else đóng vai trò dự phòng và sẵn sàng cho sử dụng ngay trong hộp đồ công nghệ Rust.

Tổng kết

Chúng ta đã trình bày xong tất cả việc đi dùng cấu trúc Enum nhắm tới một mục đích cuối: kiến tạo các data structure theo chuẩn và cho phép việc đó có được một trong số dữ kiện tập hợp định sẵn. Cập nhật này đi liền với sự hé lộ của chính cấu trúc Option<T> tại thư viện cốt lõi: đây được xem là cấu trúc giúp củng cố dữ kiện dùng, đồng thời tránh việc đưa mã tạo ứng dụng vướng bug ngoài tầm với. Những khi biến số liên quan trong cấu trúc enum đi mang lưu lượng tại chính nó, thì sử dụng match kèm theo cái cấu trúc đi rẽ hướng if let hoàn toàn đủ vững trãi giúp tiến hành gắp và dùng thử phần trị mang về đó, đây lại sẽ phụ thuộc nhiều và một loạt nhiều hay ít điều kiện bạn tính xử.

Rust Program tới nay cho thấy bạn đã biết cách áp mã theo kiểu chuẩn cùng Enum như Struct. Việc bạn đi vào tạo hẳn type riêng hướng đến cho API khi này chỉ nhắm vào 1 sự thật rằng nó củng cố đi hệ thống, Compiler có nhiệm vụ nhắc nhở việc bạn đưa chức năng cần và đủ để những thứ đang có đều luôn mang dữ kiện thuộc kiểu này trong nó.

Mục tiêu để có đủ sức hướng đến chia module hệ code khi tới tay bạn khi bạn là những Programmers sử dụng, giúp sử dụng đi không rườm rà. Hệ module sẽ nói tiếp sau.
