🚦 Control Flow (Luồng điều khiển)

Khả năng chạy một mã cụ thể khi một điều kiện cho trước là true (đúng) và khả năng lặp đi lặp lại một phân đoạn mã trong khi một điều kiện cho trước vẫn đang là true, là một phần quan trọng của các khối lệnh mã cơ bản trong hầu hết các ngôn ngữ lập trình. Các cấu trúc phổ biến nhất dùng để điều khiển luồng thực thi trong ngôn ngữ Rust là biểu thức if và các vòng lặp.

🤔 Biểu thức if

Biểu thức if cho phép bạn phân nhánh mã tùy thuộc vào các điều kiện. Bạn đưa ra một điều kiện và chỉ thị rằng, “Nếu điều kiện này được đáp ứng, thì chạy khối mã này. Nếu điều kiện không được thỏa mãn, đừng chạy khối mã này.”

Hãy tạo một dự án mới tên là branches ở trong thư mục projects để khám phá biểu thức if này. Trong thư mục src/main.rs, gõ vào đoạn sau:

Tên file: src/main.rs

fn main() {
    let number = 3;

    if number < 5 {
        println!("condition was true");
    } else {
        println!("condition was false");
    }
}

Tất cả các biểu thức if đều bắt đầu bằng từ khóa if, theo sau đó là một điều kiện. Trong trường hợp này, điều kiện để kiểm tra là liệu giá trị của tham số number có mang giá trị nhỏ hơn 5 hay không. Chúng ta viết các đoạn mã mà khi có câu điều kiện là true sẽ thực thi ngay sau ngoặc ngọn của điều kiện đó. Các khối mã gắn với các điều kiện trong các câu lệnh của if đôi khi được gọi chung là “nhánh” (arms), hoàn toàn giống với khái niệm arm của biểu thức lệnh match mà chúng ta vừa nhắc ở Phần "So sánh số dự đoán với con số bí mật" tại Chương 2.

Chúng ta cũng có thể thêm mệnh đề else để cung cấp cho chương trình một khối lệnh thay thế khi điều kiện không được thỏa mãn (tức là khi điều kiện là false). Nếu bạn không cung cấp mệnh đề else, chương trình sẽ bỏ qua khối if và tiếp tục thực thi phần tiếp theo.

Hãy xem kết quả xuất ra. Sau đó thử với code sau:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/branches`
condition was true

Hãy thử thay đổi giá trị của number thành một giá trị làm cho điều kiện trở thành false để xem điều gì xảy ra:

    let number = 7;

Chạy lại chương trình để xem kết quả:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/branches`
condition was false

Điều rất đáng lưu ý là tất cả mọi biểu thức trong đoạn code này bắt buộc phải thuộc dạng Boolean. Khi biểu thức có những nội dụng không cấu thành dạng đó, thì một báo lỗi như dưới đây sẽ làm gián đoạn mọi việc khi bạn đang chạy nó:

Tên file: src/main.rs

fn main() {
    let number = 3;

    if number {
        println!("number was three");
    }
}

Lần này Rust sẽ báo lỗi vì điều kiện không phải kiểu bool. Rust sẽ hiển thị lỗi sau:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
error[E0308]: mismatched types
 --> src/main.rs:4:8
  |
4 |     if number {
  |        ^^^^^^ expected `bool`, found integer

For more information about this error, try `rustc --explain E0308`.
error: could not compile `branches` (bin "branches") due to 1 previous error

Thông báo lỗi cho biết Rust mong đợi một bool nhưng nhận được một số nguyên. Khác với các ngôn ngữ như Ruby và JavaScript, Rust không tự động chuyển đổi các kiểu dữ liệu không phải Boolean sang Boolean. Bạn phải tường minh và luôn cung cấp cho if một giá trị Boolean làm điều kiện. Nếu chúng ta muốn khối lệnh if chỉ chạy khi một số khác 0, chẳng hạn, ta có thể thay đổi biểu thức if thành như sau:

Tên file: src/main.rs

fn main() {
    let number = 3;

    if number != 0 {
        println!("number was something other than zero");
    }
}

Chạy code này sẽ in ra: number was something other than zero.

Nhiều Điều kiện với else if

Bạn có thể sử dụng nhiều điều kiện bằng cách kết hợp if và else trong một biểu thức else if. Ví dụ:

Tên file: src/main.rs

fn main() {
    let number = 6;

    if number % 4 == 0 {
        println!("number is divisible by 4");
    } else if number % 3 == 0 {
        println!("number is divisible by 3");
    } else if number % 2 == 0 {
        println!("number is divisible by 2");
    } else {
        println!("number is not divisible by 4, 3, or 2");
    }
}

Chương trình này có bốn nhánh thực thi có thể. Sau khi chạy, bạn sẽ thấy kết quả đầu ra sau:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/branches`
number is divisible by 3

Khi chương trình thực thi, nó kiểm tra từng biểu thức if lần lượt và thực thi khối lệnh đầu tiên mà điều kiện cho ra giá trị true. Lưu ý rằng mặc dù 6 chia hết cho 2, chúng ta không thấy kết quả number is divisible by 2, cũng không thấy dòng number is not divisible by 4, 3, or 2 từ khối else. Đó là vì Rust chỉ thực thi khối lệnh cho điều kiện true đầu tiên tìm thấy, và sau khi tìm thấy rồi thì không kiểm tra những điều kiện còn lại nữa.

Sử dụng quá nhiều biểu thức else if có thể làm cho code trở nên lộn xộn, vì vậy nếu bạn có nhiều hơn một điều kiện, bạn có thể muốn tái cấu trúc code của mình. Chương 6 mô tả một cấu trúc rẽ nhánh mạnh mẽ của Rust gọi là match dành cho những trường hợp như vậy.

Dùng if trong Câu Lệnh let

Vì if là một biểu thức, chúng ta có thể dùng nó ở vế phải của câu lệnh let để gán kết quả vào một biến, như trong Listing 3-2.

Biến number sẽ được gán một giá trị dựa trên kết quả của biểu thức if. Hãy chạy đoạn code này để xem điều gì xảy ra:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.30s
     Running `target/debug/branches`
The value of number is: 5

Hãy nhớ rằng các khối lệnh trả về giá trị của biểu thức cuối cùng trong chúng, và bản thân các con số cũng là biểu thức. Trong trường hợp này, giá trị của toàn bộ biểu thức if phụ thuộc vào khối lệnh nào được thực thi. Điều này có nghĩa là các giá trị có thể là kết quả từ mỗi nhánh của if phải có cùng kiểu dữ liệu; trong Listing 3-2, kết quả của cả nhánh if và nhánh else đều là số nguyên i32. Nếu các kiểu không khớp nhau, như trong ví dụ sau, chúng ta sẽ gặp lỗi:

Tên file: src/main.rs

fn main() {
    let condition = true;

    let number = if condition { 5 } else { "six" };

    println!("The value of number is: {number}");
}

Khi chúng ta cố biên dịch đoạn code này, chúng ta sẽ gặp lỗi. Nhánh if và nhánh else có kiểu giá trị không tương thích, và Rust chỉ ra chính xác vị trí của vấn đề trong chương trình:

$ cargo run
   Compiling branches v0.1.0 (file:///projects/branches)
error[E0308]: `if` and `else` have incompatible types
 --> src/main.rs:4:44
  |
4 |     let number = if condition { 5 } else { "six" };
  |                                 -          ^^^^^ expected integer, found `&str`
  |                                 |
  |                                 expected because of this

For more information about this error, try `rustc --explain E0308`.
error: could not compile `branches` (bin "branches") due to 1 previous error

Biểu thức trong khối if trả về một số nguyên, và biểu thức trong khối else trả về một chuỗi. Điều này không hoạt động được vì các biến phải có một kiểu duy nhất, và Rust cần biết chắc chắn tại thời điểm biên dịch biến number có kiểu gì. Biết được kiểu của number cho phép trình biên dịch xác minh kiểu đó hợp lệ ở mọi nơi chúng ta sử dụng number. Rust sẽ không thể làm điều đó nếu kiểu của number chỉ được xác định tại thời điểm chạy; trình biên dịch sẽ phức tạp hơn và đưa ra ít đảm bảo hơn về code nếu nó phải theo dõi nhiều kiểu giả định cho bất kỳ biến nào.

🔁 Các vòng lặp

Thường có lúc sẽ hữu ích khi thực thi một khối lệnh nhiều hơn một lần. Để làm điều này, Rust cung cấp một số loại vòng lặp (loops), chúng sẽ chạy qua các đoạn mã trong thân vòng lặp đến cuối rồi bắt đầu lại ngay từ đầu. Để thử nghiệm với vòng lặp, hãy tạo một dự án mới có tên là loops.

Rust có ba loại vòng lặp: loop, while, và for. Hãy cùng thử từng loại.

Lặp dòng mã bằng loop

Từ khóa loop yêu cầu Rust thực thi một khối lệnh lặp đi lặp lại mãi mãi hoặc cho đến khi bạn ra lệnh dừng một cách tường minh.

Ví dụ, hãy thay đổi file src/main.rs trong thư mục loops của bạn để trông giống như sau:

Tên file: src/main.rs

fn main() {
    loop {
        println!("again!");
    }
}

Khi chúng ta chạy chương trình này, chúng ta sẽ thấy chữ again! được in ra liên tục cho đến khi chúng ta dừng chương trình thủ công. Hầu hết các terminal đều hỗ trợ phím tắt ctrl-C để ngắt một chương trình đang bị kẹt trong vòng lặp vô hạn. Hãy thử xem:

$ cargo run
   Compiling loops v0.1.0 (file:///projects/loops)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.08s
     Running `target/debug/loops`
again!
again!
again!
again!
^Cagain!

Ký hiệu ^C đại diện cho vị trí bạn đã nhấn ctrl-C.

Bạn có thể thấy hoặc không thấy chữ again! được in ra sau ^C, tùy thuộc vào vị trí code đang thực thi trong vòng lặp khi nhận được tín hiệu ngắt.

May mắn thay, Rust cũng cung cấp một cách để thoát ra khỏi vòng lặp bằng code. Bạn có thể đặt từ khóa break trong vòng lặp để báo cho chương trình biết khi nào cần ngừng thực thi vòng lặp. Hãy nhớ rằng chúng ta đã làm điều này trong trò chơi đoán số ở phần ”Thoát ra sau khi đoán đúng” tại Chương 2 để thoát khỏi chương trình khi người dùng thắng trò chơi bằng cách đoán đúng số.

Chúng ta cũng đã dùng continue trong trò chơi đoán số, từ khóa này trong một vòng lặp sẽ báo cho chương trình bỏ qua phần code còn lại trong lần lặp hiện tại và chuyển sang lần lặp tiếp theo.

Trả về Giá trị từ Vòng Lặp

Một trong những cách dùng của loop là thử lại một thao tác mà bạn biết có thể thất bại, chẳng hạn như kiểm tra xem một thread đã hoàn thành công việc chưa. Bạn cũng có thể cần truyền kết quả của thao tác đó ra ngoài vòng lặp để dùng ở phần còn lại của code. Để làm điều này, bạn có thể thêm giá trị cần trả về sau biểu thức break dùng để dừng vòng lặp; giá trị đó sẽ được trả ra ngoài vòng lặp để bạn có thể sử dụng, như ví dụ dưới đây:

fn main() {
    let mut counter = 0;

    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2;
        }
    };

    println!("The result is {result}");
}

Trước vòng lặp, chúng ta khai báo một biến tên là counter và khởi tạo nó bằng 0. Sau đó, chúng ta khai báo một biến tên là result để lưu giá trị trả về từ vòng lặp. Mỗi lần lặp, chúng ta cộng thêm 1 vào biến counter, rồi kiểm tra xem counter có bằng 10 không. Khi bằng, chúng ta dùng từ khóa break kèm theo giá trị counter * 2. Sau vòng lặp, chúng ta dùng dấu chấm phẩy để kết thúc câu lệnh gán giá trị cho result. Cuối cùng, chúng ta in ra giá trị trong result, trong trường hợp này là 20.

Bạn cũng có thể dùng return từ bên trong vòng lặp. Trong khi break chỉ thoát khỏi vòng lặp hiện tại, thì return luôn luôn thoát khỏi hàm hiện tại.

Phân biệt các vòng lặp lồng nhau bằng nhãn vòng lặp

Nếu bạn có các vòng lặp lồng nhau, break và continue sẽ áp dụng cho vòng lặp trong cùng tại điểm đó. Bạn có thể tùy chọn chỉ định một nhãn vòng lặp (loop label) trên một vòng lặp mà sau đó bạn có thể dùng với break hoặc continue để chỉ định rằng các từ khóa đó áp dụng cho vòng lặp được gắn nhãn thay vì vòng lặp trong cùng. Nhãn vòng lặp phải bắt đầu bằng một dấu nháy đơn. Đây là ví dụ với hai vòng lặp lồng nhau:

fn main() {
    let mut count = 0;
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;

        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                break;
            }
            if count == 2 {
                break 'counting_up;
            }
            remaining -= 1;
        }

        count += 1;
    }
    println!("End count = {count}");
}

Vòng lặp ngoài có nhãn 'counting_up, và nó sẽ đếm từ 0 đến 2. Vòng lặp bên trong không có nhãn sẽ đếm ngược từ 10 xuống 9. Câu lệnh break đầu tiên không chỉ định nhãn sẽ chỉ thoát khỏi vòng lặp bên trong. Câu lệnh break 'counting_up; sẽ thoát khỏi vòng lặp ngoài. Đoạn code này in ra:

$ cargo run
   Compiling loops v0.1.0 (file:///projects/loops)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.58s
     Running `target/debug/loops`
count = 0
remaining = 10
remaining = 9
count = 1
remaining = 10
remaining = 9
count = 2
remaining = 10
End count = 2

Đơn giản hóa vòng lặp có điều kiện với while

Một chương trình thường cần kiểm tra một điều kiện trong vòng lặp. Khi điều kiện là true, vòng lặp chạy. Khi điều kiện không còn là true, chương trình gọi break, dừng vòng lặp. Có thể triển khai hành vi như thế này bằng cách kết hợp loop, if, else, và break; bạn có thể thử ngay trong một chương trình nếu muốn. Tuy nhiên, mô hình này phổ biến đến nỗi Rust có một cấu trúc ngôn ngữ tích hợp cho nó, gọi là vòng lặp while. Trong Listing 3-3, chúng ta dùng while để lặp chương trình ba lần, đếm ngược mỗi lần, rồi sau vòng lặp in một thông báo và kết thúc.

Cấu trúc này loại bỏ rất nhiều cấu trúc lồng nhau sẽ cần thiết nếu bạn dùng loop, if, else, và break, và nó rõ ràng hơn. Khi điều kiện là true, code chạy; ngược lại, nó thoát khỏi vòng lặp.

Lặp qua một Collection với for

Bạn có thể chọn dùng cấu trúc while để lặp qua các phần tử của một collection, chẳng hạn như một mảng. Ví dụ, vòng lặp trong Listing 3-4 in ra từng phần tử trong mảng a.

Ở đây, code đếm qua các phần tử trong mảng. Nó bắt đầu từ chỉ số 0, rồi lặp cho đến khi đạt đến chỉ số cuối cùng trong mảng (tức là khi index < 5 không còn là true). Chạy code này sẽ in ra mọi phần tử trong mảng:

$ cargo run

    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.32s
     Running `target/debug/loops`
the value is: 10
the value is: 20
the value is: 30
the value is: 40
the value is: 50

Tất cả năm giá trị của mảng xuất hiện trên terminal, như mong đợi. Mặc dù index sẽ đạt đến giá trị 5 tại một thời điểm nào đó, nhưng vòng lặp dừng thực thi trước khi cố lấy giá trị thứ sáu từ mảng.

Tuy nhiên, cách tiếp cận này dễ gây lỗi; chúng ta có thể khiến chương trình panic nếu giá trị chỉ số hoặc điều kiện kiểm tra không đúng. Ví dụ, nếu bạn thay đổi định nghĩa mảng a để có bốn phần tử nhưng quên cập nhật điều kiện thành while index < 4, code sẽ panic. Ngoài ra, cách này còn chậm vì trình biên dịch thêm code runtime để kiểm tra điều kiện xem chỉ số có nằm trong giới hạn của mảng hay không tại mỗi lần lặp.

Là một lựa chọn ngắn gọn hơn, bạn có thể dùng vòng lặp for và thực thi một đoạn code cho từng phần tử trong một collection. Vòng lặp for trông như code trong Listing 3-5.

Khi chúng ta chạy code này, chúng ta sẽ thấy kết quả đầu ra giống như trong Listing 3-4. Quan trọng hơn, chúng ta đã tăng tính an toàn của code và loại bỏ khả năng xảy ra lỗi do vượt qua cuối mảng hoặc không đi đủ xa và bỏ sót một số phần tử. Code máy được tạo ra từ vòng lặp for cũng có thể hiệu quả hơn vì chỉ số không cần được so sánh với độ dài của mảng tại mỗi lần lặp.

Khi dùng vòng lặp for, bạn không cần phải nhớ thay đổi bất kỳ code nào khác nếu bạn thay đổi số lượng giá trị trong mảng, như bạn sẽ phải làm với phương pháp dùng trong Listing 3-4.

Tính an toàn và ngắn gọn của vòng lặp for khiến chúng trở thành cấu trúc vòng lặp được dùng phổ biến nhất trong Rust. Ngay cả trong các tình huống bạn muốn chạy một đoạn code một số lần nhất định, như trong ví dụ đếm ngược dùng vòng lặp while ở Listing 3-3, hầu hết các Rustacean sẽ dùng vòng lặp for. Cách để làm điều đó là dùng Range, được cung cấp bởi thư viện chuẩn, nó tạo ra tất cả các số theo thứ tự bắt đầu từ một số và kết thúc trước một số khác.

Đây là cách đếm ngược trông như thế nào khi dùng vòng lặp for và một phương thức chúng ta chưa đề cập, rev, để đảo ngược dãy số:

Tên file: src/main.rs

fn main() {
    for number in (1..4).rev() {
        println!("{number}!");
    }
    println!("LIFTOFF!!!");
}
