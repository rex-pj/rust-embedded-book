Tùy chỉnh Build với Release Profiles

Trong Rust, release profiles là những profile được định nghĩa sẵn và có thể tùy chỉnh — mỗi profile mang cấu hình riêng biệt, giúp lập trình viên kiểm soát nhiều tùy chọn biên dịch khác nhau. Mỗi profile hoạt động độc lập, không ảnh hưởng lẫn nhau.

Cargo có hai profile chính: profile dev được dùng khi bạn chạy cargo build, và profile release được dùng khi bạn chạy cargo build --release. Profile dev được thiết lập với những giá trị mặc định phù hợp cho quá trình phát triển, còn profile release có mặc định tối ưu hóa cho bản phát hành chính thức.

Tên hai profile này chắc hẳn bạn đã thấy trong output khi build:

$ cargo build
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.00s
$ cargo build --release
    Finished `release` profile [optimized] target(s) in 0.32s

dev và release là hai profile khác nhau mà compiler sử dụng.

Cargo có cài đặt mặc định cho mỗi profile — những mặc định này sẽ được áp dụng khi bạn chưa thêm bất kỳ section [profile.*] nào vào file Cargo.toml của dự án. Bằng cách thêm section [profile.*] cho profile mà bạn muốn tùy chỉnh, bạn có thể ghi đè lên bất kỳ tập hợp con nào của cài đặt mặc định. Ví dụ, đây là giá trị mặc định của setting opt-level cho profile dev và release:

Tên file: Cargo.toml

[profile.dev]
opt-level = 0

[profile.release]
opt-level = 3

Setting opt-level kiểm soát số lượng tối ưu hóa mà Rust áp dụng lên code, với thang đo từ 0 đến 3. Càng tối ưu hóa nhiều thì thời gian biên dịch càng dài, vì vậy khi bạn đang trong giai đoạn phát triển và biên dịch code liên tục, bạn sẽ muốn ít tối ưu hóa hơn để build nhanh hơn dù code chạy chậm hơn một chút. Đó là lý do opt-level mặc định cho dev là 0. Còn khi sắp phát hành, tốt hơn hết là bỏ thêm thời gian biên dịch. Bạn chỉ biên dịch ở chế độ release một lần, nhưng chương trình sẽ được chạy nhiều lần — nên release mode đánh đổi thời gian biên dịch lâu hơn để có code chạy nhanh hơn. Đó là lý do opt-level mặc định cho profile release là 3.

Bạn có thể ghi đè cài đặt mặc định bằng cách thêm giá trị khác trong Cargo.toml. Ví dụ, nếu muốn dùng mức tối ưu hóa 1 trong profile phát triển, ta có thể thêm hai dòng này vào Cargo.toml của dự án:

Tên file: Cargo.toml

[profile.dev]
opt-level = 1

Đoạn code này ghi đè lên giá trị mặc định 0. Giờ khi chạy cargo build, Cargo sẽ dùng các mặc định của profile dev cộng với tùy chỉnh opt-level của chúng ta. Vì ta đặt opt-level thành 1, Cargo sẽ áp dụng nhiều tối ưu hóa hơn mặc định, nhưng vẫn ít hơn so với bản release build.

Để xem danh sách đầy đủ các tùy chọn cấu hình và giá trị mặc định cho từng profile, hãy xem tài liệu của Cargo.
