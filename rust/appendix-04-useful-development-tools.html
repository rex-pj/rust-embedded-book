Phụ lục D: Các Công Cụ Phát Triển Hữu Ích

Trong phụ lục này, chúng ta sẽ nói về một số công cụ phát triển hữu ích mà dự án Rust cung cấp. Ta sẽ xem qua tính năng định dạng code tự động, cách nhanh chóng áp dụng các gợi ý sửa lỗi từ compiler, một công cụ lint chuyên nghiệp, và cách tích hợp với các IDE.

Tự Động Định Dạng Code với rustfmt

Công cụ rustfmt định dạng lại code của bạn theo phong cách cộng đồng đã thống nhất. Nhiều dự án cộng tác sử dụng rustfmt để tránh những cuộc tranh cãi vô tận về "nên dùng style nào khi viết Rust" — mọi người cứ chạy tool là xong, chẳng ai phải cãi nhau!

Cài đặt Rust đã bao gồm sẵn rustfmt, vì vậy bạn đã có hai chương trình rustfmt và cargo-fmt trên máy rồi. Hai lệnh này tương tự như rustc và cargo vậy — rustfmt cho phép kiểm soát chi tiết hơn, còn cargo-fmt hiểu rõ các quy ước của project dùng Cargo. Để định dạng bất kỳ Cargo project nào, gõ lệnh sau:

$ cargo fmt

Chạy lệnh này sẽ định dạng lại toàn bộ code Rust trong crate hiện tại. Điều này chỉ thay đổi phong cách code, không thay đổi ngữ nghĩa của code. Để biết thêm thông tin về rustfmt, xem tài liệu của nó.

Sửa Code Tự Động với rustfix

Công cụ rustfix được đóng gói sẵn cùng Rust và có thể tự động sửa các cảnh báo từ compiler khi cách sửa rõ ràng và đúng với ý bạn muốn. Chắc bạn đã thấy compiler cảnh báo nhiều lần rồi. Ví dụ, xem đoạn code này:

Tên file: src/main.rs

fn main() {
    let mut x = 42;
    println!("{x}");
}

Ở đây, ta khai báo biến x là mutable, nhưng thực ra chẳng bao giờ thay đổi giá trị của nó cả. Rust sẽ cảnh báo về điều đó:

$ cargo build
   Compiling myprogram v0.1.0 (file:///projects/myprogram)
warning: variable does not need to be mutable
 --> src/main.rs:2:9
  |
2 |     let mut x = 0;
  |         ----^
  |         |
  |         help: remove this `mut`
  |
  = note: `#[warn(unused_mut)]` on by default

Cảnh báo gợi ý rằng ta nên bỏ từ khóa mut đi. Ta có thể tự động áp dụng gợi ý đó bằng cách dùng công cụ rustfix qua lệnh cargo fix:

$ cargo fix
    Checking myprogram v0.1.0 (file:///projects/myprogram)
      Fixing src/main.rs (1 fix)
    Finished dev [unoptimized + debuginfo] target(s) in 0.59s

Khi nhìn lại src/main.rs, ta sẽ thấy cargo fix đã thay đổi code:

Tên file: src/main.rs

fn main() {
    let x = 42;
    println!("{x}");
}

Biến x giờ đã là immutable, và cảnh báo không còn xuất hiện nữa.

Bạn cũng có thể dùng lệnh cargo fix để chuyển đổi code của mình giữa các Rust edition khác nhau. Các edition được đề cập trong Phụ lục E.

Thêm Lint với Clippy

Clippy là một bộ sưu tập các lint để phân tích code, giúp bạn phát hiện những lỗi phổ biến và cải thiện chất lượng code Rust. Clippy được đóng gói sẵn trong bản cài đặt Rust tiêu chuẩn — cứ cài Rust là có Clippy luôn!

Để chạy các lint của Clippy trên bất kỳ Cargo project nào, gõ:

$ cargo clippy

Ví dụ, giả sử bạn viết một chương trình sử dụng giá trị xấp xỉ của một hằng số toán học, chẳng hạn như pi, như chương trình này:

Chạy cargo clippy trên project này sẽ ra lỗi này:

error: approximate value of `f{32, 64}::consts::PI` found
 --> src/main.rs:2:13
  |
2 |     let x = 3.1415;
  |             ^^^^^^
  |
  = note: `#[deny(clippy::approx_constant)]` on by default
  = help: consider using the constant directly
  = help: for further information visit https://rust-lang.github.io/rust-clippy/master/index.html#approx_constant

Lỗi này cho bạn biết rằng Rust đã có sẵn hằng số PI chính xác hơn, và chương trình của bạn sẽ đúng hơn nếu dùng hằng số đó thay vì gõ tay con số xấp xỉ. Bạn sẽ sửa code để dùng hằng số PI.

Tích Hợp IDE với rust-analyzer

Để hỗ trợ tích hợp IDE, cộng đồng Rust khuyến nghị dùng rust-analyzer. Công cụ này là một tập hợp các tiện ích lấy compiler làm trung tâm, giao tiếp qua giao thức Language Server Protocol — một đặc tả cho phép IDE và ngôn ngữ lập trình "nói chuyện" với nhau. Nhiều client có thể dùng rust-analyzer, chẳng hạn như plugin Rust analyzer cho Visual Studio Code.

Truy cập trang chủ của project rust-analyzer để xem hướng dẫn cài đặt, rồi cài hỗ trợ language server vào IDE của bạn. IDE của bạn sẽ "thông minh" hẳn lên với các tính năng như tự động hoàn thành code, nhảy tới định nghĩa, và hiển thị lỗi ngay tại chỗ.
