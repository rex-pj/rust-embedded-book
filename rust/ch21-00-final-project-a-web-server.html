Dự án cuối: Xây dựng Web Server Đa luồng

Đây là một hành trình dài, nhưng chúng ta đã đến cuối cuốn sách. Trong chương này, chúng ta sẽ cùng xây dựng thêm một dự án nữa để minh họa một số khái niệm đã học trong các chương cuối, đồng thời ôn lại những bài học trước đó.

Cho dự án cuối, chúng ta sẽ tạo một web server hiển thị "Hello!" trông như Hình 21-1 trong trình duyệt web.

Đây là kế hoạch xây dựng web server:

  1. Tìm hiểu một chút về TCP và HTTP.
  2. Lắng nghe kết nối TCP trên một socket.
  3. Phân tích một số HTTP request.
  4. Tạo HTTP response đúng định dạng.
  5. Cải thiện thông lượng của server với thread pool.

Hình 21-1: Dự án cuối của chúng ta

Trước khi bắt đầu, cần lưu ý hai điểm. Thứ nhất, phương pháp chúng ta sẽ dùng không phải là cách tốt nhất để xây dựng web server với Rust. Các thành viên cộng đồng đã xuất bản nhiều crate sẵn sàng dùng trong production trên crates.io với web server và thread pool hoàn chỉnh hơn những gì chúng ta sẽ xây dựng. Tuy nhiên, mục đích của chương này là giúp bạn học, không phải đi đường tắt. Vì Rust là ngôn ngữ lập trình hệ thống, chúng ta có thể chọn mức độ trừu tượng muốn làm việc và có thể đi xuống mức thấp hơn so với những gì có thể hoặc thực tế trong các ngôn ngữ khác.

Thứ hai, chúng ta sẽ không dùng async và await ở đây. Việc xây dựng thread pool đã đủ thách thức, chưa cần thêm việc xây dựng async runtime! Tuy nhiên, chúng ta sẽ chỉ ra khi nào async và await có thể áp dụng cho một số vấn đề tương tự sẽ gặp trong chương này. Cuối cùng, như đã đề cập trong Chương 17, nhiều async runtime dùng thread pool để quản lý công việc.

Do đó, chúng ta sẽ tự viết HTTP server và thread pool cơ bản để bạn có thể học các ý tưởng và kỹ thuật đằng sau các crate có thể dùng trong tương lai.

Sơ đồ kiến trúc

Luồng xử lý Request:
Client → TCP Connection → Server lắng nghe → Đọc HTTP request → Tạo HTTP response → Gửi lại Client

Chương này đi từ server đơn luồng (single-threaded) đến server đa luồng (multi-threaded) với thread pool.
