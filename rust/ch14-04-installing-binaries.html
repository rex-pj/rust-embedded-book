Cài đặt Binary với cargo install

Lệnh cargo install cho phép bạn cài đặt và dùng binary crate ngay trên máy của mình. Đây không phải để thay thế package manager của hệ thống; mục đích của nó là mang lại sự tiện lợi cho các lập trình viên Rust khi muốn cài đặt các công cụ mà người khác đã chia sẻ trên crates.io. Lưu ý rằng bạn chỉ có thể cài đặt các package có binary target. Một binary target là chương trình có thể chạy được, được tạo ra khi crate có file src/main.rs hoặc một file khác được chỉ định là binary, khác với library target — thứ không thể chạy độc lập mà chỉ phù hợp để nhúng vào các chương trình khác. Thông thường, các crate có thông tin trong file README cho biết crate đó là library, có binary target, hay cả hai.

Tất cả binary được cài đặt bằng cargo install đều được lưu vào thư mục bin của installation root. Nếu bạn cài Rust bằng rustup.rs mà không có cấu hình tùy chỉnh nào, thư mục này sẽ là $HOME/.cargo/bin. Hãy đảm bảo thư mục này nằm trong $PATH của bạn để có thể chạy các chương trình đã cài bằng cargo install.

Ví dụ, ở Chương 12 chúng ta đã đề cập đến một công cụ Rust thay thế grep tên là ripgrep dùng để tìm kiếm trong file. Để cài ripgrep, chúng ta có thể chạy lệnh sau:

$ cargo install ripgrep
    Updating crates.io index
  Downloaded ripgrep v14.1.1
  Downloaded 1 crate (213.6 KB) in 0.40s
  Installing ripgrep v14.1.1
--snip--
   Compiling grep v0.3.2
    Finished `release` profile [optimized + debuginfo] target(s) in 6.73s
  Installing ~/.cargo/bin/rg
   Installed package `ripgrep v14.1.1` (executable `rg`)

Dòng áp chót của output hiển thị vị trí và tên của binary đã cài đặt — trong trường hợp ripgrep thì đó là rg. Miễn là thư mục cài đặt nằm trong $PATH của bạn như đã đề cập trước đó, bạn có thể chạy ngay rg --help và bắt đầu dùng một công cụ tìm kiếm file nhanh hơn và "Rustier" hơn!
