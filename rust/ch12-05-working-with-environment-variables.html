Làm việc với biến môi trường

Ta sẽ nâng cấp binary minigrep bằng cách thêm một tính năng bổ sung: tuỳ chọn tìm kiếm không phân biệt hoa thường mà người dùng có thể bật lên qua một biến môi trường. Ta có thể biến tính năng này thành một tham số dòng lệnh và bắt người dùng nhập vào mỗi khi muốn dùng, nhưng bằng cách biến nó thành biến môi trường, ta cho phép người dùng đặt biến một lần và mọi lần tìm kiếm trong phiên terminal đó đều không phân biệt hoa thường.

Viết test thất bại cho tìm kiếm không phân biệt hoa thường

Đầu tiên ta thêm một hàm search_case_insensitive mới vào thư viện minigrep — hàm này sẽ được gọi khi biến môi trường có giá trị. Ta tiếp tục tuân theo quy trình TDD, vì vậy bước đầu tiên lại là viết một test thất bại. Ta sẽ thêm một test mới cho hàm search_case_insensitive mới và đổi tên test cũ từ one_result thành case_sensitive để làm rõ sự khác biệt giữa hai test, như trong Listing 12-20.

Lưu ý ta cũng đã chỉnh sửa contents của test cũ. Ta thêm một dòng mới với nội dung "Duct tape." dùng chữ hoa D — dòng này không nên khớp với query "duct" khi tìm kiếm phân biệt hoa thường. Việc thay đổi test cũ như vậy giúp đảm bảo ta không vô tình phá vỡ chức năng tìm kiếm phân biệt hoa thường đã được hiện thực. Test này vẫn phải vượt qua ngay bây giờ và tiếp tục vượt qua khi ta làm việc với tìm kiếm không phân biệt hoa thường.

Test mới cho tìm kiếm không phân biệt hoa thường dùng "rUsT" làm query. Trong hàm search_case_insensitive ta sắp thêm, query "rUsT" phải khớp với dòng chứa "Rust:" có chữ hoa R và khớp cả dòng "Trust me." dù cả hai có cách viết hoa thường khác với query. Đây là test thất bại của ta, và nó sẽ không compile được vì ta chưa định nghĩa hàm search_case_insensitive. Hãy thoải mái thêm một phần hiện thực bộ khung luôn trả về vector rỗng, tương tự cách ta làm với hàm search ở Listing 12-16, để xem test compile và thất bại.

Hiện thực hoá hàm search_case_insensitive

Hàm search_case_insensitive, như trong Listing 12-21, sẽ gần giống hệt hàm search. Điểm khác biệt duy nhất là ta sẽ lowercase cả query và từng line, để dù đối số đầu vào có kiểu chữ hoa thường như thế nào, chúng cũng sẽ cùng kiểu khi ta kiểm tra dòng có chứa query không.

Đầu tiên, ta lowercase chuỗi query và lưu vào một biến mới cùng tên, che khuất query ban đầu. Gọi to_lowercase trên query là cần thiết để dù query của người dùng là "rust", "RUST", "Rust", hay "rUsT", ta đều xử lý nó như thể là "rust" và không quan tâm đến hoa thường. Mặc dù to_lowercase xử lý được Unicode cơ bản, nó không đảm bảo 100% chính xác. Nếu đây là ứng dụng thực tế, ta sẽ cần làm kỹ hơn, nhưng phần này nói về biến môi trường chứ không phải Unicode, nên ta tạm để vậy.

Lưu ý rằng query bây giờ là một String thay vì string slice vì gọi to_lowercase tạo ra dữ liệu mới thay vì tham chiếu dữ liệu hiện có. Lấy ví dụ query là "rUsT": string slice đó không chứa chữ u hay t viết thường để ta dùng, vì vậy ta phải cấp phát một String mới chứa "rust". Khi ta truyền query làm đối số cho method contains, ta cần thêm dấu ampersand vì chữ ký của contains được định nghĩa nhận string slice.

Tiếp theo, ta thêm lời gọi to_lowercase trên mỗi line để lowercase tất cả các ký tự. Giờ khi đã chuyển cả line và query về chữ thường, ta sẽ tìm được các khớp dù query có kiểu chữ như thế nào.

Tuyệt! Cả hai đều vượt qua. Giờ hãy gọi hàm search_case_insensitive mới từ hàm run. Đầu tiên, ta thêm một tuỳ chọn cấu hình vào struct Config để chuyển đổi giữa tìm kiếm phân biệt và không phân biệt hoa thường. Việc thêm field này sẽ gây lỗi compiler vì ta chưa khởi tạo field đó ở bất kỳ đâu:

Tên file: src/main.rs

use std::env;
use std::error::Error;
use std::fs;
use std::process;

use minigrep::{search, search_case_insensitive};

// --snip--


fn main() {
    let args: Vec<String> = env::args().collect();

    let config = Config::build(&args).unwrap_or_else(|err| {
        println!("Problem parsing arguments: {err}");
        process::exit(1);
    });

    if let Err(e) = run(config) {
        println!("Application error: {e}");
        process::exit(1);
    }
}

pub struct Config {
    pub query: String,
    pub file_path: String,
    pub ignore_case: bool,
}

impl Config {
    fn build(args: &[String]) -> Result<Config, &'static str> {
        if args.len() < 3 {
            return Err("not enough arguments");
        }

        let query = args[1].clone();
        let file_path = args[2].clone();

        Ok(Config { query, file_path })
    }
}

fn run(config: Config) -> Result<(), Box<dyn Error>> {
    let contents = fs::read_to_string(config.file_path)?;

    let results = if config.ignore_case {
        search_case_insensitive(&config.query, &contents)
    } else {
        search(&config.query, &contents)
    };

    for line in results {
        println!("{line}");
    }

    Ok(())
}

Ta đã thêm field ignore_case kiểu Boolean. Tiếp theo, hàm run cần kiểm tra giá trị của field ignore_case để quyết định gọi hàm search hay search_case_insensitive, như trong Listing 12-22. Code này vẫn chưa compile được.

Cuối cùng, ta cần kiểm tra biến môi trường. Các hàm làm việc với biến môi trường nằm trong module env của thư viện chuẩn, vốn đã được import ở đầu src/main.rs. Ta sẽ dùng hàm var từ module env để kiểm tra xem có giá trị nào được đặt cho biến môi trường tên IGNORE_CASE không, như trong Listing 12-23.

Ở đây, ta tạo một biến mới ignore_case. Để gán giá trị cho nó, ta gọi hàm env::var và truyền vào tên biến môi trường IGNORE_CASE. Hàm env::var trả về một Result — variant Ok thành công chứa giá trị của biến môi trường nếu nó được đặt, hoặc variant Err nếu biến môi trường không được đặt.

Ta dùng method is_ok trên Result để kiểm tra biến môi trường có được đặt không, nghĩa là chương trình sẽ thực hiện tìm kiếm không phân biệt hoa thường. Nếu biến môi trường IGNORE_CASE không được đặt thành bất kỳ giá trị gì, is_ok sẽ trả về false và chương trình sẽ tìm kiếm phân biệt hoa thường. Ta không quan tâm đến giá trị của biến môi trường, chỉ cần biết nó có được đặt hay không, nên ta dùng is_ok thay vì unwrap, expect, hay các method khác trên Result.

Ta truyền giá trị trong biến ignore_case vào instance Config để hàm run có thể đọc giá trị đó và quyết định gọi search_case_insensitive hay search, như ta đã hiện thực ở Listing 12-22.

Hãy thử! Đầu tiên, ta chạy chương trình không có biến môi trường và với query to — query này phải khớp với những dòng chứa từ to viết thường:

$ cargo run -- to poem.txt
   Compiling minigrep v0.1.0 (file:///projects/minigrep)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.0s
     Running `target/debug/minigrep to poem.txt`
Are you nobody, too?
How dreary to be somebody!

Vẫn hoạt động! Giờ hãy chạy chương trình với IGNORE_CASE được đặt thành 1 nhưng với cùng query to:

$ IGNORE_CASE=1 cargo run -- to poem.txt

Nếu bạn đang dùng PowerShell, bạn cần đặt biến môi trường và chạy chương trình thành hai lệnh riêng biệt:

PS> $Env:IGNORE_CASE=1; cargo run -- to poem.txt

Cách này sẽ khiến IGNORE_CASE tồn tại suốt phiên shell còn lại. Có thể xoá nó bằng cmdlet Remove-Item:

PS> Remove-Item Env:IGNORE_CASE

Ta sẽ nhận được các dòng chứa to có thể có chữ hoa:

Are you nobody, too?
How dreary to be somebody!
To tell your name the livelong day
To an admiring bog!

Xuất sắc, ta còn lấy được cả các dòng chứa To viết hoa! Chương trình minigrep của ta giờ có thể tìm kiếm không phân biệt hoa thường được điều khiển bởi biến môi trường. Bây giờ bạn đã biết cách quản lý các tuỳ chọn được đặt qua cả tham số dòng lệnh lẫn biến môi trường.

Một số chương trình cho phép dùng cả đối số lẫn biến môi trường cho cùng một cấu hình. Trong những trường hợp đó, chương trình quyết định cái nào được ưu tiên. Như một bài tập thêm cho bản thân, hãy thử kiểm soát độ nhạy hoa thường qua cả tham số dòng lệnh lẫn biến môi trường. Hãy quyết định xem tham số dòng lệnh hay biến môi trường sẽ được ưu tiên khi chương trình được chạy với một cái đặt phân biệt hoa thường và cái kia đặt bỏ qua hoa thường.

Module std::env còn chứa nhiều tính năng hữu ích khác để làm việc với biến môi trường: hãy xem tài liệu của nó để biết thêm những gì có sẵn.
