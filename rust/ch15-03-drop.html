Chạy Code Khi Dọn Dẹp Với Trait Drop

Trait quan trọng thứ hai trong mô hình smart pointer là Drop — nó cho phép bạn tuỳ chỉnh điều gì xảy ra khi một giá trị sắp ra khỏi scope. Bạn có thể cung cấp implementation cho trait Drop trên bất kỳ kiểu nào, và code đó có thể được dùng để giải phóng tài nguyên như files hay kết nối mạng.

Ta giới thiệu Drop trong bối cảnh smart pointers vì chức năng của trait Drop hầu như lúc nào cũng được dùng khi implement smart pointer. Ví dụ, khi Box<T> bị drop, nó sẽ deallocate vùng nhớ trên heap mà box đó đang trỏ tới.

Trong một số ngôn ngữ, với một số kiểu, lập trình viên phải tự gọi code để giải phóng bộ nhớ hay tài nguyên mỗi khi dùng xong instance của kiểu đó. File handles, sockets, locks đều là những ví dụ điển hình. Nếu lập trình viên quên, hệ thống có thể bị quá tải và crash. Với Rust, bạn có thể chỉ định một đoạn code chạy mỗi khi giá trị ra khỏi scope, và compiler sẽ chèn code đó vào tự động. Kết quả là bạn không cần lo lắng về việc đặt cleanup code khắp nơi trong chương trình — bạn sẽ không bao giờ bị rò rỉ tài nguyên!

Bạn chỉ định code cần chạy khi giá trị ra khỏi scope bằng cách implement trait Drop. Trait Drop yêu cầu bạn implement một method tên drop nhận mutable reference tới self. Để xem khi nào Rust gọi drop, hãy implement drop với các câu lệnh println! trước đã.

Listing 15-14 cho thấy struct CustomSmartPointer với chức năng tuỳ chỉnh duy nhất là in ra Dropping CustomSmartPointer! khi instance ra khỏi scope, để thấy Rust chạy method drop lúc nào.

Trait Drop đã có trong prelude nên ta không cần đưa nó vào scope. Ta implement trait Drop trên CustomSmartPointer và cung cấp implementation cho method drop gọi println!. Body của method drop là nơi bạn đặt bất kỳ logic nào muốn chạy khi instance của kiểu ra khỏi scope. Ta in một đoạn text ở đây để trực quan hoá thời điểm Rust gọi drop.

Trong main, ta tạo hai instance của CustomSmartPointer rồi in CustomSmartPointers created. Cuối main, các instance CustomSmartPointer sẽ ra khỏi scope và Rust gọi code ta đặt trong method drop, in ra thông điệp cuối cùng. Lưu ý rằng ta không cần gọi method drop tường minh.

Khi chạy chương trình này, ta sẽ thấy output sau:

$ cargo run
   Compiling drop-example v0.1.0 (file:///projects/drop-example)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.60s
     Running `target/debug/drop-example`
CustomSmartPointers created
Dropping CustomSmartPointer with data `other stuff`!
Dropping CustomSmartPointer with data `my stuff`!

Rust tự động gọi drop cho ta khi các instance ra khỏi scope, chạy đúng code ta đã chỉ định. Các biến được drop theo thứ tự ngược lại với thứ tự tạo ra, nên d được drop trước c. Ví dụ này nhằm mục đích cho thấy trực quan cách method drop hoạt động — thông thường bạn sẽ chỉ định cleanup code mà kiểu của mình cần chạy, thay vì chỉ in thông điệp.

Thật tiếc, không đơn giản để tắt chức năng drop tự động. Tắt drop thường không cần thiết — toàn bộ ý nghĩa của trait Drop là nó được xử lý tự động. Tuy nhiên, đôi khi bạn muốn dọn dẹp giá trị sớm hơn. Một ví dụ điển hình là khi dùng smart pointers quản lý locks: bạn có thể muốn buộc method drop giải phóng lock để code khác trong cùng scope có thể acquire lock đó. Rust không cho phép bạn gọi method drop của trait Drop thủ công — thay vào đó, bạn phải gọi hàm std::mem::drop từ thư viện chuẩn nếu muốn buộc một giá trị bị drop trước khi kết thúc scope.

Thử gọi method drop của trait Drop thủ công bằng cách sửa hàm main từ Listing 15-14 sẽ không hoạt động, như Listing 15-15 cho thấy.

Khi thử compile code này, ta nhận được lỗi sau:

$ cargo run
   Compiling drop-example v0.1.0 (file:///projects/drop-example)
error[E0040]: explicit use of destructor method
  --> src/main.rs:16:7
   |
16 |     c.drop();
   |       ^^^^ explicit destructor calls not allowed
   |
help: consider using `drop` function
   |
16 -     c.drop();
16 +     drop(c);
   |

For more information about this error, try `rustc --explain E0040`.
error: could not compile `drop-example` (bin "drop-example") due to 1 previous error

Thông báo lỗi này nói rằng ta không được gọi drop tường minh. Lỗi dùng thuật ngữ destructor — thuật ngữ lập trình chung cho hàm dọn dẹp một instance. Destructor tương tự như constructor tạo ra instance. Hàm drop trong Rust là một destructor đặc biệt.

Rust không cho phép ta gọi drop tường minh vì Rust vẫn sẽ tự động gọi drop trên giá trị khi kết thúc main. Điều này sẽ gây ra lỗi double free vì Rust cố dọn dẹp cùng một giá trị hai lần.

Ta không thể tắt tự động chèn drop khi giá trị ra khỏi scope, và cũng không thể gọi method drop tường minh. Vì vậy, nếu cần buộc giá trị được dọn dẹp sớm, ta dùng hàm std::mem::drop.

Hàm std::mem::drop khác với method drop trong trait Drop. Ta gọi nó bằng cách truyền giá trị muốn buộc drop vào làm đối số. Hàm này có trong prelude, nên ta có thể sửa main trong Listing 15-15 để gọi hàm drop như trong Listing 15-16.

Chạy code này sẽ in ra:

$ cargo run
   Compiling drop-example v0.1.0 (file:///projects/drop-example)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.73s
     Running `target/debug/drop-example`
CustomSmartPointer created
Dropping CustomSmartPointer with data `some data`!
CustomSmartPointer dropped before the end of main

Dòng Dropping CustomSmartPointer with data `some data`! được in ra giữa CustomSmartPointer created và CustomSmartPointer dropped before the end of main, cho thấy code trong method drop được gọi để drop c đúng tại thời điểm đó.

Bạn có thể dùng code trong implementation trait Drop theo nhiều cách để làm việc cleanup tiện lợi và an toàn: ví dụ, bạn có thể dùng nó để tạo bộ cấp phát bộ nhớ của riêng mình! Với trait Drop và hệ thống ownership của Rust, bạn không phải nhớ dọn dẹp vì Rust làm điều đó tự động.

Bạn cũng không phải lo về các vấn đề phát sinh từ việc vô tình dọn dẹp giá trị đang còn được dùng: Hệ thống ownership đảm bảo references luôn hợp lệ cũng đảm bảo drop chỉ được gọi đúng một lần khi giá trị không còn được dùng nữa.

Giờ đã xem xét Box<T> và một số đặc điểm của smart pointers, hãy cùng khám phá thêm các smart pointers khác được định nghĩa trong thư viện chuẩn.
