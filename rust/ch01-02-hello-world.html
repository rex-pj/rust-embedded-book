👋 Hello, World!

Giờ bạn đã cài Rust xong, đã đến lúc viết chương trình Rust đầu tiên của bạn rồi. Theo truyền thống khi học một ngôn ngữ mới, người ta thường viết một chương trình nhỏ in ra dòng chữ Hello, world! lên màn hình — và chúng ta cũng sẽ làm điều tương tự!

Lưu ý: Cuốn sách này giả định bạn đã có hiểu biết cơ bản về dòng lệnh. Rust không yêu cầu bạn dùng công cụ soạn thảo hay công cụ phát triển cụ thể nào, cũng không quan tâm code của bạn lưu ở đâu, vì vậy nếu bạn muốn dùng IDE thay vì dòng lệnh, cứ thoải mái dùng IDE yêu thích của bạn. Nhiều IDE hiện đã có mức độ hỗ trợ Rust nhất định; hãy xem tài liệu của IDE để biết thêm chi tiết. Đội ngũ Rust đang tập trung cải thiện hỗ trợ IDE thông qua rust-analyzer. Xem Phụ lục D để biết thêm chi tiết.

📁 Thiết lập thư mục dự án

Bạn sẽ bắt đầu bằng cách tạo một thư mục để lưu mã Rust. Rust không quan tâm code của bạn ở đâu, nhưng với các bài tập và dự án trong sách này, chúng tôi gợi ý bạn tạo một thư mục projects trong thư mục home và giữ tất cả dự án ở đó.

Mở terminal và nhập các lệnh sau để tạo thư mục projects và thư mục cho dự án "Hello, world!" bên trong thư mục projects.

Trên Linux, macOS và PowerShell trên Windows, nhập:

$ mkdir ~/projects
$ cd ~/projects
$ mkdir hello_world
$ cd hello_world

Trên Windows CMD, nhập:

> mkdir "%USERPROFILE%\projects"
> cd /d "%USERPROFILE%\projects"
> mkdir hello_world
> cd hello_world

✍️ Kiến thức cơ bản về chương trình Rust

Tiếp theo, tạo một file nguồn mới và đặt tên là main.rs. Các file Rust luôn có phần mở rộng .rs. Nếu tên file gồm nhiều từ, quy ước là dùng dấu gạch dưới để phân tách. Ví dụ, dùng hello_world.rs thay vì helloworld.rs.

Bây giờ mở file main.rs bạn vừa tạo và nhập mã trong Listing 1-1.

Lưu file và quay lại cửa sổ terminal trong thư mục ~/projects/hello_world. Trên Linux hoặc macOS, nhập các lệnh sau để biên dịch và chạy file: 

$ rustc main.rs
$ ./main
Hello, world!

Trên Windows, nhập lệnh .\main thay vì ./main: 

> rustc main.rs
> .\main
Hello, world!

Dù bạn dùng hệ điều hành nào, chuỗi Hello, world! vẫn sẽ in ra terminal. Nếu bạn không thấy kết quả này, hãy quay lại phần "Kiểm tra và xử lý sự cố" trong phần Cài đặt để tìm cách trợ giúp.

Nếu Hello, world! đã in ra, chúc mừng! Bạn đã chính thức viết một chương trình Rust. Điều đó làm bạn trở thành một lập trình viên Rust — chào mừng!

🔬 Giải phẫu một chương trình Rust

Hãy xem lại chương trình "Hello, world!" này một cách chi tiết. Đây là phần đầu tiên của câu đố: 

fn main() {

}

Những dòng này định nghĩa một hàm tên là main. Hàm main rất đặc biệt: Đây luôn là đoạn mã đầu tiên chạy trong mọi chương trình Rust có thể thực thi. Ở đây, dòng đầu tiên khai báo hàm tên main không có tham số và không trả về gì. Nếu có tham số, chúng sẽ được đặt trong dấu ngoặc đơn ().

Thân hàm được bao bởi {}. Rust yêu cầu dấu ngoặc nhọn bao quanh tất cả thân hàm. Phong cách tốt là đặt dấu ngoặc nhọn mở cùng dòng với khai báo hàm, thêm một khoảng trắng ở giữa.

Lưu ý: Nếu bạn muốn tuân theo một phong cách chuẩn nhất quán trong các dự án Rust, bạn có thể dùng công cụ định dạng tự động gọi là rustfmt để định dạng mã theo phong cách cụ thể (thêm về rustfmt trong Phụ lục D). Đội ngũ Rust đã tích hợp công cụ này vào bản phân phối Rust chuẩn, giống như rustc, vậy nên nó đã được cài trên máy tính của bạn!

Thân hàm main chứa đoạn mã sau:

println!("Hello, world!");

Dòng này thực hiện toàn bộ công việc của chương trình nhỏ này: In văn bản ra màn hình. Có ba chi tiết quan trọng cần chú ý ở đây.

Thứ nhất, println! gọi một macro Rust. Nếu đó là gọi hàm thay vì macro, thì sẽ là println (không có dấu !). Macro Rust là cách viết mã để tạo ra mã khác nhằm mở rộng cú pháp Rust, và chúng ta sẽ thảo luận chi tiết hơn trong Chương 20. Hiện tại, bạn chỉ cần biết rằng dùng ! có nghĩa là bạn đang gọi một macro thay vì hàm thông thường, và macro không luôn tuân theo các quy tắc giống như hàm.

Thứ hai, bạn thấy chuỗi "Hello, world!". Chúng ta truyền chuỗi này làm đối số cho println!, và chuỗi được in ra màn hình.

Thứ ba, chúng ta kết thúc dòng bằng dấu chấm phẩy (;), cho biết biểu thức này đã kết thúc và biểu thức tiếp theo đã sẵn sàng bắt đầu. Hầu hết các dòng mã Rust đều kết thúc bằng dấu chấm phẩy.

⚡ Biên dịch và thực thi

Bạn vừa chạy một chương trình mới tạo, vậy hãy xem xét từng bước trong quá trình này.

Trước khi chạy một chương trình Rust, bạn phải biên dịch nó bằng trình biên dịch Rust bằng cách nhập lệnh rustc và truyền tên file nguồn cho nó, như sau: 

$ rustc main.rs

Nếu bạn có kiến thức về C hoặc C++, bạn sẽ nhận thấy điều này tương tự với gcc hoặc clang. Sau khi biên dịch thành công, Rust tạo ra một file thực thi nhị phân.

Trên Linux, macOS và PowerShell trên Windows, bạn có thể xem file thực thi bằng cách nhập lệnh ls trong shell: 

$ ls
main  main.rs

Trên Linux và macOS, bạn sẽ thấy hai file. Với PowerShell trên Windows, bạn sẽ thấy ba file giống như dùng CMD. Với CMD trên Windows, bạn sẽ nhập: 

> dir /B %= tùy chọn /B chỉ hiển thị tên file =%
main.exe
main.pdb
main.rs

Kết quả hiển thị file mã nguồn với phần mở rộng .rs, file thực thi (main.exe trên Windows, nhưng là main trên tất cả các nền tảng khác), và trên Windows còn có file chứa thông tin debug với phần mở rộng .pdb. Từ đây, bạn chạy file main hoặc main.exe, như sau: 

$ ./main # hoặc .\main trên Windows

Nếu file main.rs của bạn là chương trình "Hello, world!", dòng này sẽ in ra Hello, world! lên terminal.

Nếu bạn quen với ngôn ngữ động như Ruby, Python hoặc JavaScript, bạn có thể chưa quen với việc biên dịch và chạy chương trình là hai bước riêng biệt. Rust là ngôn ngữ biên dịch trước (ahead-of-time compiled), nghĩa là bạn có thể biên dịch chương trình và đưa file thực thi cho người khác, và họ có thể chạy nó dù không cài Rust. Nếu bạn đưa ai đó file .rb, .py hoặc .js, họ cần có cài sẵn Ruby, Python hoặc JavaScript (tương ứng). Nhưng với những ngôn ngữ đó, bạn chỉ cần một lệnh để biên dịch và chạy chương trình. Mọi thứ đều là sự đánh đổi trong thiết kế ngôn ngữ.

Chỉ biên dịch bằng rustc là ổn với các chương trình đơn giản, nhưng khi dự án của bạn phát triển, bạn sẽ muốn quản lý tất cả các tùy chọn và chia sẻ mã dễ dàng hơn. Tiếp theo, chúng ta sẽ giới thiệu công cụ Cargo, thứ sẽ giúp bạn viết các chương trình Rust thực tế.
