Semihosting

Semihosting là một cơ chế cho phép thiết bị nhúng thực hiện I/O trên host và chủ yếu được dùng để ghi log message ra host console. Semihosting yêu cầu một debug session và hầu như không cần gì khác (không cần thêm dây!) nên rất tiện để dùng. Nhược điểm là nó rất chậm: mỗi thao tác ghi có thể mất vài mili giây tùy thuộc vào hardware debugger (ví dụ ST-Link) bạn dùng.

Crate cortex-m-semihosting cung cấp API để thực hiện các thao tác semihosting trên thiết bị Cortex-M. Chương trình dưới đây là phiên bản semihosting của "Hello, world!": 

#![no_main]
#![no_std]

use panic_halt as _;

use cortex_m_rt::entry;
use cortex_m_semihosting::hprintln;

#[entry]
fn main() -> ! {
    hprintln!("Hello, world!").unwrap();

    loop {}
}

Nếu bạn chạy chương trình này trên phần cứng, bạn sẽ thấy thông điệp "Hello, world!" trong log của OpenOCD. 

$ openocd
(..)
Hello, world!
(..)

Bạn cần enable semihosting trong OpenOCD từ GDB trước:

(gdb) monitor arm semihosting enable
semihosting is enabled

QEMU hiểu các thao tác semihosting nên chương trình trên cũng hoạt động với qemu-system-arm mà không cần bắt đầu một debug session. Lưu ý rằng bạn cần truyền flag -semihosting-config cho QEMU để enable semihosting; các flag này đã được bao gồm trong file .cargo/config.toml của template. 

$ # chương trình này sẽ block terminal
$ cargo run
     Running `qemu-system-arm (..)
Hello, world!

Cũng có một thao tác semihosting exit có thể dùng để kết thúc tiến trình QEMU. Quan trọng: đừng dùng debug::exit trên phần cứng thực; hàm này có thể làm hỏng session OpenOCD của bạn và bạn sẽ không thể debug thêm chương trình nào cho đến khi khởi động lại OpenOCD. 

#![no_main]
#![no_std]

use panic_halt as _;

use cortex_m_rt::entry;
use cortex_m_semihosting::debug;

#[entry]
fn main() -> ! {
    let roses = "blue";

    if roses == "red" {
        debug::exit(debug::EXIT_SUCCESS);
    } else {
        debug::exit(debug::EXIT_FAILURE);
    }

    loop {}
}
$ cargo run
     Running `qemu-system-arm (..)

$ echo $?
1

Một mẹo cuối: bạn có thể đặt hành vi panicking thành exit(EXIT_FAILURE). Điều này cho phép bạn viết các run-pass test no_std có thể chạy trên QEMU.

Để tiện lợi, crate panic-semihosting có tính năng "exit" mà khi được enable sẽ gọi exit(EXIT_FAILURE) sau khi ghi thông điệp panic ra host stderr. 

#![no_main]
#![no_std]

use panic_semihosting as _; // features = ["exit"]

use cortex_m_rt::entry;
use cortex_m_semihosting::debug;

#[entry]
fn main() -> ! {
    let roses = "blue";

    assert_eq!(roses, "red");

    loop {}
}
$ cargo run
     Running `qemu-system-arm (..)
panicked at 'assertion failed: `(left == right)`
  left: `"blue"`,
 right: `"red"`', examples/hello.rs:15:5

$ echo $?
1

LƯU Ý: Để enable tính năng này trên panic-semihosting, sửa phần dependencies trong Cargo.toml nơi khai báo panic-semihosting: 

panic-semihosting = { version = "VERSION", features = ["exit"] }

trong đó VERSION là phiên bản mong muốn. Để biết thêm thông tin về dependency features, xem phần specifying dependencies trong Cargo book.
