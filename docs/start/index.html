Bắt đầu — Getting Started

Viết, build và chạy chương trình embedded Rust đầu tiên của bạn

Phần này hướng dẫn bạn từng bước từ việc thiết lập dự án đến chạy code trên phần cứng thực hoặc máy ảo QEMU. Mỗi chương tập trung vào một khía cạnh cụ thể của quy trình phát triển embedded.

Quy trình phát triển embedded

Vòng lặp phát triển Embedded Rust

Nội dung chương này

Phần "Bắt đầu" gồm các chương sau, được thiết kế để học theo thứ tự:

🖥️
QEMU — Chạy trên máy ảo
Chạy chương trình embedded mà không cần phần cứng thực. Sử dụng QEMU để giả lập Cortex-M. Thiết lập cortex-m-quickstart, memory.x, và cargo run.
🔌
Phần cứng thực — Real Hardware
Flash code lên STM32 thực. ST-Link probe, probe-rs, cấu hình .cargo/config.toml, xem output qua ITM/RTT.
📡
Semihosting
In debug output từ MCU qua debug probe đến terminal PC. Dùng hprintln! từ cortex-m-semihosting. Chậm nhưng tiện cho debug.
💥
Panicking — Xử lý panic
panic! trong no_std không có backtrace mặc định. Các crate: panic-halt, panic-reset, panic-semihosting. Viết #[panic_handler] tùy chỉnh.
⚠️
Ngoại lệ — Exceptions
Xử lý ngoại lệ phần cứng: HardFault, SysTick, ngắt ngoại vi. Dùng #[exception] và #[interrupt] attribute.
🗄️
Thanh ghi bộ nhớ
Memory-mapped I/O — điều khiển GPIO, UART qua thanh ghi. PAC được sinh từ SVD file. HAL bọc PAC với API an toàn.

Điều kiện tiên quyết

Trước khi bắt đầu, hãy đảm bảo bạn đã:

  1. Cài Rust toolchain qua rustup.rs
  2. Thêm target ARM: rustup target add thumbv7em-none-eabihf
  3. Cài cargo-generate: cargo install cargo-generate
  4. Cài QEMU (để chạy máy ảo): apt install qemu-system-arm hoặc brew install qemu
  5. Cài probe-rs (để flash phần cứng): cargo install probe-rs-tools
  6. Cài arm-none-eabi-gdb hoặc gdb-multiarch
Nếu bạn chỉ muốn thử nghiệm mà chưa có phần cứng, hãy bắt đầu với chương QEMU. Bạn có thể chạy toàn bộ quy trình phát triển chỉ với máy tính.

Template dự án khởi đầu

Cách nhanh nhất để bắt đầu là dùng template cortex-m-quickstart: 

# Tạo dự án mới từ template
cargo generate --git https://github.com/rust-embedded/cortex-m-quickstart \
    --name my-first-embedded-app
cd my-first-embedded-app

# Cấu trúc dự án được tạo:
# my-first-embedded-app/
# ├── .cargo/config.toml   ← target, runner
# ├── Cargo.toml           ← dependencies
# ├── memory.x             ← linker script
# ├── build.rs             ← copy memory.x
# └── src/
#     └── main.rs          ← entry point
