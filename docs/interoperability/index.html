Khả năng tương tác — Interoperability

Khả năng tương tác giữa Rust và code C luôn phụ thuộc vào việc chuyển đổi dữ liệu giữa hai ngôn ngữ. Vì mục đích này, có một module chuyên dụng trong stdlib gọi là std::ffi.

std::ffi cung cấp các định nghĩa kiểu cho các kiểu nguyên thủy của C như char, int, và long. Nó cũng cung cấp một số tiện ích để chuyển đổi các kiểu phức tạp hơn như string, ánh xạ cả &str và String sang các kiểu C dễ và an toàn hơn để xử lý.

Kể từ Rust 1.30, các tính năng của std::ffi có thể dùng trong core::ffi hoặc alloc::ffi tùy thuộc vào việc có liên quan đến memory allocation hay không. Crate cty và crate cstr_core cũng cung cấp các tính năng tương tự.

Kiểu Rust Trung gian Kiểu C
String CString char *
&str CStr const char *
() c_void void
u32 hoặc u64 c_uint unsigned int
v.v. … …

Một giá trị của kiểu nguyên thủy C có thể được dùng như một trong các kiểu Rust tương ứng và ngược lại, vì kiểu trước chỉ đơn giản là một type alias của kiểu sau. Ví dụ, đoạn code sau biên dịch được trên các platform mà unsigned int có độ dài 32-bit. 

fn foo(num: u32) {
    let c_num: c_uint = num;
    let r_num: u32 = c_num;
}

Tương tác với các build system khác

Một yêu cầu phổ biến khi tích hợp Rust vào dự án embedded của bạn là kết hợp Cargo với build system hiện có của bạn, chẳng hạn như make hoặc cmake.

Chúng tôi đang thu thập các ví dụ và use case cho vấn đề này tại issue tracker trong issue #61.

Tương tác với RTOS

Việc tích hợp Rust với RTOS như FreeRTOS hoặc ChibiOS vẫn đang trong quá trình phát triển; đặc biệt việc gọi các hàm RTOS từ Rust có thể phức tạp.
