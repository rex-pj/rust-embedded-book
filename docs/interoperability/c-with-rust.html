Một chút C trong Rust — A little C with your Rust

Việc dùng C hoặc C++ bên trong một project Rust bao gồm hai phần chính:

  • Bọc C API được expose để dùng với Rust
  • Build code C hoặc C++ của bạn để tích hợp với code Rust

Vì C++ không có ABI ổn định để compiler Rust nhắm đến, nên khuyến khích dùng ABI C khi kết hợp Rust với C hoặc C++.

Quy trình build: Rust + C → binary ELF

Định nghĩa giao diện

Trước khi dùng code C hoặc C++ từ Rust, cần phải định nghĩa (trong Rust) những kiểu dữ liệu và chữ ký hàm nào tồn tại trong code được link. Trong C hoặc C++, bạn sẽ include một file header (.h hoặc .hpp) định nghĩa điều này. Trong Rust, cần phải dịch thủ công các định nghĩa này sang Rust, hoặc dùng công cụ để tự động sinh ra chúng.

Đầu tiên, chúng ta sẽ đề cập đến việc dịch thủ công các định nghĩa này từ C/C++ sang Rust.

Bọc các hàm và kiểu dữ liệu C

Thông thường, các thư viện được viết bằng C hoặc C++ sẽ cung cấp một file header định nghĩa tất cả các kiểu và hàm được dùng trong các giao diện công khai. Một ví dụ file có thể trông như thế này: 

/* File: cool.h */
typedef struct CoolStruct {
    int x;
    int y;
} CoolStruct;

void cool_function(int i, char c, CoolStruct* cs);

Khi được dịch sang Rust, giao diện này sẽ trông như thế này:

/* File: cool_bindings.rs */
#[repr(C)]
pub struct CoolStruct {
    pub x: cty::c_int,
    pub y: cty::c_int,
}

extern "C" {
    pub fn cool_function(
        i: cty::c_int,
        c: cty::c_char,
        cs: *mut CoolStruct
    );
}

Hãy cùng xem xét định nghĩa này từng phần một để giải thích từng phần.

#[repr(C)]
pub struct CoolStruct { ... }

Theo mặc định, Rust không đảm bảo thứ tự, padding hay kích thước của dữ liệu được chứa trong một struct. Để đảm bảo tương thích với code C, chúng ta bao gồm attribute #[repr(C)], hướng dẫn Rust compiler luôn dùng các quy tắc giống C để tổ chức dữ liệu trong struct. 

pub x: cty::c_int,
pub y: cty::c_int,

Do sự linh hoạt trong cách C hoặc C++ định nghĩa int hoặc char, nên khuyến khích dùng các kiểu dữ liệu nguyên thủy được định nghĩa trong cty, sẽ ánh xạ các kiểu từ C sang các kiểu trong Rust. 

extern "C" { pub fn cool_function( ... ); }

Câu lệnh này định nghĩa chữ ký của một hàm sử dụng C ABI, gọi là cool_function. Bằng cách định nghĩa chữ ký mà không định nghĩa phần thân hàm, định nghĩa của hàm này sẽ phải được cung cấp ở nơi khác, hoặc được link vào thư viện hay binary cuối từ một static library. 

i: cty::c_int,
c: cty::c_char,
cs: *mut CoolStruct

Tương tự với kiểu dữ liệu ở trên, chúng ta định nghĩa các kiểu dữ liệu của các đối số hàm bằng cách dùng các định nghĩa tương thích với C. Chúng ta cũng giữ nguyên các tên đối số, để rõ ràng.

Chúng ta có một kiểu mới ở đây, *mut CoolStruct. Vì C không có khái niệm về reference của Rust (trông sẽ như: &mut CoolStruct), thay vào đó chúng ta có một raw pointer. Vì việc dereference pointer này là unsafe, và pointer có thể thực sự là một null pointer, cần phải cẩn thận để đảm bảo các đảm bảo thông thường của Rust khi tương tác với code C hoặc C++.

Tự động sinh giao diện

Thay vì tự sinh các giao diện này (có thể tẻ nhạt và dễ lỗi), có một công cụ gọi là bindgen sẽ thực hiện các chuyển đổi này tự động. Để biết hướng dẫn sử dụng bindgen, hãy tham khảo bindgen user's manual, tuy nhiên quy trình điển hình bao gồm:

  1. Thu thập tất cả các header C hoặc C++ định nghĩa các giao diện hoặc kiểu dữ liệu mà bạn muốn dùng với Rust.
  2. Viết file bindings.h có #include "..." mỗi file bạn thu thập ở bước một.
  3. Đưa file bindings.h này cùng với bất kỳ compilation flag nào được dùng để biên dịch code của bạn vào bindgen. Mẹo: dùng Builder.ctypes_prefix("cty") / --ctypes-prefix=cty và Builder.use_core() / --use-core để làm cho code sinh ra tương thích với #![no_std].
  4. bindgen sẽ tạo ra code Rust được sinh tới output của cửa sổ terminal. Output này có thể được pipe vào một file trong project của bạn, chẳng hạn như bindings.rs. Bạn có thể dùng file này trong Rust project để tương tác với code C/C++ được biên dịch và link như thư viện ngoài. Mẹo: đừng quên dùng crate cty nếu các kiểu trong bindings được sinh ra có tiền tố là cty.

Build code C/C++ của bạn

Vì Rust compiler không biết cách biên dịch code C hoặc C++ trực tiếp (hoặc code từ bất kỳ ngôn ngữ nào khác có giao diện C), cần phải biên dịch code không phải Rust của bạn trước.

Đối với các project embedded, điều này thường có nghĩa là biên dịch code C/C++ thành static archive (như cool-library.a), sau đó có thể được kết hợp với code Rust của bạn ở bước link cuối cùng.

Nếu thư viện bạn muốn dùng đã được phân phối dưới dạng static archive, thì không cần phải rebuild code của bạn. Chỉ cần chuyển đổi file header giao diện được cung cấp như mô tả ở trên, và include static archive vào lúc compile/link.

Nếu code của bạn tồn tại như một source project, sẽ cần phải biên dịch code C/C++ của bạn thành static library, bằng cách kích hoạt build system hiện có của bạn (như make, CMake, v.v.), hoặc bằng cách chuyển các bước biên dịch cần thiết để dùng một công cụ gọi là crate cc. Đối với cả hai bước này, cần phải dùng script build.rs.

Rust build.rs build scripts

Script build.rs là một file được viết bằng cú pháp Rust, được thực thi trên máy biên dịch của bạn, SAU KHI các dependency của project đã được build, nhưng TRƯỚC KHI project của bạn được build.

Tài liệu đầy đủ có thể được tìm thấy tại https://doc.rust-lang.org/cargo/reference/build-scripts.html. Script build.rs hữu ích cho việc sinh code (chẳng hạn qua bindgen), gọi đến các build system bên ngoài như Make, hoặc biên dịch trực tiếp C/C++ thông qua crate cc.

Kích hoạt build system bên ngoài

Đối với các project với project hoặc build system bên ngoài phức tạp, có thể dễ nhất là dùng std::process::Command để "shell out" tới các build system khác bằng cách duyệt qua các đường dẫn tương đối, gọi một lệnh cố định (như make library), và sau đó copy static library kết quả vào vị trí thích hợp trong thư mục build target.

Trong khi crate của bạn có thể nhắm đến nền tảng embedded no_std, build.rs của bạn chỉ thực thi trên máy biên dịch crate. Điều này có nghĩa là bạn có thể dùng bất kỳ crate Rust nào sẽ chạy trên compilation host của bạn.

Build code C/C++ với crate cc

Đối với các project có dependency hoặc độ phức tạp hạn chế, hoặc các project khó thay đổi build system để tạo ra static library (thay vì binary hoặc executable cuối cùng), có thể dễ hơn khi sử dụng crate cc, cung cấp giao diện Rust idiomatic cho compiler được cung cấp bởi host.

Trong trường hợp đơn giản nhất là biên dịch một file C đơn như dependency cho static library, một script build.rs ví dụ dùng crate cc sẽ trông như thế này: 

fn main() {
    cc::Build::new()
        .file("src/foo.c")
        .compile("foo");
}

File build.rs được đặt ở root của package. Sau đó cargo build sẽ biên dịch và thực thi nó trước khi build package. Một static archive có tên libfoo.a được sinh ra và đặt vào thư mục target.
