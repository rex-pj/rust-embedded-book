Rust with C

Việc dùng code Rust bên trong project C hoặc C++ chủ yếu bao gồm hai phần:

  • Tạo C-friendly API trong Rust
  • Nhúng Rust project vào build system bên ngoài

Ngoài cargo và meson, hầu hết các build system không có hỗ trợ Rust native. Vì vậy có lẽ tốt nhất là chỉ dùng cargo để biên dịch crate và các dependency của bạn.

Thiết lập project

Tạo một cargo project mới như thường lệ.

Có các flag để báo cho cargo emit ra một systems library, thay vì rust target thông thường. Điều này cũng cho phép bạn đặt tên output khác nhau cho thư viện, nếu bạn muốn nó khác với phần còn lại của crate. 

[lib]
name = "your_crate"
crate-type = ["cdylib"]      # Tạo dynamic lib
# crate-type = ["staticlib"] # Tạo static lib

Build C API

Vì C++ không có ABI ổn định để Rust compiler nhắm đến, chúng ta dùng C cho bất kỳ tương tác nào giữa các ngôn ngữ khác nhau. Điều này không ngoại lệ khi dùng Rust bên trong code C và C++.

#[no_mangle]

Rust compiler mangling tên symbol theo cách khác so với những gì native code linker mong đợi. Vì vậy, bất kỳ hàm nào Rust export để dùng bên ngoài Rust cần được báo là không bị mangle bởi compiler.

extern "C"

Theo mặc định, bất kỳ hàm nào bạn viết trong Rust sẽ dùng Rust ABI (cũng chưa được ổn định hóa). Thay vào đó, khi build các FFI API hướng ra ngoài, chúng ta cần báo cho compiler dùng system ABI.

Tùy thuộc vào nền tảng của bạn, bạn có thể muốn nhắm đến một phiên bản ABI cụ thể, được ghi lại trong Rust Reference.

Kết hợp các phần này lại với nhau, bạn sẽ có một hàm trông giống như thế này:

#[no_mangle]
pub extern "C" fn rust_function() {

}

Cũng giống như khi dùng code C trong Rust project, bây giờ bạn cần chuyển đổi dữ liệu từ và sang dạng mà phần còn lại của ứng dụng sẽ hiểu.

Link và bối cảnh project lớn hơn

Vậy thì, đó là một nửa vấn đề đã được giải quyết. Làm thế nào để dùng điều này?

Điều này phụ thuộc rất nhiều vào project và/hoặc build system của bạn

cargo sẽ tạo ra file my_lib.so/my_lib.dll hoặc my_lib.a, tùy thuộc vào nền tảng và cài đặt của bạn. Thư viện này có thể đơn giản là được link bởi build system của bạn.

Tuy nhiên, việc gọi một hàm Rust từ C yêu cầu một file header để khai báo chữ ký hàm.

Mỗi hàm trong Rust-ffi API cần có một hàm header tương ứng.

#[no_mangle]
pub extern "C" fn rust_function() {}

sẽ trở thành

void rust_function();

v.v.

Có một công cụ để tự động hóa quá trình này, gọi là cbindgen, phân tích code Rust của bạn rồi sinh ra các header cho các project C và C++ từ nó.

Ở thời điểm này, việc dùng các hàm Rust từ C đơn giản như include header và gọi chúng! 

#include "my-rust-project.h"
rust_function();
