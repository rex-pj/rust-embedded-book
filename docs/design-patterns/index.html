Design Patterns

Các mẫu thiết kế phổ biến khi viết HAL và BSP trong Rust embedded

Khi viết HAL (Hardware Abstraction Layer) hoặc BSP (Board Support Package) cho Rust embedded, có một số design patterns được cộng đồng áp dụng rộng rãi để đạt được API an toàn, rõ ràng và dễ dùng.

Kiến trúc lớp BSP → HAL → PAC → Hardware

Các lớp abstraction trong embedded Rust ecosystem

1. Builder Pattern — Cấu hình ngoại vi

Builder pattern rất phổ biến trong HAL để cấu hình ngoại vi với nhiều tùy chọn mà vẫn có giá trị mặc định hợp lý: 

// HAL cung cấp builder cho UART configuration
use stm32f3xx_hal::{serial::Serial, time::Bps};

let serial = Serial::new(
    dp.USART1,
    (tx_pin, rx_pin),
    9600.bps(),      // Baud rate
    clocks,
    &mut rcc.apb2,
);

// Hoặc với builder pattern đầy đủ:
let serial = Serial::builder(dp.USART1)
    .baud_rate(115200.bps())
    .word_length(WordLength::Bits8)
    .parity(Parity::None)
    .stop_bits(StopBits::One)
    .build((tx_pin, rx_pin), clocks, &mut rcc.apb2);

2. Typestate Pattern — GPIO và ngoại vi

Typestate pattern mã hóa trạng thái của ngoại vi vào type system, ngăn việc dùng sai cách tại compile time: 

// GPIO với typestate — không thể gọi set_high() trên pin input!
let mut gpioa = dp.GPIOA.split(&mut rcc.ahb);

// PA5 ban đầu là Floating Input
let pa5 = gpioa.pa5; // PA5<Input<Floating>>

// Chuyển sang Output
let mut led = pa5.into_push_pull_output(
    &mut gpioa.moder,
    &mut gpioa.otyper,
); // PA5<Output<PushPull>>

led.set_high().unwrap();  // OK: pin là output

// led.is_high() ← Compile error! is_high() chỉ có trên Input pin
// → Xem thêm: static-guarantees/typestate-programming.html

3. Extension Traits — Mở rộng PAC API

Extension traits cho phép thêm method tiện lợi vào các type từ PAC mà không cần fork PAC: 

// Định nghĩa extension trait
pub trait GpioExt {
    type Parts;
    fn split(self, ahb: &mut AHB) -> Self::Parts;
}

// Implement cho PAC type
impl GpioExt for pac::GPIOA {
    type Parts = GpioAParts;
    fn split(self, ahb: &mut AHB) -> GpioAParts {
        ahb.enr().modify(|_, w| w.iopaen().enabled());
        GpioAParts { pa0: PA0 { _mode: PhantomData }, ... }
    }
}

// User: dp.GPIOA.split(&mut rcc.ahb) — method mới trên PAC type
use stm32f3xx_hal::prelude::*; // import extension traits

4. Singleton Pattern — Ngăn trùng lặp peripheral

// PAC đảm bảo chỉ có thể lấy Peripherals một lần
let dp = pac::Peripherals::take().unwrap();
// Lần 2 sẽ panic:
// let dp2 = pac::Peripherals::take().unwrap(); // → None → panic

// Tại sao? Rust type system + runtime singleton đảm bảo
// không ai có thể có hai owner của cùng một peripheral.

5. Board Support Package (BSP)

BSP đóng gói toàn bộ setup cụ thể cho một board mạch, cho phép user khởi tạo board chỉ trong một vài dòng: 

// BSP cho STM32F3Discovery
use stm32f3_discovery::{leds::Leds, Switch};

let dp = pac::Peripherals::take().unwrap();
let mut rcc = dp.RCC.constrain();
let clocks = rcc.cfgr.freeze(&mut dp.FLASH.constrain().acr);
let gpioe = dp.GPIOE.split(&mut rcc.ahb);

// BSP cung cấp abstraction cho 8 LED có sẵn trên board
let mut leds = Leds::new(gpioe);

leds[0].on();   // Bật LED đầu tiên
leds[4].off();  // Tắt LED thứ 5

HAL Design Checklist

Khi thiết kế HAL, nên tuân theo các nguyên tắc sau:

Tiêu chí Hướng dẫn
Đặt tên nhất quán Dùng tên giống embedded-hal: set_high(), write(), read()
Singleton cho peripheral Dùng move semantics hoặc Option::take() để ngăn trùng lặp
Implement embedded-hal traits Cho GPIO, I2C, SPI, UART để driver portable dùng được
Typestate cho trạng thái Mã hóa mode (Input/Output, Enabled/Disabled) vào generic parameter
Zero unsafe trong public API Bọc unsafe bên trong, không để user phải dùng unsafe
Builder với defaults hợp lý Cung cấp giá trị mặc định phổ biến, không bắt buộc mọi tham số
Xử lý lỗi với Result Dùng Result<T, E>, không panic trong library code
Tham khảo HAL Design Guidelines chính thức của Rust Embedded Working Group để biết thêm chi tiết.
